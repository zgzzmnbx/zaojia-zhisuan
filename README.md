# 造价智算-工程造价辅助智能体

## 项目简介

造价智算是一个本地运行的工程造价辅助智能体。当前核心场景来自长输管道勘察测量最高投标限价编制，能力已经扩展为“规则引擎 + 经验池预警 + 大模型解释 + Word/Excel 成果输出”的造价数字员工雏形。

本项目不是让大模型直接猜价格。`基价 / 单价`、`实物工作费调整系数`、`技术工作费调整系数` 由本地二维知识库、规则表和经验提示层处理；右侧“智算”助手只负责问答、解释、风险报告、导出提示和行级复核。

面向普通读者、汇报对象、参赛评委或其他 AI 的项目说明，以本 README 和 `00-PRD/00-产品总览.md` 为准；早期一页式介绍已归档到 `docs/归档/项目介绍-给人看的版本-历史.md`。

## 当前能力概览

### Excel 填价与三数字匹配

- 输入标准控制价 Excel，识别空单价、空白价格或包含“空单价”的价格列。
- 主界面采用两阶段体验：点击“开始转换”先生成待匹配预览；在预览窗口点击“批量匹配”后，再执行基价 / 单价、实物工作费调整系数和技术工作费调整系数的批量识别填写。
- 使用本地二维知识库匹配 `基价 / 单价`。
- 技术工作费调整系数走第一层标准规则；实物工作费调整系数第一层暂未启用。
- 第一层命中标绿，第二层经验提示标黄，未命中 / 待复核标红。

详细需求见：

```text
00-PRD/01-模块PRD/01-Excel填价与三数字匹配/PRD.md
```

### 填价结果预览与输出

- 输出回填后的 Excel 和 Word 报告。
- 预览支持多 sheet、表头行偏好、显示列偏好、隐藏指定列无值行、最大列宽、按 `sheet 名 + 列标题` 记忆手动列宽、公式兜底计算；表格预览、列映射、工作量抓取和问问智算窗口的项目默认值统一来自 `config/project-default-settings.json`，弹窗内修改只临时生效，新用户和绿色版首次打开沿用该项目默认口径。
- 下载 Excel 与填价结果预览共用隐藏空行口径；第一列以 `合计` 开头的汇总行始终保留。
- 结果预览支持人工修改输出 Excel 副本的普通值单元格；保存单元格时快速写入输出 Excel 和预览，必要时再手动点击“重算公式”刷新汇总公式和 Word 报告，并在 Excel 单元格和运行目录留存修改记录。
- “开始转换”只生成待匹配预览，不自动跳页；右侧智算引导进入表格预览，预览窗口、智算按钮、快捷指令和输入“批量匹配”共用同一批量匹配动作。

详细需求见：

```text
00-PRD/01-模块PRD/02-填价结果预览与输出/PRD.md
```

### 辅助填价与人工复核闭环

- 当前已完成基价 / 单价辅助填价 MVP：结果预览固定 `AI` 列中的书本图标打开弹窗，展示知识库相似候选、经验池同类均值候选和当前行价格轻量追溯。
- 辅助填价只做候选组织和人工确认写入，不让大模型生成数字，不改变主匹配引擎，不反写原始输入、知识库、规则表或经验池。
- 后续已独立成 P1 模块：扩展到实物工作费调整系数、技术工作费调整系数，补齐自定义值备注、分字段人工确认、候选配置和复核处理状态。

### 经验池预警

- 经验池独立于正式知识库，只用于转换后比选预警，不参与主匹配和第二层经验提示。
- 初次转换不自动运行预警，用户点击“运行经验池预警分析”后才写入 `预警参数`、`预警细节`。
- 当前同类识别按 `字段完全匹配 -> 非空要素顺序匹配 -> 未找到可比选同类记录` 执行。
- v5.4.0 后，经验池页新增只读治理报告，可检查重复记录、单位不一致、样本不足、极端离散、关键字段空值和经验数不可解析，并生成 Markdown 报告；治理不自动清洗、不反写正式知识库、不改变预警算法。

详细需求见：

```text

```

### 问问智算 AI 助手

- 大模型不参与价格裁决，只用于风险报告、行级复核、问答、导出提示和过程解释。
- v5.4.0 后，后端提供结构化风险清单，把待复核行和经验池偏离统一为可跳转风险项；Word 报告页可查看风险清单，原大模型风险报告仍只做说明和润色。
- 知识库问答先由本地程序检索 Markdown / Excel / CSV 证据，再交给 DeepSeek 做依据解释。
- 用户以 `查库：`、`查库:`、`@知识库`、`#知识库` 开头提问时，会强制进入知识库检索模式，界面显示“已调用知识库”。
- `@知识库` 已按三层方案优化：补充功能类知识卡，行级复核固定携带关键诊断字段，风险报告、导出、行级 AI 等功能类问题可命中本地证据。
- 价格类 `@知识库` 问答优先检索二维价格库；条件明确时引用精确行，条件模糊时列出相似候选项供用户选择，不让大模型凭空报价格。

### 运行入口

- 当前对外交付维护三类运行入口：传统开发版、Windows 绿色版、Windows Tauri 桌面版。
- 传统开发版用于本机开发调试：后端 `8000`、前端 Vite `5174`，依赖本机 Python 和 Node 环境。
- Windows 绿色版用于发给其他 Windows 电脑直接运行：仍然是后端 `8000`、前端 Vite `5174` 传统双服务启动，但随包携带 Python、Node 和前端依赖。
- Windows Tauri 桌面版用于发给其他 Windows 电脑直接双击 exe：随包携带 Python 和后端依赖，不需要目标电脑安装 Python、Node、npm、Rust 或 Cargo；前端使用 `frontend/dist`，由本地 FastAPI 静态托管。
- 绿色版和 Tauri 桌面版打包时会复制项目根目录现有 `.env.local`，便于目标电脑直接使用问问智算；`.env.local` 仍不写入源码和代码存档。

## 开发运行

后端：

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

前端：

```powershell
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5174
```

打开：

```text
http://127.0.0.1:5174/
```

一键启动：

```text
启动造价智算.bat
```

大模型 API Key 放在项目根目录 `.env.local`，官方 DeepSeek 使用：

```text
DEEPSEEK_API_KEY=你的key
```

`.env.local` 只用于本机运行，不进入代码存档。

## 版本存档

代码存档：

双击：

```text
备份代码.bat
```

或运行：

```powershell
python tools/archive_code.py
```

规则：

- 每个版本完成时运行一次代码存档。
- 存档只打包代码和恢复所需上下文，不打包比赛资料、知识库原始大目录、PPT 和运行临时输出。
- `01-assets/01-UI参考图/01-assets/` 与 `01-assets/01-UI参考图/00-reference/` 属于 UI 图片素材和参考资料，不进入代码版存档；`02-notes/` 和静态 demo HTML 可保留在存档中。
- `tools/archive_code.py` 会在生成根目录 `04-输出版本存档` 内的压缩包后，同步复制一份到 `D:\Desktop\01-工作流-【造价】\工作流\260612-【ai大赛】-管勘智算-V2.0\04-输出版本存档`，作为工作流目录副本。
- 存档过程必须通过脚本完成，脚本运行结束后不要再额外检查 zip 是否存在、工作流副本是否复制成功、SHA256 是否一致或目录是否完整；是否存档成功由大尾巴人工查看。
- 改完代码后默认不自动重启 `8000` / `5174` 服务，除非用户明确要求或改了启动器、端口、环境变量、依赖、后端启动逻辑。

### 三类导出 / 备份脚本区别

| 脚本 | 用途 | 输出 |
| --- | --- | --- |
| `备份代码.bat` / `tools/archive_code.py` | 代码版恢复存档，版本完成时用 | `04-输出版本存档/造价智算-yyyy-mm-dd-v版本.zip` |
| `导出给其他AI查看-核心代码与规则.bat` / `tools/export_ai_review_bundle.py` | 给其他 AI 阅读评判，合并前端、后端、PRD 和规则说明 | `04-输出版本存档/给其他AI查看-核心代码与规则-yyyy-mm-dd/` |
| `备份PRD.bat` / `tools/archive_prd.py` | 手动备份当前 PRD Markdown 快照，便于回看旧版产品设计 | `00-PRD/archive/yyyy-mm-dd-vN/` |

## 上下文维护规则

- `README.md` 只做项目入口、当前状态摘要和关键链接，不再承载模块细节流水账。
- `00-PRD/` 写产品意图、模块边界、需求状态和验收口径。
- UI 参考图、Bug 截图、参考设计、技术经验、踩坑记录和 UI 小样实验统一进入 `01-assets/01-UI参考图/`；其中外部参考图放 `00-reference/`，自制设计稿 / Bug 图放 `01-assets/`，分析笔记放 `02-notes/`，静态原型放 `code/`，正式代码仍在 `frontend/`、`backend/`、`src-tauri/`。
- `CHANGELOG.md` 写已经完成的实际变化；每次集中修改后运行一次 `python tools/trim_changelog.py`，保持最新 3 条记录，更早内容进入 `CHANGELOG - 【归档】.md`。
- `AGENTS.md` 写协作规则、重要文件入口和长期注意事项。
- `docs/` 写绿色版、兼容性和历史计划等专项说明。
- 项目根目录只保留源码、规则资料、正式资料入口、文档和必要启动文件；临时缓存、Playwright 检查输出、pytest 缓存、临时外包件、截图 / 过程产物等统一放到 `Codex-Temp/` 下。`node_modules` 是根目录 npm 依赖，不按临时文件移动。
- 旧任务看板已归档到 `docs/归档/TASKS-归档-2026-06-18.md`，日常不再读取。
