### 两类调整系数的定义

`实物工作费附加调整系数` 是对**自然条件、作业内容、作业方式、复杂程度差异**进行修正的系数。标准总则明确指出，这类系数只用于**实物工作费**的计算，不再对技术工作费重复乘用。fileciteturn0file2

`技术工作费调整系数` 本质上是**技术工作费费率或费率映射值**。不同专业并不统一：工程测量为 22%，岩土工程勘察按甲/乙/丙为 120%/100%/80%，水文地质勘察按项目类别与复杂程度分档，工程水文气象勘察为 22%，工程物探为 22%，室内试验为 10%。因此，这不是一个“全项目通用的单一字典”，而是一个**按专业分支分流**的映射体系。fileciteturn0file2

### 与 H-基价匹配逻辑的关系

程序应把 `H-基价` 的匹配与两类调整系数的匹配分成三个阶段，而不是一个混合模型：

```text
阶段一：先匹配基价
阶段二：再识别实物工作费附加调整因素并形成综合附加调整系数
阶段三：最后根据专业与类别匹配技术工作费调整系数
```

标准总则 1.0.8 以及各专业技术工作费章节共同支持以下统一公式框架：

```text
调整后实物工作费小计 = 基价 × 工程量 × 实物工作费附加调整系数
技术工作费小计 = 调整后实物工作费小计 × 技术工作费调整系数
金额合计 = 调整后实物工作费小计 + 技术工作费小计
```

但表2中引用 2009 测绘成本定额的个别行，样表未单独计取技术工作费小计；这类行需要单独建模为“技术系数可填但默认不参与小计计算”的特殊规则。fileciteturn0file2turn0file3

## 实物工作费附加调整系数规则.md

### 规则总览

`实物工作费附加调整系数` 的程序实现口径应统一为：

```text
先识别单项附加因素
再按标准组合规则形成综合附加调整系数
综合系数仅作用于实物工作费
技术工作费不再重复乘附加调整系数
```

《使用手册》总则 1.0.8 明确：当附加调整系数为两个或两个以上时，**不能连乘**，而应“各附加调整系数相加，减去附加调整系数的个数，再加上定值 1”，并且“附加调整系数只用于计算工程勘察实物工作收费，计算工程勘察技术工作费不再乘以附加调整系数”。这条是本规则引擎的最高优先级算法依据。fileciteturn0file2

### 表2-通用工程测量费用规则

表2可以分成三类来源完全不同的业务块，因此不能用一个统一口径硬套：

| 业务块 | 典型行 | 实物工作费附加调整系数来源 | 处理方式 |
|---|---|---|---|
| 工程测量标准行 | 控制测量、地形图测绘、断面测量、地下管线测量 | 2002 使用手册第2章地面测量/水域测量附加调整系数 | 可自动匹配 |
| 2009 成本定额行 | 航测、像控点联测、地物地貌调绘、DLG/DEM/DOM、地图编制 | 2009 定额基价 + 样表经验折增折减 | 需要“样表覆盖表” |
| 工点特化行 | 山岭隧道、站场、阀室、穿越等 | 标准系数 + 工点备注 + 组合算法 | 标准规则与样表覆盖并用 |

表2中高置信规则有三类。第一类是**不造标系数 0.6**。地面测量实物工作收费附加调整系数表明确列有“二、三、四等三角（边）不造标=0.6”和“GPS 测量 C 级、D 级、E 级不造标=0.6”；样表中首级控制测量和多类静态/RTK 行都使用了 `0.6`。其中 GPS C/D/E 非造标属于标准明确，高置信；但样表把 `0.6` 用到了“三角（边）一级”RTK 行，这一部分缺少同表直接条文支撑，只能作为样表类比规则。第二类是**数字化测绘系数 1.5**。表 2.2-3 明确给出了“数字化测绘=1.5”。第三类是**覆盖/隐蔽度大于 60% 的系数区间 1.2~1.5**。样表多处实际使用了 `1.3`，因此程序应把“存在植被覆盖或隐蔽度高关键词”与“具体取值 1.3”分开处理：前者属于标准明确，后者属于样表取值策略。fileciteturn0file2

表2中的低置信样表经验主要集中在 2009 定额行。比如“航线网布点核减15%→0.85”“室内外综合判调，定额核减30%→0.70”“室内外综合判调增加20%+带宽系数15%→1.35”，这些不是你提供标准中的直接条文，而是样表对 2009 定额条目的项目化折增折减处理。程序应把这类规则单独放在 `survey_sample_overrides` 表中，来源类型标为 `样表经验`，并允许项目管理员在后台开启、关闭或修改。`1.35` 的形成方式与总则 1.0.8 一致，即 `1 + (1.20-1) + (1.15-1) = 1.35`。fileciteturn0file2turn0file3

### 表3-地质测绘规则

表3的结构最清楚，也最适合自动化。标准依据来自岩土工程勘察第 3 章：工程地质测绘实物工作收费基价表给出按成图比例和简单/中等/复杂分档的基价；同页明确列出“带状工程地质测绘，附加调整系数为 1.3”，以及“工程地质测绘与地质测绘同时进行，附加调整系数为 1.5”。因此，表3的 `AC` 列可以拆成三条程序规则：

| 规则 | 系数 | 来源类型 | 置信度 |
|---|---:|---|---|
| 无附加因素 | 1.0 | 标准明确 | 高 |
| 带状工程地质测绘 | 1.3 | 标准明确 | 高 |
| 工程地质测绘与地质测绘同时进行 | 1.5 | 标准明确 | 高 |

当前样表有两个现象需要区分。其一，行 12~13 的备注直接写了“带状工程地质测绘系数 1.3”，这一部分可以高置信自动化。其二，行 7、10、11 也写了 `1.3`，但没有同时保留足够的说明文本，而且部分行工程量为空，因此如果未来只根据行内容而没有项目上下文验证，不建议仅凭比例尺直接断定 `1.3`，应要求命中“带状/线路/廊道”等上下文触发再自动填值。该部分应设为“标准规则可用，但当前行级证据不足时待复核”。fileciteturn0file2

### 表4-通用工程勘察费用规则

表4的实物工作费附加调整系数最复杂，因为它把岩土工程勘察、水文地质勘察、工程水文、工程物探、室内试验等多个专业揉进了一个 sheet。程序上应分专业子树处理，不应把整个表4当单一规则集。

### 组合规则

组合规则必须写成代码规则，而不应硬编码在二维表里。

```text
如果命中 0 个附加因素，则综合附加调整系数 = 1
如果命中 1 个附加因素，则综合附加调整系数 = 该系数
如果命中 2 个及以上附加因素，且都属于可叠加的“附加调整系数”家族，
则综合附加调整系数 = 1 + Σ(单项系数 - 1)
```

总则 1.0.8 已经明文否定“直接相乘”的做法，这一点必须作为默认算法。标准示例和样表都支持这个口径：例如 `1.5 + 1.3` 的组合应得 `1.8`，而不是 `1.95`。fileciteturn0file2

程序上的冲突处理建议如下：

| 场景 | 默认处理 | 原因 |
|---|---|---|
| 两个以上标准明确因素，且同属附加调整系数体系 | 按 `1 + Σ(k-1)` 组合 | 与总则一致 |
| 一个标准因素 + 一个样表经验因素，且样表已有明确合成值 | 用样表合成值 | 优先复现样表 |
| 一个标准因素 + 一个样表经验因素，但样表无合成值 | `待复核` | 避免误叠加 |
| 两个以上样表经验因素同时触发 | `待复核` | 风险高 |
| 区间值规则，例如 `1.2~1.5`、`1.1~1.3` | 先触发“存在该因素”，再按项目策略选值；无项目策略则待复核 | 区间本身不是唯一值 |
| 备注只写增减幅而未写最终系数 | 可按 `1 + Σ(增量)` 试算，但来源类型应为 `AI推断` 或 `样表经验` | 非直接标准值 |

### 待复核规则

以下情况程序不应自动落最终值，而应输出 `待复核`：

| 待复核场景 | 原因 | 建议输出 |
|---|---|---|
| 表2中“三角（边）一级”行使用 `0.6` | 标准表 2.2-3 直接覆盖的是二/三/四等三角（边）和 GPS C/D/E，不直接覆盖“三角（边）一级” | 候选值 `0.6`，来源=`样表经验` |
| 表2备注出现“植被覆盖系数0.3”而非“1.3” | 文本表达是增量还是完整系数不够统一 | 如果同时命中植被覆盖规则，则候选 `1.3` |
| 表2 2009 定额行同时出现多个折增折减 | 标准未给出统一叠加规则 | 输出候选组合并待复核 |
| 表3出现 `AC=1.3` 但无“带状”上下文 | 标准条件不足以唯一定位 | 默认候选 `1.3`，需项目经理确认 |
| 表4物探项 `1.7/1.5/1.3` 无备注来源 | 可能是地形/障碍综合，也可能是线路作业、双工况或行业参考值 | 输出候选值并指向备注/项目说明核验 |
| 表4样表 row113 之类特殊行 | 系数口径与普通行不一致，且来源为 2022 资料 | 一律待复核 |
| 同一行既命中标准提高系数，又命中标准降低系数 | 跨家族组合风险高 | 待复核 |
| 触发条件只来自项目名或备注模糊词，如“复杂”“特殊”“市场价” | 不能唯一对应标准系数 | 待复核 |

## 技术工作费调整系数规则.md

### 规则总览

技术工作费调整系数不是“单一映射表”，而是**按专业分支选择映射表**。程序分流顺序建议如下：

```text
先判定 sheet
再判定业务大类
再判定类别字段的语义
最后映射技术工作费调整系数
```

标准依据显示：工程测量技术工作费收费比例为 22%；岩土工程勘察技术工作费按甲/乙/丙为 120%/100%/80%；水文地质勘察按项目类别与复杂程度分为两套比例；工程水文气象勘察为 22%；工程物探为 22%；室内试验为 10%。因此，技术工作费列绝不能只根据“甲级/乙级/丙级”一套规则处理全部 sheet。fileciteturn0file2

### 表2-通用工程测量费用规则

表2的技术工作费调整系数主规则很简洁：

```text
如果表2行属于 2002 工程测量计费口径，则技术工作费调整系数 = 0.22
```

《使用手册》第 2.1 节明确给出“工程测量技术工作费收费比例为 22%”。因此，控制测量、地形图测绘、断面测量、地下管线测量等标准工程测量行，`技术工作费调整系数` 可以高置信自动填 `0.22`。fileciteturn0file2

但表2里又混入了 2009 年成本定额行，例如线路航测、像控点联测、地物地貌调绘、DLG/DEM/DOM、地图编制。样表在这些行上仍写入了 `0.22`，却没有让技术工作费小计实际参与总价公式。因此，Codex 应把表2技术工作费字段拆成两个属性：

| 字段 | 含义 |
|---|---|
| `tech_coeff_value` | 应填入单元格的显示值，例如 `0.22` |
| `tech_fee_effective` | 是否参与金额小计公式，表2中的 2009 成本定额行建议默认为 `false` |

### 表3-地质测绘规则

表3规则最明确：

```text
如果 AE=甲级，则 AF=1.2
如果 AE=乙级，则 AF=1.0
如果 AE=丙级，则 AF=0.8
如果 AE 缺失，则优先继承项目级岩土工程勘察等级；仍无法确定则待复核
```

《使用手册》表 3.1-1 明确给出岩土工程勘察技术工作费收费比例：甲级 120、乙级 100、丙级 80。当前样表全部写作 `甲级→1.2`，因此 `AF` 明确应被解释为技术工作费调整系数。fileciteturn0file2

### 表4-通用工程勘察费用规则

表4必须先分业务类型，再映射技术工作费调整系数。下表是建议的主分流规则。

| 表4业务类型 | 识别条件 | 技术工作费调整系数规则 |
|---|---|---|
| 岩土工程勘察 | `要素1=岩土工程勘察` 或 `O` 为甲/乙/丙 | 按甲/乙/丙映射 1.2/1.0/0.8 |
| 水文地质勘察 | `要素1=水文地质勘察` 或 `O` 为简单/中等/复杂 | 先区分“供水井凿井”与“其他水文地质勘察”，再按复杂度映射 |
| 工程水文/工程气象 | `要素1=工程水文` 或 `O=水文气象` | 固定 0.22 |
| 工程物探 | `要素1=工程物探` 或 `O=物探` | 固定 0.22 |
| 室内试验 | `要素1=室内试验` 或 `O=试验` | 固定 0.10 |

这意味着你在问题 B 中提出的“技术工作费调整系数是否主要由技术工作类别决定”只能回答为：**表3和表4中的岩土工程勘察部分，主要由技术工作类别（甲/乙/丙）决定；但表4整体并不遵从这一条，水文地质、工程水文气象、工程物探、室内试验都各有专属映射。**fileciteturn0file2
