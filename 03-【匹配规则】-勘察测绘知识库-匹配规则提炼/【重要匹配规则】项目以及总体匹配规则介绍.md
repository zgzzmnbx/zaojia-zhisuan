# 【重要匹配规则】项目以及总体匹配规则介绍

更新时间：2026-06-22  
适用版本：v4.9  
适用对象：普通用户、后续 Codex 线程、其他智能体、维护人员

本文说明“管勘智算”当前项目边界、三个关键数字的匹配来源、颜色、待复核口径，以及经验池预警、原始工作量抓取、Word 报告和大模型助手的职责边界。

## 1. 一句话说明

管勘智算是一个本地运行的长输管道勘察测量最高投标限价编制原型：用户上传控制价 Excel，系统按结构化规则回填基价、两个调整系数，生成 Excel 和 Word 报告，并用经验池做风险预警。

它的核心原则是：

```text
三个最终数字由结构化知识库和规则表裁决，大模型只做解释、提示、报告和问答辅助。
```

## 2. 当前项目能做什么

当前主流程：

1. 上传控制价计算 Excel。
2. 读取映射行，自动识别 `要素1-5`、`单位`、输出价格列、两个系数列和报告列。
3. 对候选 sheet 逐行匹配基价和两个调整系数。
4. 给成功、经验提示、待复核写颜色和逐行说明。
5. 输出回填后的 Excel。
6. 输出 Word 版控制价报告。
7. 用户可手动运行经验池预警分析，把 `预警参数`、`预警细节` 写回输出表。
8. 用户可用原始工作量抓取模块，把委托方工作量表中的数量、系数和备注抓到控制价计算表。
9. 用户可通过右侧“智算”助手做问答、风险报告、导出指令和行级复核。

当前不做：

- 不让大模型直接生成最终单价或系数。
- 不把经验池预警结果反向写入正式知识库。
- 不在首次转换时自动运行经验池预警。
- 不因为端口占用自动换端口。

## 3. 三个关键数字

当前重点输出三个数字：

```text
1. 基价 / 单价
2. 实物工作费调整系数
3. 技术工作费调整系数
```

三个数字共同使用三层判断思路：

| 层级 | 名称 | 是否自动写值 | 颜色 | 说明 |
|---|---|---:|---|---|
| 第一层 | 标准规则 / 固定知识库命中 | 是 | 绿色 | 可确定命中，直接填写 |
| 第二层 | 经验提示层 | 是 | 黄色 | 第一层未命中，但知识库经验数可用 |
| 第三层 | 大模型辅助层 | 否 | 不直接着色 | 只做解释、总结、报告、问答 |

当前颜色值：

```text
第一层绿色：C6EFCE
第二层黄色：FFF2CC
未命中红色：FFC7CE
```

## 4. 固定数据库 / 知识库

固定数据库文件：

```text

```

当前读取范围：

```text
A-H：基价 / 单价第一层匹配
J-M：实物工作费调整系数、技术工作费调整系数第二层经验提示
```

字段说明：

| 列 | 字段 | 用途 |
|---|---|---|
| A | 序号 | 数据库行号 |
| B | 要素1 | 基价匹配字段 |
| C | 要素2 | 基价匹配字段 |
| D | 要素3 | 基价匹配字段 |
| E | 要素4 | 基价匹配字段 |
| F | 要素5 | 基价匹配字段 |
| G | 单位 | 基价匹配字段 |
| H | 基价 | 单价 / 基价输出值 |
| I | 备注 | 人工说明 |
| J | 【经验数】实物工作费调整系数 | 第二层实物系数经验数 |
| K | 【经验数解释】-实物工作费调整系数 | 第二层实物系数经验说明 |
| L | 【经验数】技术工作费调整系数 | 第二层技术费系数经验数 |
| M | 【经验数解释】-技术工作费调整系数 | 第二层技术费系数经验说明 |

维护规则：

- `基价` 只从 H 列取值。
- J-M 不参与基价裁决，只作为两个调整系数的第二层经验提示。
- 经验数字段必须是可解析数字；`/`、空值、`待复核`、说明性文字不会作为经验数使用。
- 修改此文件后，应运行 `python -m pytest backend/tests -v`。

## 6. 基价 / 单价匹配

基价 / 单价只使用固定数据库：

```text
03-知识库-二维数据库制作/【数据库】【导入】.xlsx
```

匹配字段：

```text
要素1、要素2、要素3、要素4、要素5、单位
```

匹配模式详见：

```text
03-【匹配规则】-勘察测绘知识库-匹配规则提炼/【重要匹配规则】要素1-5和单位的匹配模式介绍.md
```

当前基价匹配只使用两档：

1. 字段完全匹配。
2. 非空要素顺序匹配。

命中规则：

- 唯一匹配且价格一致：填入基价，标绿。
- 无候选：填 `待复核`，标红，并说明从哪个要素开始没有匹配。
- 多候选且价格冲突：填 `待复核`，标红，并说明候选数量和冲突原因。

归一化口径包括：

- 忽略空格、换行、零宽字符。
- 统一全角半角符号。
- `1:500` 等比例尺可与 `比例-1:500` 互认。
- `Ⅰ类` 等级可与 `级别-Ⅰ类` 在顺序匹配中互认。
- 输入表里的 `成图比例` 按空值处理。
- 映射配置中选择 `空元素列` 时，该元素默认按空值参与匹配。

忽略行：

- `要素1` 为空。
- `要素1` 为 `小计`。
- `要素1` 为 `合计`。

这些行不参与匹配，也不计入报告统计。

列映射设置里的主匹配过滤：

- 价格匹配模块默认开启“不匹配指定列无值的行”。
- 指定列默认是 `数量`，程序按当前 sheet 表头定位实际列。
- 空白、数字 `0`、公式结果 `0` 都视为无值。
- 满足排除条件的行直接跳过，不做基价匹配，不做实物工作费调整系数匹配，不做技术工作费调整系数匹配，也不写逐行匹配报告、不计入匹配统计。

## 7. 实物工作费调整系数

当前口径：

```text
实物工作费调整系数第一层标准规则暂未启用。
```

当前匹配顺序：

1. 不走第一层 `physical_factor_rules` / `physical_factor_overrides`。
2. 如果当前行能唯一匹配到单价知识库，并且知识库 J 列有可解析经验数，则写入第二层经验数，标黄。
3. 如果第二层经验也没有命中，则写 `待复核`，标红。

第二层来源：

```text
03-知识库-二维数据库制作/【数据库】【导入】.xlsx
J列：【经验数】实物工作费调整系数
K列：【经验数解释】-实物工作费调整系数
```

待复核说明应明确写出：

```text
实物工作费调整系数第一层规则暂未启用，且第二层经验提示未命中。
```

## 9. 匹配报告和输出文件

如果开启“匹配报告输出”，系统会在输出 Excel 中增加：

```text
匹配状态
候选数量
匹配说明
匹配说明-实物工作费调整系数
匹配说明-技术工作费调整系数
```

报告列位置：

- 如果映射行中有 `匹配报告预留位置`，优先从该列之后写入。
- 否则从 R 列开始写。
- 如果目标列已被占用，自动向后顺延。

报告列颜色：

- `已匹配` 不额外标色，价格单元格继续标绿。
- 匹配不上 / 无候选的报告列标红。
- 多候选、冲突或其他需要人工复核的报告列标黄。
- 价格单元格原有口径保持不变：成功标绿，待复核标红。

输出命名：

```text
价格匹配 Excel：【输出】-控制价计算表-yyyymmdd-hhmm.xlsx
Word 报告：【输出】-控制价报告-yyyymmdd-hhmm.docx
工作量抓取控制价表：【临时】-控制价计算表（填好数量后）-yyyymmdd-hhmm.xlsx
工作量抓取标注原表：【临时】-原表-(工作量信息抓取后标注符合用)-yyyymmdd-hhmm.xlsx
```

## 10. 经验池预警

经验池预警不是价格裁决层。它只在转换完成后，由用户手动运行，用于发现输出结果和历史经验池之间的异常差异。

经验池文件：

```text
05-经验池-预警数据/【经验池】-管勘智算-【codex】.xlsx
```

经验池模板：

```text
05-经验池-预警数据/【经验池】【模板勿动】-管勘智算.xlsx
```

预警设置：

```text
05-经验池-预警数据/experience-warning-settings-【codex】.json
```

当前经验池找同类只使用：

```text
字段完全匹配 -> 非空要素顺序匹配 -> 未找到可比选同类记录
```

当前不再采纳“单位一致 + 要素前缀匹配”，避免把相似但不可靠的记录拉入平均值比选。

当前风险分级：

```text
偏离率 <= 低风险阈值：无预警
低风险阈值 < 偏离率 <= 高风险阈值：低风险
偏离率 > 高风险阈值：高风险
```

默认阈值：

```text
低风险：5%
高风险：20%
```

候选行过滤：

- 当前默认开启“只核查指定列有值的行”。
- 默认按 `数量` 字段定位实际列。
- `0`、公式结果为 `0`、空白都视为没有值，不进入预警候选。
- 如果找不到指定字段，应明确报错，不要静默跳过。

## 11. 原始工作量抓取

原始工作量抓取是价格匹配前的独立预处理模块。

输入：

- 委托方工作量表。
- 控制价计算表。

输出：

- 标注后的工作量表。
- 填好数量、系数、备注后的控制价计算表。

匹配口径：

```text
术语归并 -> 字段完全匹配 -> 非空要素顺序匹配
```

术语归并来源：

```text
03-【匹配规则】-勘察测绘知识库-匹配规则提炼/【术语归并】术语归并与匹配放宽规则表.xlsx
```

当前边界：

- 只作用于原始工作量抓取，不反向影响主价格匹配、经验池预警或第二层经验提示。
- 当前支持 `单位严格等价`、`要素严格等价`，以及“允许自动写值”的 `要素弱等价`。
- `字段名` 留空时，表示作用于 `要素1-5` 全部字段。
- 不完整规则行自动跳过。

只在一对一确定时写入。无匹配、一对多或风险行写入抓取日志，不强行写值。

重要边界：

- 工作量抓取不改变基价匹配逻辑。
- 工作量抓取不改变第二层经验提示层。
- 工作量抓取不改变经验池预警规则。

## 12. Word 报告和公式兜底

Word 报告按模板生成，写入项目名称、日期、费用汇总、匹配摘要、风险提示等内容。

报告模板当前路径：

```text
03-知识库-二维数据库制作/01-报告模板-招标控制价报告模板/【模板勿动】控制价报告模板-yyyy-mm-dd.docx
```

费用汇总读取：

- 优先使用 Excel 公式缓存值。
- 如果公式缓存为空，程序会对常见公式做兜底计算。
- 当前支持跨 sheet 引用、区域求和、四则运算、`ROUND/MIN/MAX` 等本项目常见汇总公式。
- 前端预览和 Word 报告数值最多显示 2 位小数。

## 13. 大模型和“智算”助手边界

当前大模型入口集中在右侧可折叠“智算”助手。

它可以做：

- 本地知识库依据问答。
- 风险报告。
- 行级复核解释。
- 转换过程提示。
- 经验池预警摘要。
- Excel / Word 下载快捷指令。

它不可以做：

- 直接决定基价。
- 直接决定实物工作费调整系数。
- 直接决定技术工作费调整系数。
- 覆盖结构化规则得出的确定结果。

当前知识库依据问答采用旁路设计：

```text
用户在“问问智算”提问
-> 后端本地检索标准资料、项目规则文档、结构化规则卡片和当前行上下文
-> DeepSeek 只基于已检索证据解释
-> 回答展示在原“问问智算”窗口
```

这套问答能力只解释依据，不进入三个最终数字的裁决链路。新增接口分工如下：

```text
POST /api/knowledge/search：只做本地检索，不调用 DeepSeek
POST /api/knowledge/ask：先检索本地证据，再调用 DeepSeek 解释
```

如果本地知识库没有检索到明确依据，必须回答：

```text
当前知识库未找到明确依据，需要人工复核。
```

运行缓存：

```text
Codex-Temp/runtime/knowledge-qa-index-【codex】.json
```

该缓存只用于问答检索，不覆盖原始资料，不参与基价、实物工作费调整系数、技术工作费调整系数的写值判断。

默认模型：

```text
官方 DeepSeek deepseek-v4-flash
```

API Key 只从后端环境变量或 `.env.local` 读取，不写入前端。
