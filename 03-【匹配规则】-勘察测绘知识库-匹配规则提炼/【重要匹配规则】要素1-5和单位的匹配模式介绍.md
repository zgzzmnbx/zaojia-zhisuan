# 【重要匹配规则】要素1-5和单位的匹配模式介绍

更新时间：2026-06-22  
适用版本：v4.9  
适用对象：普通用户、后续 Codex 线程、其他智能体、维护人员

## 1. 这份文档是干什么的

这份文档单独解释本项目里最基础、也最容易被复用的一条规则：

```text
要素1、要素2、要素3、要素4、要素5、单位
```

它是项目里的“同类记录识别底座”。以后无论是：

- 基价 / 单价匹配
- 调整系数第二层经验提示
- 经验池预警“找同类”
- 原始工作量抓取
- 其他需要按项目特征找同类记录的模块

只要说“按项目既有匹配模式处理”，默认先参考这份文档。

## 2. 为什么要单独写清楚

Excel 来源不同，字段写法也不同。输入表、知识库、经验池、工作量表里，同一条业务记录可能出现：

- 空格、换行、全角半角不同。
- `1:500` 和 `比例-1:500` 不同。
- `Ⅰ类` 和 `级别-Ⅰ类` 不同。
- 中间某个要素列为空。
- 用户在映射设置里把某个要素设成 `空元素列`。

如果每个模块都临时发明一套“差不多匹配”，正式填价、经验池预警和工作量抓取就会互相打架。所以本文件把可复用模式先定清楚，再说明每个模块当前实际采用哪几档。

## 3. 基础字段

默认字段：

```text
要素1、要素2、要素3、要素4、要素5、单位
```

基本约束：

- `单位` 是硬边界。单位不一致，默认不是同类。
- `要素1` 是起点字段。没有 `要素1` 或没有 `单位`，通常不进入有效匹配。
- `要素2-5` 可以为空，但空值怎么处理由具体模式决定。
- 映射配置中的 `空元素列` 表示这个要素列在当前表中不存在，按空值参与匹配。

忽略行：

- `要素1` 为空。
- `要素1` 为 `小计`。
- `要素1` 为 `合计`。

这些行不参与正式匹配，不计入匹配报告。

## 4. 归一化规则

进入匹配前，文本会先做归一化。当前代码口径见：

```text
backend/app/normalization.py
backend/app/knowledge_base.py
```

当前归一化要点：

1. 统一全角半角。
2. 中文括号转英文括号。
3. 中文冒号转英文冒号。
4. 去掉空格、换行、零宽字符等空白差异。
5. 统一转小写。
6. `none / null / nan / 空 / 无` 这类空标记按空值处理。
7. 形如 `1:500`、`1:1000` 的比例写法会识别成 `比例-1:500`、`比例-1:1000`。
8. 在顺序匹配中，`级别-`、`比例-` 这类前缀会去掉后再比较。
9. 输入表里文字值 `成图比例` 视为占位说明，不作为实际比例值参与正式填价匹配。

这意味着很多“看起来不一样、实际意思一样”的 Excel 文本，进入匹配后会归到同一个口径。

另外，原始工作量抓取当前在进入模式 A+B 之前，还会额外读取：

```text
03-【匹配规则】-勘察测绘知识库-匹配规则提炼/【术语归并】术语归并与匹配放宽规则表.xlsx
```

只对工作量抓取先做一层术语归并。当前支持：

- `单位严格等价`
- `要素严格等价`
- “允许自动写值”的 `要素弱等价`

其中 `字段名` 留空时，表示该条规则作用于 `要素1-5` 全部字段；这层规则当前不反向影响主价格匹配或经验池预警。

## 5. 四档匹配模式

下面四档是项目统一使用的“模式词典”。注意：不是每个模块都会启用全部四档。

### 模式 A：字段完全匹配

定义：

```text
要素1、要素2、要素3、要素4、要素5、单位
归一化后逐字段完全一致
```

特点：

- 最严格。
- 可信度最高。
- 适合作为第一优先级。

### 模式 B：非空要素顺序匹配

定义：

```text
单位一致；
忽略空要素；
把要素1-5中的非空内容按原顺序组成序列；
输入行与候选行的非空要素序列完全一致。
```

示例：

```text
知识库：岩土工程勘察、地质测绘、空、比例-1:500、复杂，单位 km2
输入表：岩土工程勘察、地质测绘、比例-1:500、复杂、空，单位 km2
```

二者忽略空值后的非空序列都是：

```text
岩土工程勘察 -> 地质测绘 -> 1:500 -> 复杂
```

因此可以视为同类。

重要边界：

- 非空要素的相对顺序必须一致。
- 不能把两个非空要素互换后仍当作匹配。
- 这一档解决的是“空列位置不同”，不是任意乱序匹配。

### 模式 C：单位一致 + 要素前缀匹配

定义：

```text
单位一致；
从要素1开始连续比较；
优先取连续命中前缀最长的候选集合。
```

例如：

```text
要素1、要素2、要素3 都对上；
要素4 开始不同。
```

则可记为：

```text
单位一致 + 要素1至要素3前缀匹配
```

注意：

- 必须从 `要素1` 开始连续命中。
- 不能跳着命中，例如 `要素1` 和 `要素4` 对上，但中间断了，不算前缀匹配。
- 这一档更适合覆盖分析、辅助比选、覆盖不足提示。
- 当前正式填价、经验池预警、原始工作量抓取都不启用模式 C。

### 模式 D：未找到可比选同类记录

定义：

```text
当前模块允许的模式都未命中
```

这时应明确告诉用户：

```text
未找到可比选同类记录
```

不要把它误表述成：

```text
0 条预警
```

因为“没有预警”和“没有找到同类”不是一回事。

## 6. 各模块当前实际采用模式

| 模块 | 当前采用模式 | 是否启用模式 C | 结果用途 |
|---|---|---:|---|
| 基价 / 单价匹配 | A -> B | 否 | 第一层确定性填价 |
| 调整系数第二层经验提示 | 跟随基价知识库唯一匹配结果 | 否 | 第一层未命中时填经验数 |
| 经验池预警找同类 | A -> B -> D | 否 | 只用于风险比选，不裁决价格 |
| 原始工作量抓取 | 术语归并 -> A -> B | 否 | 一对一确定时抓数量、系数、备注 |
| 覆盖分析 / 辅助比选 | 可讨论 A -> B -> C -> D | 视任务而定 | 只做辅助说明 |

## 7. 当前单价匹配

当前“单价匹配”在程序里实际是：

```text
基价 / 单价匹配
```

它使用固定知识库：

```text
03-知识库-二维数据库制作/【数据库】【导入】.xlsx
```

实际模式：

```text
字段完全匹配 -> 非空要素顺序匹配
```

命中后：

- 直接输出基价。
- 单元格标绿。
- 报告写明匹配到知识库哪一行。

未命中或冲突时：

- 输出 `待复核`。
- 单元格标红。
- 报告写明无候选、多候选或从哪个要素开始没有匹配。

## 8. 当前经验池预警

经验池预警当前只用于转换后的风险检查，不参与价格裁决。

找同类模式：

```text
字段完全匹配 -> 非空要素顺序匹配 -> 未找到可比选同类记录
```

当前不启用模式 C。原因是经验池预警要拿同类记录计算平均值，如果同类范围放得过宽，平均值和偏离率会失真。

经验池预警还有一个前置候选过滤：

- 默认只核查指定列有值的行。
- 默认字段是 `数量`。
- `0`、公式结果为 `0`、空白都视为没有值，不进入预警候选。
- 如果找不到指定字段，应明确报错。

风险分级只看“当前值相对经验池平均值的偏离率”。默认阈值：

```text
低风险：5%
高风险：20%
```

## 9. 当前原始工作量抓取

原始工作量抓取模块当前口径：

```text
术语归并 -> 字段完全匹配 -> 非空要素顺序匹配
```

术语归并来源：

```text
03-【匹配规则】-勘察测绘知识库-匹配规则提炼/【术语归并】术语归并与匹配放宽规则表.xlsx
```

写入条件：

- 只在一对一确定时写入。
- 无匹配、一对多或风险行写入抓取日志。
- 当前不启用模式 C。

当前边界：

- 只在工作量抓取里启用，不影响主价格匹配。
- `字段名` 留空时，表示作用于 `要素1-5` 全部字段。
- 不完整规则行自动跳过。

它的作用是把工作量表里的数量、两个系数和备注抓到控制价计算表；它不改变正式填价规则，不改变经验池预警规则。

## 10. 后续线程默认怎么用

后续线程如果要改涉及“同类记录识别”的逻辑，默认按下面顺序处理：

1. 先判断这是正式填价、经验池预警、工作量抓取，还是辅助分析。
2. 如果是正式基价 / 单价匹配，默认守住 `A -> B`，不要放宽到 `C`。
3. 如果是经验池预警，当前默认守住 `A -> B -> D`，不要恢复 `C`，除非用户明确要求提高覆盖率并接受均值可能变松。
4. 如果是工作量抓取，默认守住 `A -> B`，因为抓取会直接写数量、系数和备注。
5. 如果是覆盖分析、辅助比选、覆盖不足提示，可以讨论是否使用 `A -> B -> C -> D`。
6. 如果要新增模块，优先复用本文件的模式词典，并在本文件补充该模块实际采用哪几档。

## 11. 一句话结论

“要素1-5 + 单位”不是简单拼字符串查表，而是本项目多个模块共享的同类识别口径。当前正式写值模块只启用更稳的 A/B 两档；模式 C 只作为辅助分析的备选词汇，不默认参与裁决。
