# Claude 建议包说明

**生成日期**：2026-07-03  
**适用项目**：造价智算 v5.3.7  
**建议包版本**：v1.0

---

## 一、本建议包的目的

本建议包是 Claude 作为外部产品规划顾问、技术路线顾问和 UI 审查顾问，在充分理解"造价智算"当前项目现状后，发散出的未来可开发功能建议、版本路线建议、技术实现建议和 UI 优化建议。

**重要边界声明**：

1. **本建议包不是正式 PRD**，不直接代表项目需求，不替代当前 `00-PRD/` 中的正式产品需求文档。
2. **本建议包不修改任何现有项目文件**，所有内容仅作为外部建议，供 Codex 后续自主消化吸收。
3. **本建议包遵守项目核心原则**：结构化规则裁决，大模型解释辅助，人工复核兜底；不建议推翻当前架构，不建议让大模型直接裁决价格、单价、系数或审减金额。

---

## 二、建议包不修改当前 PRD 的原因

造价智算当前已有完整的 PRD 体系和版本计划（`00-PRD/02-当前版本计划.md`），已明确当前版本 v5.3.7 的完成项、待确认项和暂不做项。外部建议包如果直接污染当前 PRD，会造成以下问题：

- 把"未经验证的设想"混入"已经在做的需求"，干扰开发优先级。
- 把"远期储备功能"误当作"近期必做任务"，打乱版本节奏。
- 把"外部顾问发散思路"误当作"用户明确要求"，造成需求膨胀。

因此，本建议包采用独立文件夹 `【Claude】/` 的方式输出，所有建议都标明"供 Codex 后续消化吸收"，由 Codex 结合用户真实反馈、业务优先级和技术可行性，自主判断哪些建议值得纳入正式 PRD，哪些建议只作为远期参考。

## 四、Codex 后续如何阅读这些文件

### 第一次接手本建议包

1. 先读 `00-README-本次Claude建议包说明.md`（本文件），理解建议包的边界。
2. 再读 `01-项目理解与现状判断.md`，确认 Claude 是否准确理解了项目核心。
3. 然后读 `09-Codex后续消化提示词.md`，按提示词逐步消化其他文件。

### 如果用户明确要求开发某个功能

- 如果用户要求"开发辅助填价"，读 `04-辅助填价功能PRD建议.md`，结合用户具体需求，提炼正式 PRD 后再开发。
- 如果用户要求"开发结算审核助手"，读 `05-结算审核助手PRD建议.md`，结合用户具体需求，提炼正式 PRD 后再开发。
- 如果用户要求"准备统信迁移"，读 `06-统信UOS迁移预备建议.md`，按阶段性路线推进。

### 如果用户要求"看看 Claude 有什么建议"

- 先读 `02-未来功能发散清单.md`，浏览功能机会地图，挑选 1-3 个高价值、低复杂度的功能与用户讨论。
- 再读 `03-近期功能优先级与版本路线.md`，结合当前版本计划，判断哪些建议适合纳入下一版本。

### 如果用户要求"优化 UI"

- 读 `08-UI优化PRD建议.md`，结合当前 `00-PRD/03-整体UI设计PRD.md` 和 `00-PRD/03-UI设计规范.md`，提炼 UI 优化清单。

## 五、哪些建议适合近期吸收，哪些建议只适合远期储备

### 近期适合吸收（用户明确需求或当前版本计划已提及）

- **辅助填价功能**：用户已提出设想，且与当前填价能力强相关，复用度高。
- **结算审核助手**：用户已提出设想，且与当前经验池预警、风险报告能力强相关。
- **统信 UOS 迁移预备**：当前 `docs/统信UOS兼容性开发说明.md` 已明确为未来迁移预案，Claude 建议可作为技术预研补充。
- **UI 优化建议中的"信息架构调整""辅助填价窗口""风险卡片 UI"**：与当前大尾巴主题优化方向一致。

### 远期储备（业务价值高但需要更多验证或技术准备）

- **造价指标库**：需要跨项目数据积累和治理能力。
- **多项目对比分析**：需要项目库和对比分析引擎。
- **规则维护后台**：需要权限体系和规则治理流程。
- **数字化项目造价扩展**：需要新的业务场景验证。
- **估概算审核扩展**：需要新的业务场景验证。

### 明确暂缓（当前阶段不适合做）

- **Tauri 统信安装包 / deb 包 / AppImage**：当前 `00-PRD/02-当前版本计划.md` 已明确暂缓。
- **让大模型直接裁决价格、系数或审减金额**：违反项目核心原则。
- **推翻当前匹配引擎或规则体系**：违反项目核心原则。

1. **本建议包所有内容均为外部建议，不代表正式需求**。Codex 消化吸收时，必须结合用户真实需求、当前版本计划和技术可行性，自主判断哪些建议值得纳入正式 PRD。
2. **本建议包不修改任何现有项目文件**。如需将建议转化为正式 PRD，应由 Codex 在 `00-PRD/01-模块PRD/` 中新建或更新对应模块 PRD，而不是直接引用本建议包文件。
3. **本建议包遵守项目核心原则**：结构化规则裁决，大模型解释辅助，人工复核兜底。所有建议均在此边界内发散，不建议让大模型直接裁决价格、单价、系数或审减金额。
4. **本建议包中的"假设与待确认项"**：因资料不足，部分建议基于合理假设。Codex 吸收时应优先与用户确认真实需求，而不是按假设直接开发。
