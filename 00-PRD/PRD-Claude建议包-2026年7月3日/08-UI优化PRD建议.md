# UI 优化 PRD 建议

**生成日期**：2026-07-03  
**基于版本**：造价智算 v5.3.7（大尾巴主题 / Navattic 极简 SaaS 基线）  
**审查依据**：已直接查看 `04-输出版本存档/当前软件截图-2026年7月3日/` 全部 4 张截图（填价工作台、结果预览、经验池预警、工作量抓取），并结合 `frontend/src/App.tsx`、`frontend/src/styles.css` 结构、`00-PRD/03-整体UI设计PRD.md` 与 UI 设计规范  
**性质**：外部建议，供 Codex 后续消化吸收，不直接污染当前 PRD；**本文不改任何代码**

## 一、UI 总体诊断

### 总体结论

**当前 UI 基线是健康的，不需要重做，只需要"结构性补强"。**

v5.3.x 的 Navattic 极简收口成效明显：白底、1px 分割线、扁平面板、克制选中态、三栏工作台在截图中均已落地，整体观感已达到"企业级 SaaS 工作台"水准，明显优于常见参赛作品的"大屏风"。四个模块页（填价工作台/结果预览/经验池预警/工作量抓取）层级清楚，左侧环形仪表盘是全场亮点。

当前的短板不在视觉风格，而在三处**结构性缺口**：

1. **流程感缺口**：模块之间是"页面切换"关系，不是"任务推进"关系。用户/评委看不到"我在主流程的第几步、下一步该干什么"。
2. **处理闭环缺口**：系统很会"发现问题"（红色待复核、风险预警），但界面没有给"处理问题"留位置——没有本项填价窗口、没有复核清单、风险明细是大段文字块而非可操作卡片。
3. **右侧 AI 信息形态缺口**：AI 侧栏承载了大量"过程叙述文字"，纯文本流密度过高，关键信息（风险摘要、下一步建议）淹没在长段落里。

### 诊断计分卡

| 审查项 | 现状评价 | 结论 |
|---|---|---|
| 首页信息密度是否过高 | 填价工作台适中偏低（上传+概览+工作状态三块，留白充足）；**经验池预警页和右侧 AI 流偏高**（大段密排文字） | 局部过高，非全局问题 |
| 模块层级是否清楚 | 左侧"图标栏+模块列表+仪表盘"清楚；模块内二级层级（如预警页的模块/卡片/字段）偏弱 | 一级清楚，二级待强化 |
| 各模块主次关系 | 中间工作区为主、AI 为辅的关系正确；填价工作台内"转换后概览"的 100% 大数字最抢眼，但"下一步去哪"无引导 | 主次对，衔接弱 |
| 右侧 AI 是否过重或过弱 | 宽度和位置合理（不压主区），但**内容形态过重**：连续多条长文本气泡，信息利用率低 | 形态过重、价值感偏弱 |
| 是否需要主流程进度条 | 旧流程条已删除是对的（避免占空间的装饰性步骤条），但**轻量任务进度指示**确有缺口 | 需要，轻量形态 |
| 是否需要当前任务上下文条 | 顶部已有文件名/行数摘要（结果预览页表头区），但跨模块不一致 | 需要，统一化 |
| 是否需要更明显的"下一步建议" | 目前靠 AI 聊天文字提示，易被忽略 | 需要，工作状态区放动作化入口 |
| 是否需要本项填价窗口 | 当前待复核行只能看说明或改单元格，无候选组织交互 | **需要**（配合辅助填价功能） |
| 是否需要风险卡片抽屉 | 预警明细目前是页面内大块红底文字卡，跨模块查看不便 | **需要**（全局抽屉复用） |
| 是否需要审核工作台模式 | 结算审核功能立项后需要独立模块页，非现有页面改造 | 需要（随 V2 功能） |
| 表格预览是否需要重构 | **不需要重构**，需要工具条升级（现有 UI PRD 已列 [待开发]）+ 列头/长文本优化 | 优化非重构 |
| 手动编辑/重算公式/导出位置 | 重算公式与下载在表格工具条区，位置成立；但"人工修改过 N 处、需要重算"缺少状态提醒 | 位置合理，缺状态反馈 |
| 是否适合演示/评委/业务人员 | 业务人员可用；评委演示缺"一键样例+讲解引导"；首屏欢迎态对评委不够"自解释" | 需演示模式补强 |
| 普通笔记本分辨率是否拥挤 | 截图为宽屏；按三栏固定宽度推算，1366×768 下中间区约 700-800px，结果预览宽表会明显局促 | 有风险，需折叠策略 |

## 三、信息架构建议

### 3.1 保持不变的骨架

左侧"极窄图标栏 + 当前模块列表 + 底部状态仪表盘"、中间单模块工作区、右侧可折叠智算侧栏——**这套三栏骨架不动**。

### 3.2 模块导航建议增补（随 V2/V3 功能）

```text
填价工作台
结果预览
经验池预警
工作量抓取
Word 报告
知识库问答
──────────（新增分组线：审核能力，V2 后）
结算审核        ← 新模块（审核工作台模式）
```

待复核清单、辅助填价不设一级模块——它们是结果预览的下钻交互，不是独立页面。

### 3.3 新增两个全局层（本文核心信息架构主张）

| 新增层 | 形态 | 承载内容 |
|---|---|---|
| **任务上下文条** | 中间工作区顶部一条 32-36px 高的细条，跨模块常驻 | 当前文件名 · 输入/匹配行数 · 当前阶段（已转换/已预警/已出报告）· 下一步建议按钮 |
| **风险卡片抽屉** | 从右侧滑出的全局抽屉（Dialog 级浮层，允许阴影） | 结构化风险卡片列表（预警+待复核+未来审核风险），筛选、跳转、处理入口 |

这两层解决 P2/P3/P5 三个问题，且都不推翻现有布局。

## 四、首页（填价工作台）布局建议

现状（v5.3.5 布局）：上层"上传标准 Excel（左）+ 转换后概览（右）"，下层"工作状态"横跨。**该布局保持**，建议做 4 个增量：

1. **顶部加任务上下文条**（见 3.3）。未上传时显示"未开始 · 请上传标准 Excel"；转换后显示"【文件名】· 101 行 · 已转换 · 预警未运行"。
2. **工作状态区加"下一步建议"动作位**：在匹配质量分布右侧或下方放 1-2 个语义按钮，随状态切换：
   - 已转换且有待复核 → `处理 3 条待复核 →`（跳结果预览并过滤待复核行）
   - 已转换未预警 → `运行经验池预警 →`
   - 已预警 → `查看风险卡片(5) →`（开抽屉）/ `生成 Word 报告 →`
3. **精简冗余数字**（P6）：转换概览保留 100% 大数字与四个统计；工作状态里的小环形图与左栏仪表盘二选一即可——建议保留左栏（全局常驻），工作状态内改为纯色条+数字，降低同屏三个圆环的重复感。（低优先级，可不做）
4. **欢迎空态强化评委自解释**：未上传时中间区已有欢迎标题；建议空态区加一行产品定位副标（"规则裁决 · 模型解释 · 人工兜底"）+ `加载演示样例` 按钮（演示模式入口，见第十一节）。

## 六、右侧 AI 助手优化建议（解决 P1）

**定性**：位置、宽度、可折叠、顶部身份区、底部输入框都正确，问题只在**消息形态**。四条建议：

1. **过程消息卡片化+可折叠**：转换/预警执行时的多段过程叙述（"正在读取…已识别…已写入…"）合并为**一条可展开的过程卡**，默认只显示最终态摘要（"转换完成 · 101/101 行 · 耗时 8s ▸展开过程"）。聊天流长度立即下降 60% 以上。
2. **风险摘要消息结构化**：现在的"高风险第N条：…大段文字"改为**迷你风险卡**（等级色点 + 一行摘要 + 跳到表格/查看卡片两个动作），与风险卡片抽屉数据同源。AI 的文字解释放在卡片下方一行，而不是包裹全部信息。
3. **区分"叙述"与"回答"两种气泡权重**：系统过程通知用更淡的样式（浅灰小字，无头像或小图标），只有真正的 AI 回答用标准气泡+Z 头像。当前所有内容同权重是密度感的主因。
4. **快捷指令保持现状**（上弹式已完成），但建议随当前模块动态排序：结果预览页优先"输出风险报告/行级复核"，预警页优先"预警摘要解释"。

**边界**：不加宽 AI 栏，不给 AI 增加任何写值/确认按钮（候选确认只在本项填价窗口完成，AI 流中最多出现"打开填价窗口"跳转按钮——与 `04-辅助填价功能PRD建议.md` 第十一节一致）。

## 七、三个新界面的 UI PRD

### 7.1 本项填价窗口（配合辅助填价功能）

- **形态**：模态 Dialog，居中，宽 720-840px，高 ≤ 80vh；浮层允许阴影（符合规范例外）；内部禁卡片套娃——上下文区/候选区/确认区用分割线分隔。
- **结构**：三段式（上下文区只读 → 候选列表可滚动 → 底部确认区固定），详细字段见 `04-辅助填价功能PRD建议.md` 第五节窗口图。
- **候选行视觉**：单选 radio + 候选值（等宽数字、加粗）+ 来源标签（知识库-相似/经验池/历史确认，用现有中性 chip 样式）+ 可信度标识（★推荐 用主题绿 `rgb(117,188,146)`；谨慎 用琥珀，不用红——红保留给"未命中/高风险"语义）+ 相似度小字 + 风险提示一行（超出折叠）。
- **交互**：未选中时"确认写入"禁用；自定义输入选中时备注必填校验；确认成功后窗口关闭，表格目标单元格闪烁一次（复用既有定位闪烁），并显示人工确认标识。
- **状态色新增**：需要一个"人工确认"单元格底色。建议复用现有人工修改浅蓝，批注区分来源；若用户要求区分，再引入第五色（需同步 UI 规范的状态色章节）。

### 7.2 结算审核助手（审核工作台模式）

- **入口**：左侧导航新增"结算审核"模块，不复用经验池预警页。
- **页面结构**（中间工作区）：
  ```text
  [任务上下文条：结算文件 · 基准文件 · 比对状态]
  [上传/设置区：结算表 + 基准表 两个上传位 + 阈值设置入口]（比对完成后自动收窄为一行摘要）
  [风险总览条：高风险 N · 低风险 N · 提示 N · 无法比对 N（可点击筛选）]
  [风险卡片列表：等级/类型筛选 chip + 卡片流（每行一卡，非网格）]
  [底部动作条：导出风险清单 Excel ·（二阶段）生成意见草稿]
  ```
- **风格约束**：卡片列表区扁平无阴影（页面内嵌形态）；等级只用色点+文字标签表达，禁止整卡红底铺色（吸取经验池预警页 P2 教训）。
- **与预警页关系**：视觉组件与经验池预警的新风险卡片完全同源，只是数据来源不同（审核比对 vs 经验偏离）。

### 7.3 风险卡片（全局组件 + 抽屉）

- **卡片本体**（两种密度）：
  - 列表态（页面内嵌/抽屉内）：`[等级色点] 风险类型标题  位置摘要（表4·第18行·钻探）  关键数字对比（当前 156.00 / 基准 128.00 / +21.9%）  [跳到表格] [详情]`，高度约 64-80px，白底 1px 边框，左侧 3px 等级色条。
  - 迷你态（AI 聊天流内）：一行摘要 + 两个文字动作。
- **抽屉**：右侧滑出宽 400-480px（覆盖在 AI 栏之上或临时替换 AI 栏内容——建议前者，Dialog 级浮层），顶部筛选（等级/类型/模块来源），中部卡片流，底部"导出清单"。任何模块页可通过任务上下文条的"风险 N"入口打开。
- **等级色**：高风险=现有红、低风险=琥珀黄、提示/无法比对=中性灰蓝；色值沿用现有状态色体系，不新造颜色。
- **卡片动作**：跳到表格（复用既有定位）、详情（展开依据与建议）、（V2 后）标记处理/打开辅助填价。

## 九、表格预览优化建议（P4，优化不重构）

1. **列头降噪**：列头去掉与数据重复的后缀语义（如"-测试"类调试残留应清理）；两行长列头改为一行+悬浮完整名；列头字号可比数据小一号、用次级文字色 `#64748B`。
2. **长文本列策略**：预警细节/匹配说明列默认收窄至 2 行截断（已有），建议增加"点击单元格弹出只读详情浮层"（比悬浮 tooltip 更适合大段复核文字，且触屏/演示投屏可用）。
3. **状态列 chip 统一**：预警参数列的 无预警/高风险/待复核 chip 已成立，建议匹配状态、预警参数两列 chip 样式与风险卡片等级色点严格同源。
4. **工具条升级**（现有 [待开发] 项）：左侧 `Sheet chips`，右侧 `搜索 · 状态筛选(待复核/高风险/人工修改) · 列设置 · 重算公式 · 导出`。**新增"状态筛选"是本文重点加码**——一键只看待复核行/风险行，是复核动线的关键（也是辅助填价的批量入口）。
5. **人工修改状态反馈**：有未重算的人工修改时，"重算公式"按钮加小圆点标记+文案"重算公式（3 处修改待重算）"，解决"改了但忘了重算"的隐患。
6. **行级操作区预留**：固定 AI 列扩展为"行操作"列（AI 解释 + 辅助填价 + 标记复核，图标按钮，随功能上线逐个启用），避免未来加按钮时再改表格结构。

## 十一、演示模式 / 评委模式 UI 建议

1. **入口**：欢迎空态放 `加载演示样例` 按钮；或右上角"页面设置"内放"演示模式"开关。不做常驻大按钮，避免污染日常工作态。
2. **一键样例**：点击后自动加载预置 100 行样例并完成转换（调用既有流程），让评委 10 秒内看到"满屏结果"。
3. **步骤引导**：演示模式下任务上下文条变为 5 步 chip：`① 上传 → ② 转换 → ③ 预警 → ④ 问答 → ⑤ 输出`，当前步高亮，点击 chip 跳转对应模块并附一句话讲解气泡。非演示模式不显示 5 步条（保持工作态克制）。
4. **讲解点高亮**：演示模式下，绿/黄/红三色行、经验池偏离卡、依据追溯各有一次性的脉冲高亮+一句话说明，突出"规则裁决/经验预警/可追溯"三个评分点。
5. **脱敏提示**：演示模式页脚固定小字"演示数据，非真实价格"（配合数据脱敏交付版）。
6. **克制约束**：演示模式不引入新视觉语言，不加动效轰炸，只是"预置数据+引导层"，可随时退出。

## 十二、视觉风格建议

**总原则：不换风格，守住现有基线。**

1. 延续 Navattic 极简 SaaS：白底、1px 边框、无常驻阴影、克制选中态、深蓝主按钮、Inter+等宽数字——**全部保持**。
2. **明确禁止**（与用户要求一致）：不做国企数字大屏、不用霓虹/泛光/强蓝光/粒子/扫光动效、不做深色科技风主界面（深色风格只留给汇报 PPT）。
3. 状态色纪律强化：绿=第一层命中、黄/琥珀=第二层经验与低风险、红=未命中与高风险、蓝=知识库调用与人工修改；新功能（辅助填价推荐★、审核卡片）从这套色里取，不新增色相；大面积铺底色只允许出现在 chip 和色点，**不允许整卡/整行铺红**（修正经验池预警页现状）。
4. 密度基调：表格区高密度（数据工作台），说明区中密度，AI 区低密度（第六节降噪后自然达成）。

## 十三、MVP 优先级（UI 工作排序）

| 序 | UI 工作 | 解决问题 | 依赖 |

## 十四、给 Codex 的 UI 实现关键路径

> 建议路径，Codex 结合 `App.tsx` 实际组件结构自行判断。

1. 新组件一律走 `daweiba-*` 命名空间，放独立组件文件，不再向 `App.tsx` 单文件堆叠（7300+ 行已近维护上限；新界面组件建议逐个拆文件，但**不要求**对既有代码做拆分重构）。
2. 顺序：`RiskCard` 列表态组件（先在经验池预警页替换现有红底明细）→ 任务上下文条（读全局转换/预警状态）→ 表格工具条状态筛选 → 抽屉（复用 RiskCard）→ 演示引导层 → FillAssistDialog → 结算审核页。
3. 每步只跑 `npm run frontend:build` 验证（UI 小修改轻量流程，遵守 AGENTS.md 口径）；涉及新后端接口时才动后端并跑 pytest。
4. 状态色、间距、字号从现有 CSS 变量取值，不新增硬编码色值；改动前读 `00-PRD/03-UI设计规范.md` 对应章节。
5. 窄屏策略（P7）：建议在 `≤1440px` 断点让右侧 AI 默认折叠、`≤1280px` 让左侧一级栏收为图标态（用现有折叠能力自动触发，不新造布局）；演示前在 1366×768 实测一次。

## 十五、UI 验收标准

1. **零逻辑破坏**：所有 UI 改动不改变匹配、预警、报告、下载和写回逻辑；`npm run frontend:build` 通过。
2. **AI 流降噪**：完成一次转换+预警后，右侧聊天流首屏消息条数 ≤ 4 条（过程卡折叠态计 1 条）；展开可见完整过程。
3. **风险卡片**：经验池预警页明细以卡片列表呈现，无整卡红底；每卡有等级点、位置、数字对比、跳转动作；跳转定位行为与既有验收一致（切 sheet/滚动/闪烁）。
4. **任务上下文条**：上传前、转换后、预警后三种状态文案正确；下一步按钮点击可达对应模块/动作。
5. **表格**：状态筛选可一键只看待复核/高风险行；筛选后行数统计正确；有未重算修改时重算按钮出现标记。
6. **演示模式**：从空态点"加载演示样例"到看到满屏转换结果 ≤ 15 秒；5 步引导可逐步点击走完；退出演示模式后界面回到常规态。
7. **窄屏**：1366×768 下三栏不溢出、不遮挡，结果预览表格可横向滚动且 AI 列仍固定可点。
8. **风格纪律**：新增界面无卡片套娃、无常驻阴影、无新增高饱和铺色；状态色与既有体系一致。
