**生成日期**：2026-07-03  
**基于版本**：造价智算 v5.3.7  
**性质**：外部建议，供 Codex 后续消化吸收，不直接污染当前 PRD  
**如需正式立项**：由 Codex 在 `00-PRD/01-模块PRD/` 中新建正式模块 PRD

---

## 一、功能定位

### 一句话定位

辅助填价是在**待复核行**上，把知识库、经验池、历史项目、同类记录、规则说明中的**候选价格或候选系数组织成结构化建议**，由**用户在"本项填价窗口"中人工确认后写入**输出 Excel 副本的辅助能力。

### 它是什么

- ✅ 候选组织器：把散落在多个数据源的候选值收拢到一个窗口
- ✅ 证据展示器：每个候选带来源、匹配理由、相似度、风险提示
- ✅ 人工确认写入器：用户点确认才写值，全程留痕

### 它不是什么

- ❌ 不是"AI 自动填价"：大模型不产生候选数字，不裁决写入哪个候选
- ❌ 不是第一层规则的替代：第一层已命中的绿色行不需要辅助填价
- ❌ 不是放宽主匹配：主匹配仍守 A/B 两档，辅助填价的"相似候选"只出现在候选列表里，永不自动写值

### 与三层判断体系的关系

| 层级 | 现状 | 辅助填价介入点 |
|---|---|---|
| 第一层标准规则（绿） | 自动写值 | **不介入**（已确定命中） |
| 第二层经验提示（黄） | 自动写值 | 可选介入：用户想换值时可打开窗口看其他候选 |
| 未命中/待复核（红） | 填"待复核" | **主要介入点**：为待复核行组织候选，人工确认填值 |

## 二、用户场景

### 场景 1：待复核行快速处理（核心场景）

> 造价员小张转换完 100 行，其中 8 行标红"待复核"。以前他要打开知识库 Excel、翻历史项目、查规则说明，逐行人肉找参考。现在他点击待复核行的"辅助填价"按钮，窗口列出：二维库里最接近的 3 条相似记录、经验池同类均值、上个项目同要素的填值。他对比来源和风险提示后，选择其中一条，点"确认写入"，该行变为"人工确认"状态并留痕。

### 场景 2：多候选冲突裁决

> 某行匹配到 2 条知识库记录但价格冲突（当前逻辑填"待复核"）。辅助填价窗口把两条候选并排展示，各自显示完整要素、单位、备注列说明。小张看出其中一条的要素4其实不符合本项目场景，选择另一条确认写入。

### 场景 3：第二层经验值人工改判

> 某行第二层经验提示填了黄色经验值 1.0，但小张知道这个项目情况特殊。他打开辅助填价窗口，看到经验池里同类记录均值是 1.2（含 5 条历史样本），风险提示显示"当前经验值低于经验池均值 20%"。他选择经验池候选，确认写入，单元格变为"人工确认"色并批注原值。

## 三、核心流程

```text
1. 用户在结果预览中定位到待复核行（红色）或想改判的行
2. 点击该行的"辅助填价"入口
3. 弹出"本项填价窗口"
4. 窗口上方显示：本行要素1-5、单位、当前值、待填字段、匹配状态、匹配说明
5. 窗口下方列出候选项（按可信度排序）：
   候选值 | 来源 | 匹配理由 | 相似度 | 风险提示 | 是否推荐
6. 用户选择某候选（或手工输入自定义值）
7. 点击"确认写入"
8. 后端写入输出 Excel 副本（复用现有单元格写回接口的校验和留痕机制）
9. 预览刷新，该单元格显示"人工确认"标识
10. 操作记录写入填价操作日志
11. （可选）用户点击"重算公式"刷新汇总和 Word 报告
```

## 四、页面入口

| 入口 | 位置 | 触发条件 |
|---|---|---|
| 主入口 | 结果预览表格行级操作区（与行级 AI 按钮同区） | 该行存在待复核字段或用户主动点击 |
| 待复核清单入口 | 填价工作台"工作状态"模块的待复核数字 → 待复核清单 → 每条"辅助填价" | 转换完成后有待复核行 |
| 风险卡片入口（进阶） | 风险卡片上的"去处理"按钮 | 风险报告增强完成后 |

不建议做的入口：右侧 AI 聊天流中直接完成填价交互（AI 窗口只做摘要提示，见第十节）。

## 五、本项填价窗口结构

```text
┌─────────────────────────────────────────────────┐
│  本项填价 - 表4 第23行                    [关闭 ×] │
├─────────────────────────────────────────────────┤
│ 上下文区（只读）                                   │
│  要素1: 岩土工程勘察   要素2: 钻探      要素3: -    │
│  要素4: 黏性土        要素5: 30m以内   单位: m     │
│  待填字段: 基价/单价    当前值: 待复核              │
│  匹配状态: 未命中      说明: 从要素4开始没有匹配     │
├─────────────────────────────────────────────────┤
│ 候选列表区（可滚动）                                │
│  ○ 候选1  128.00  [知识库-相似]  相似度92%  ★推荐  │
│     匹配理由: 要素1-3+单位完全一致，要素4"黏土"近似   │
│     风险提示: 要素4不完全一致，需人工判断土类口径     │
│  ○ 候选2  135.50  [经验池均值]   样本5条            │
│     匹配理由: 非空要素顺序匹配同类5条，均值135.50    │
│     风险提示: 样本间最大离差18%                     │
│  ○ 候选3  128.00  [历史项目]    2026-05-XX项目     │
│     匹配理由: 同要素组合人工确认过一次               │
│  ○ 自定义输入: [________]                          │
│     风险提示: 自定义值不在任何候选来源中，请填写依据   │
├─────────────────────────────────────────────────┤
│ 确认区                                            │
│  填写依据备注: [________________（自定义值必填）]    │
│        [取消]              [确认写入输出副本]        │
└─────────────────────────────────────────────────┘
```

窗口要点：

- 窗口为模态弹窗（Dialog），符合当前 UI 规范中"浮层可用阴影"的例外条款。
- 上下文区字段与行级 AI 复核携带的诊断字段保持同一来源（匹配状态、匹配说明、候选数量等）。
- 候选列表单选，选中后"确认写入"才可用。
- 自定义输入必须填备注才能写入。

## 六、候选来源

| 来源 | 说明 | 生成方式 | 前提 |
|---|---|---|---|
| 知识库-相似 | 二维库中 A/B 未命中但"高相似"的记录 | 复用归一化后按要素前缀/单要素差异计算相似度（引入模式 C 及"仅一字段不同"作为**候选生成专用**口径） | 单位一致为硬条件 |
| 经验池同类 | 经验池 A/B 匹配同类记录及均值 | 复用经验池预警找同类逻辑 | 经验池有同类样本 |
| 历史人工确认 | 过往辅助填价/人工修改中确认过的同要素组合值 | 读取填价操作日志 | 有历史确认记录 |
| 第二层经验提示 | 知识库 J-M 列经验数（对系数字段） | 复用现有第二层逻辑 | 知识库唯一匹配且经验数可解析 |
| 规则说明参考 | 规则表中相关条目（对技术费系数） | 读取规则表相近条目 | 规则表有相近技术类别 |

**重要边界**：模式 C（前缀匹配）在本功能中**只用于生成候选列表**，永不自动写值。这与《要素1-5和单位的匹配模式介绍》中"模式 C 只作为辅助分析备选词汇"的既有口径一致。

---

## 七、候选排序规则

按以下优先级排序（结构化规则排序，不用大模型排序）：

1. **来源可信度**：知识库-相似 > 第二层经验提示 > 经验池同类 > 历史人工确认 > 规则说明参考
2. **相似度**：同来源内按相似度降序（相似度 = 命中要素数/总非空要素数，单位一致为前提）
3. **样本量**：经验池候选按样本数降序
4. **新近度**：历史确认候选按时间降序

并列时知识库来源优先。排序规则应写入配置（JSON），不硬编码。

---

## 八、候选可信度分级

| 级别 | 标识 | 条件 | UI 表现 |
|---|---|---|---|
| 高 | ★推荐 | 单位一致 + 仅 1 个要素差异 + 该差异为近义词/包含关系；或经验池样本≥5 且离差≤10% | 绿色标识，默认排最前 |
| 中 | 可参考 | 单位一致 + 2 个要素差异内；或经验池样本 3-4 条 | 默认样式 |
| 低 | 谨慎 | 前缀匹配仅命中要素1-2；或经验池样本≤2；或候选间价格离差>20% | 灰色 + 风险提示前置 |

分级阈值放入配置文件，允许后续调整，不硬编码。

---

## 九、风险提示规则

每个候选必须生成结构化风险提示（规则生成，非大模型生成）：

| 风险类型 | 触发条件 | 提示文案示例 |
|---|---|---|
| 要素差异 | 候选与本行存在要素差异 | "要素4不一致：本行'黏性土'，候选'黏土'" |
| 样本不足 | 经验池样本 < 3 | "仅 2 条历史样本，均值代表性有限" |
| 样本离散 | 经验池样本离差 > 20% | "样本间最大离差 35%，均值可能失真" |
| 候选冲突 | 多个高可信候选价格不一致 | "存在 2 个高可信候选但价格不同，请核对要素口径" |
| 偏离经验 | 选中值偏离经验池均值超过预警阈值 | "该值高于经验池均值 23%（高风险阈值 20%）" |
| 无依据 | 自定义输入 | "自定义值无来源依据，已要求填写备注" |

## 十、人工确认写入机制

1. **写入前校验**（复用 `/api/preview/cell` 现有校验）：公式单元格、合并单元格非左上角、隐藏行、系统生成列拒绝写入。
2. **写入内容**：值写入输出 Excel 副本对应单元格；单元格底色改为"人工确认"色（建议与现有人工修改浅蓝一致或单独定义，需与现有绿/黄/红体系区分）；写入批注（原值、候选来源、确认时间）。
3. **留痕**：写入独立的辅助填价日志（建议 `fill-assist-log.json`，或与 `preview-manual-edits.json` 统一为一套留痕 schema——**建议统一**，避免两套日志），记录：sheet、行号、列、原值、新值、候选来源、匹配理由、用户备注、时间。
4. **匹配说明列同步**：若开启匹配报告输出，该行"匹配说明"追加"人工经辅助填价确认，来源：XX"。
5. **汇总刷新**：沿用 v5.3.7 口径——写入只做快速写值，公式汇总与 Word 报告由用户手动点"重算公式"。

## 十一、与右侧 AI 助手的关系

| 分工 | 本项填价窗口 | 右侧 AI 助手 |
|---|---|---|
| 候选展示与选择 | ✅ 主交互 | ❌ 不承载选择交互 |
| 候选摘要提示 | — | ✅ 用户打开窗口时，可在聊天流弹出**前 2 条候选的摘要**（只读文本 + "打开填价窗口"按钮） |
| 候选解释 | 展示结构化理由 | ✅ 用户可追问"为什么推荐候选1"，AI 基于候选的结构化理由和知识库证据解释 |
| 写入动作 | ✅ 唯一写入入口 | ❌ 永不代为写入 |

AI 助手在此功能中保持"解释员"角色：可以解释候选来源和差异，不能新增候选数字，不能替用户点确认。

---

## 十二、与经验池预警的关系

- **复用**：候选来源中"经验池同类"直接复用预警模块的找同类逻辑（A→B→D）和均值计算。
- **联动**：确认写入后，若该行已有预警参数，提示"写入值与经验池均值偏离 X%"（复用预警阈值判断）。
- **边界**：辅助填价确认的值**不自动进入经验池**。经验池数据只通过现有"经验池导入模块"进入，维持数据来源的可控性。

---

## 十三、与当前 Excel 预览人工编辑功能的关系

当前 v5.3.6/5.3.7 已有"预览人工修改"（双击单元格直接改值）。两者关系：

| 维度 | 预览人工修改（已有） | 辅助填价（新增） |
|---|---|---|
| 定位 | 自由改值（任何普通值单元格） | 带候选依据的辅助改值（价格/系数字段） |
| 交互 | 单元格内联编辑 | 独立弹窗 + 候选列表 |
| 依据 | 用户自行判断 | 系统组织候选和风险提示 |
| 留痕 | `preview-manual-edits.json` | 建议统一到同一套留痕 schema，加 `source` 字段区分 |
| 写回通道 | `/api/preview/cell` | **建议复用同一写回接口**，只是入参多候选元数据 |

**给 Codex 的关键建议**：辅助填价的写入通道应复用现有 `/api/preview/cell` 的校验与写回，不要另起一套写 Excel 的代码路径，避免两处写值口径漂移。

---

## 十四、不做什么

- ❌ 不做"一键批量填入所有推荐候选"（第一版不做，防止绕过逐行人工判断；进阶版可讨论"批量打开逐条确认"）
- ❌ 不让大模型生成候选数字或决定排序
- ❌ 不放宽主匹配规则（A/B 两档不变，模式 C 只在候选生成中使用）
- ❌ 不把确认值反写经验池或正式知识库
- ❌ 不修改原始输入 Excel
- ❌ 不做多人审批流（单机版无用户体系）

## 十五、MVP 版本

**范围**：只针对"基价/单价"字段的待复核行。

1. 结果预览待复核行增加"辅助填价"按钮。
2. 弹窗显示上下文区 + 候选列表（先做 2 个来源：知识库-相似 + 经验池同类）。
3. 候选排序与可信度分级按本文规则（简化为高/低两级可接受）。
4. 单选候选 → 确认写入 → 复用现有写回接口 → 留痕 + 批注。
5. 无候选时明确显示"未找到可参考候选，请人工处理"，允许自定义输入 + 必填备注。

**MVP 明确不含**：系数字段辅助、右侧 AI 摘要联动、历史人工确认来源、风险卡片入口。

## 十六、进阶版本

1. 扩展到实物/技术工作费调整系数字段（增加第二层经验提示和规则说明来源）。
2. 右侧 AI 助手联动：打开窗口时聊天流推送前 2 条候选摘要；支持追问候选解释。
3. 历史人工确认来源接入（读取填价操作日志）。
4. 待复核清单批量入口：从清单逐条打开窗口（仍逐条确认，不批量写入）。
5. 与人工复核闭环打通：确认写入后该行复核状态自动置为"已处理"。

## 十七、验收标准

### MVP 验收

1. 100 行样例表中人为构造 3 行待复核，每行都能打开窗口并显示上下文信息，与该行实际匹配说明一致。
2. 知识库-相似候选：构造"仅要素4不同"的输入行，候选列表能列出该知识库记录，相似度和差异说明正确。
3. 经验池候选：经验池中有同类记录时，候选显示均值和样本数，与预警模块计算一致。
4. 确认写入后：输出 Excel 副本单元格值正确、底色和批注正确、留痕 JSON 记录完整（原值/新值/来源/时间）。
5. 写入公式单元格、合并单元格非左上角被拒绝且提示明确。
6. 无候选时显示明确空态；自定义输入不填备注无法提交。
7. 现有回归测试全部通过（`python -m pytest backend/tests -v`），主匹配结果与答案表一致性不受影响。

### 进阶验收

8. 系数字段候选能区分第二层经验提示来源和经验池来源。
9. AI 摘要只在聊天流显示前 2 条候选文本，不提供写入按钮。
10. 重新转换后，旧确认记录不自动应用，界面明确提示。

## 十八、给 Codex 的技术关键路径

> 以下仅为建议路径，Codex 应结合现有代码结构自行判断。

1. **后端候选服务**：新增 `fill_assist.py`（建议独立模块，不改 `fill_engine.py` 主逻辑），提供：
   - `POST /api/fill-assist/candidates`：入参 sheet/行号/目标字段，返回候选列表（复用 `knowledge_base.py` 的归一化与匹配、`experience_warning.py` 的找同类）。
   - 候选生成中的相似匹配（模式 C 变体）写在 `fill_assist.py` 内部，**不要**改动 `knowledge_base.py` 的 A/B 主匹配函数签名和行为。
2. **写回复用**：确认写入走现有 `/api/preview/cell`（或在其上包一层 `POST /api/fill-assist/confirm` 附带候选元数据），留痕 schema 与 `preview-manual-edits.json` 统一。
3. **前端**：新增 `FillAssistDialog` 组件（模态弹窗），入口挂在结果预览行级操作区；样式走 `daweiba-*` 命名空间，遵守 UI 规范（弹窗可有阴影，内部无卡片套娃）。
4. **配置**：候选排序权重、可信度阈值放入 `05-经验池-预警数据/` 下新 JSON 配置（与现有 settings 文件同目录同风格）。
