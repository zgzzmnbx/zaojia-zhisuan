## 设计目标

为“造价智算”建立统一、可延续的界面原则，让后续前端优化、参赛展示、AI 助手和表格预览不再只靠临时审美调整。

目标不是做一个“漂亮网页”，而是做一个可信、可复核、能反复使用的工程造价工作台。

## 设计定位

造价智算不是营销落地页，也不是普通聊天机器人。它是：

```text
Excel 工作台 + 规则审查台 + AI 复核助手 + 成果输出中心
```

界面应体现“工程造价辅助智能体”的专业、可信、可复核和高效，优先服务重复使用、表格审查、规则追溯和成果输出。

当前参赛口径属于“桌面办公助手创新赛道”。界面应保留 Excel 和 Word 交付习惯，同时把 AI 问答、行级复核、风险报告和经验池预警嵌入同一个工作台，让用户感知 AI 正在解决真实工作问题，而不是额外增加一个孤立系统入口。

## UI 基线已固化

从 `v5.3.3` 起，造价智算 UI 基线正式固化为：

```text
大尾巴主题唯一主界面 + Navattic 极简 SaaS 工作台
```

执行口径：

- 不单独新建 `design.md`；本文件就是全局 UI 总纲，`00-PRD/03-UI设计规范.md` 是可执行设计规范。
- 后续前端线程做 UI，只以本文件、`00-PRD/03-UI设计规范.md` 和 `01-assets/01-UI参考图/00-reference/` 中带 `【尾巴选】` 的参考图为准。
- 候选 A / D、旧主题、旧宽屏方案、旧流程条、旧演示 demo 都不再作为新 UI 依据。
- 如现有实现与 Navattic 极简绝对法则冲突，应视为待修复问题，而不是审美分歧。
- README 只保留 UI 入口，不展开设计细则；AGENTS 只保留长期协作规则，不承载视觉细节。

后续做 UI 优化时，不再只说“高级一点”“好看一点”。应采用“生产级工作台参考 + 本项目业务约束”的方式：

- 优先参考生产级 SaaS、数据工作台、审查台、表格工具、AI 助手侧栏。
- 当前视觉标准优先参考 `01-assets/01-UI参考图/00-reference/` 中带 `【尾巴选】` 的图片，以及 `Navattic web Mar 2026-精选` 下的 144、42、155、124 等截图。
- `00-PRD/PRD-Claude建议包-2026年7月3日/08-UI优化PRD建议.md` 和两张 Claude UI 建议图可作为信息架构、演示节奏、风险摘要表达和局部交互参考；暂不作为新的视觉标准，不覆盖本文件和 `03-UI设计规范.md` 已固化的 Navattic 极简风格。
- 可以学习 Linear、Stripe、Notion、现代 BI 工具的克制排版、清晰层级和组件节奏。

- 参考图只学习布局、密度、色彩、层级和交互节奏，不复制无关内容，不改变造价智算业务流程。

## Navattic 极简绝对法则

本节为当前 UI 总纲最高优先级。后续前端实现、UI 修复、模块重排，只要与本节冲突，一律按本节执行。

### 1. 布局绝对法则：严禁卡片套娃

- 废除“灰底白卡”的层级结构，不再用大块灰色背景承托多个白色卡片。
- 大工作区背景必须使用统一的纯白或极浅灰，例如 `#FFFFFF`、`#F7F9FC`。
- 左侧栏、中间工作区、右侧智算侧栏、表格区之间只通过 `1px solid var(--db-border)`、分割线和留白分组。
- 功能面板不能再套一层有背景色的卡片容器；表格、预警、报告、知识库状态页都应像 SaaS 工作台一样直接落在统一工作区内。

### 2. 视觉绝对法则：扁平化与去阴影

- 除 Dialog / Modal、Dropdown、Toast 这类浮层外，系统内所有常驻面板、侧边栏、表格区、工具条、模块工作区一律禁用 `box-shadow`。
- 常驻界面不使用玻璃拟态、半透明浮层、大面积阴影和厚重浮卡。
- 层级靠边框、字号、字重、行距、分割线和内容密度表达，不靠阴影堆叠。

### 3. 导航与交互法则：克制选中态

- 左侧导航激活态禁止高饱和或高对比度大色块填充。
- 激活态必须改为文字加粗加深，配合极弱浅灰底，或透明底加 `1px` 细线 / 左侧细条指示。
- 鼠标 hover 也只允许轻微浅灰底，不允许整块蓝色、绿色、紫色铺满。
- 左侧模块导航不显示 `01/02/03` 等数字序号；模块识别依赖图标、名称、说明和克制选中态，避免编号占用视觉焦点。

### 4. 组件视觉法则：按钮精调

- 主按钮取消边框，使用纯深蓝底、白字、圆角 `6px-8px`，参考 `【尾巴选】蓝色按钮-Pasted image 20260701083915.png`。
- 次按钮透明底或白底，加浅灰边框，文字为深灰或深蓝。
- 所有按钮 padding 必须充足，禁止干瘪、文字贴边、纵向过厚或胶囊化过度。
- 设置按钮、工具按钮、导出按钮都应使用同一套低高度、轻边框、无阴影规则。
- 结果预览窗口顶部工具条属于紧凑表格工具条：按钮高度应控制在约 `28px-32px`，完成态按钮应降权为浅色轻量态，避免与主操作按钮争抢视觉层级。

## UI 主题与布局策略

当前前端只保留一套主界面：`大尾巴主题 / daweiba`。

- 不再维护多主题、多布局组合。
- 不再提供旧布局、旧主题、旧宽屏方案、隐藏旧流程条等页面设置入口。
- 大尾巴主题不是皮肤，而是唯一产品级布局层。
- 后续 UI 工作默认直接在大尾巴主题中推进，除非大尾巴重新提出新的独立布局方向。

### 大尾巴主题方向

“大尾巴主题”是造价智算当前唯一正式主界面。它重新回答：用户进入系统后如何选择模块、如何查看当前表格、如何让 AI 随行复核。

布局目标：

- 左侧为模块导航区：用于选择上传填价、结果预览、经验池预警、工作量抓取、报告输出、知识库问答等模块。
- 中间为程序主体区：优先承载当前模块的核心工作对象，尤其是 Excel 表格预览、转换结果、预警列表和报告生成状态。
- 右侧为 AI 随行助手：承载问问智算、行级复核、知识库问答、风险报告和当前页面解释。
- 左侧导航和右侧 AI 助手都应可折叠；折叠后中间主体区获得更多宽度。
- 当前阶段仍复用既有业务组件和数据流，不重构各模块内部功能逻辑。
- 视觉上必须完全转向 Navattic 式极简 SaaS：白底、细线、扁平、少色、低干扰，不再做厚卡片工作台。

### 管勘智算 v2.0 主页界面逻辑

创建日期::2026-06-30 18:59

本节是大尾巴对“管勘智算 v2.0 / 造价智算”主页工作台的最新界面逻辑要求，后续实现优先级高于此前第一阶段落地描述。

整体布局必须是宽屏软件工作台，不是可滚动网页。默认面向桌面宽屏使用，不再为了竖屏 / 移动端适配牺牲主工作台体验。

布局实现不得按某个固定分辨率写死，例如只针对 1366、1440 或 1920 宽度手工堆像素。应采用“桌面优先”的响应式工作台布局：左侧导航和右侧智算侧栏可使用稳定宽度 / 可调宽度，中间工作区使用 `flex: 1`、`minmax()`、`clamp()`、容器宽度约束和必要断点吃满剩余空间；普通笔记本、宽屏显示器、Tauri WebView 窗口和浏览器窗口缩放时，三栏比例应自然重排，不出现主工作区被挤成窄列、右侧溢出、大片空白或必须依赖固定窗口尺寸才能好看的情况。

主界面固定为三段：

- 左侧“当前模块”窗口：负责模块导航和当前模块状态。
- 中间模块工作区：只展示当前选中模块的主体内容，必须填满中间区域。
- 右侧“随行助手 / 智算”窗口：对标左侧“当前模块”窗口的视觉和交互逻辑，上下顶格，宽度可调。

左侧“当前模块”窗口包含以下模块：

| 当前模块 | 中间工作区内容 |
| --- | --- |
| 填价工作台 | 上传标准 Excel、转换概览、工作状态 |
| 结果预览 | 可视化表格窗口 |
| 经验池预警 | 经验池预警模块、经验池导入模块 |
| 工作量抓取 | 原始工作量抓取模块 |
| 报告生成和预览 | 当前报告生成状态、下载按钮、报告预览窗口 |
| 知识库问答 | 知识库状态显示页面、索引状态、资料覆盖范围、检索入口、常用查库问题、近期命中证据摘要 |
| 智能协同 | 飞书连接状态、新建协同任务、我的任务 / 待我复核 / 待我审批、任务详情、处理进度、风险摘要、成果和同步入口 |

左侧“当前模块”窗口底部需要增强为状态仪表盘：

- 模块导航下方留出固定底部状态区，不再只放零散数字。
- 底部状态区放一个环形图，用于展示当前表格的匹配状态，例如结构匹配 / 待复核 / 预警 / 未运行。
- 环形图旁或下方展示关键表格数据：输入行数、可视化行数、待复核数、预警数、当前预警是否已运行。
- 环形图必须参考 Dashboard donut chart，不做普通饼图：中间显示总数和 `Total`，外侧用浅灰引线标注“数值 + 百分比”，颜色分段克制，不使用粗糙的大色块饼图；所有饼图 / 环形图的绿色分段统一使用 `rgb(117, 188, 146)`。
- 当没有上传或没有转换结果时，底部状态区显示空状态，例如“未运行”，但仍保留仪表盘位置，避免界面跳动。
- 点击左侧不同模块时，底部状态区仍显示全局处理状态，而不是被模块内容挤掉。
- 左侧收起时，环形图和详细数字可收敛为小图标 / 小圆环 / 简短数字，展开后恢复完整状态。

中间模块工作区要求：

- 中间区域必须按“当前模块”切换，不再把所有模块纵向堆在一个长网页里。
- 当前模块内容要尽量填满中间区域，上下顶格，视觉上像桌面软件窗口。
- 当前模块内容不得缩成左侧小卡片后在中间或右侧留下大片空白；填价工作台、结果预览、报告预览等模块都必须按中间区可用宽度重新分配比例。
- 填价工作台不再显示表格预览；宽屏下采用上层两列、下层工作状态：上层从左至右为上传标准 Excel、转换后概览，下层为工作状态并横跨整行。
- 填价工作台的工作状态模块承接原表格预览底部摘要、匹配状态和匹配质量分布：包括“输入 xx 行、匹配 xx 行”、价格列、输出文件、实物 / 技术一层命中、经验提示、复核数量、预警数量、高风险和低风险数量。
- 工作状态模块中的“匹配状态”和“匹配质量分布”应按大尾巴截图口径展示：左侧为完成行数、待复核和预警摘要，右侧为环形比例图和高置信 / 低风险 / 高风险分布。
- 结果预览模块只保留填价结果预览表格和必要工具条，必须横向铺满中间工作区；即使当前预览列较少，也应让窗口和表格承载区占满主区域，而不是只按内容宽度收缩。
- 填价工作台的转换概览主卡属于信息概览，不属于“标准命中”业务状态；即使转换完成，也不得用绿色或薄荷色整块铺底。绿色只用于明确的业务状态点、状态标签、状态环图分段。
- 左侧窗口收起时，中间工作区自动获得更多横向宽度。
- 右侧智算窗口收起时，中间工作区自动获得更多横向宽度。
- 左右两侧都收起时，中间工作区应成为主工作台，占据最大可用宽度。
- 右侧智算消息头像使用单字母角色胶囊：助手显示 `Z`，用户显示 `U`；助手使用与左侧环形图一致的绿色底，用户使用蓝色底，头像底色不得与相邻消息气泡混成一体。
- 表格类模块必须保留横向滚动、固定 AI 列、状态色和数值右对齐。
- 经验池预警页拆成上下两个模块：上方为经验池预警模块，承接“运行经验池预警分析”按钮、预警进度、预警摘要、风险详情和定位预览入口；下方为“经验池导入模块”，承接原“导入已有控制价文件到经验池”，默认折叠，展开后再显示文件选择、字段选择、sheet 映射和导入结果。
- 报告生成和预览模块应像独立成果页：左侧或顶部展示生成状态和下载动作，主体展示报告预览，不与填价表格混杂。
- 知识库问答模块不是普通聊天页，应展示知识库运行状态，包括是否已索引、资料来源、二维库 / 规则卡 / 标准资料覆盖情况、检索入口和常用问题。

右侧“随行助手 / 智算”窗口要求：

- 右侧 AI 工具栏必须对齐左侧“当前模块”窗口的面板逻辑：同一套白底、1px 轻边框、低圆角、顶部身份区、分割线、底部状态 / 操作区；禁止分组卡片套娃。
- 右侧 AI 窗口上下顶格，像软件侧栏，不像页面中漂浮的聊天卡片。
- 右侧宽度保持可调，默认宽度仍按当前智算窗口设置；折叠态只保留窄条入口。
- 顶部身份区参考 Navattic Support / Settings 右侧面板：返回 / 收起、图标、名称、状态、展开或关闭动作清楚排列。
- 对话区不得沿用旧式聊天窗口视觉；消息、证据、快捷操作和输入框应融入右侧面板体系。
- 对话消息参考 Pipedrive Live Chat 红框中的聊天流：会话顶部为轻量身份栏，消息区白底独立滚动，AI / 对方消息靠左使用浅灰圆角气泡，用户 / 操作消息靠右使用蓝色圆角气泡；气泡旁使用头像或短标识，不再在每条消息上方重复大段“智算”标签。
- 旧版蓝色整块提示卡、厚重卡片套卡片、每条消息上方重复角色文字的样式不再用于大尾巴主题。
- 输入框固定在底部，消息区独立滚动。
- 智算内容应服务当前模块：填价时解释转换和匹配，预览时解释当前行，经验池时解释预警，工作量时解释抓取结果，报告页时辅助报告说明，知识库页时解释证据来源。
- AI 助手不能压过中间主工作区；它是随行复核员，不是主页面。

### 修改原则

- 后续 UI 优化默认作用于大尾巴主题，不再同时维护多套主题和布局。
- 外部 AI 的 UI 建议必须先抽象为“信息架构 / 交互问题 / 局部组件参考”，再与本项目 UI 规范比对；不得因为建议图好看就改动当前色彩、圆角、阴影、导航选中态和按钮标准。
- 新增样式优先进入 `daweiba-*` 布局层或当前模块内部组件，避免把模块逻辑重新堆成网页流。
- 后续前端线程如发现现有实现存在灰底白卡、卡片套娃、常驻阴影、激活态大色块，必须视为待修复问题，而不是可选审美差异。
- 状态色不随视觉调整乱变：第一层命中为绿色，第二层经验为黄色 / 琥珀色，未命中 / 高风险为红色，知识库调用为蓝色。
- 状态色不随主题乱变：第一层命中为绿色，第二层经验为黄色 / 琥珀色，未命中 / 高风险为红色，知识库调用为蓝色。

## 设计原则

- 业务优先：上传、转换、预览、预警、问答、输出这些主流程必须清晰可见。
- 表格优先：控制价表格是核心工作对象，表格预览应有足够宽度、稳定列宽和清晰滚动体验。
- 解释随行：AI 助手应像随行复核员，贴近当前行、当前结果和当前风险，不做孤立聊天入口。
- 设置克制：设置入口统一、低高度、少文字；复杂设置进入弹窗或侧栏，不挤占主流程。
- 状态明确：长任务必须显示阶段、进度和当前动作，避免用户误以为卡死。
- 可审计：颜色、图标、标签和说明都要服务“命中层级、风险等级、是否待复核”的判断。
- 过程可见：系统不仅展示结果，还要展示依据来源、规则命中、经验偏离、待复核原因和下一步动作。

## 布局原则

- 当前只保留大尾巴主题主布局：左侧模块导航、中间工作主体、右侧 AI 随行助手。
- 左侧适合放模块选择和全局导航，中间优先给表格预览和当前模块主体，右侧适合放“问问智算”。
- 左侧导航和右侧 AI 助手必须可折叠，折叠后优先扩大中间主体区。
- 宽屏模式下优先扩大表格预览，而不是堆叠更多说明卡片。
- 欢迎页和界面微调属于展示层，不应影响后端处理和输出结果。

## 表格预览原则

- 表格预览是主视觉中心，不被宣传卡片、AI 助手或设置区压缩成附属区域。
- 数字最多显示小数点后 2 位。
- 长文本默认按列宽换行，超出 2 行隐藏，悬浮显示完整内容。
- 匹配成功、经验提示、未命中、预警等状态要用稳定颜色体系，不随主题任意漂移。
- AI 操作列应固定在容易点击的位置，不要求用户横向滚到最后。
- 隐藏空行、表头行、显示列、列宽等偏好应可保存，并在首次打开后自动生效。

## AI 助手原则

- 默认开场简洁，不做过度拟人化欢迎。
- 可动虚拟形象只允许作为“随行复核员”的轻量身份和状态反馈层，优先放在右侧智算顶部身份区、折叠入口或演示讲解节点；不得做成漂浮桌宠、全屏角色、营销插画或遮挡表格的动效。
- 行级 AI 复核由用户手动触发，不自动发送。
- 大模型等待态要显示“检索依据 / 整理证据 / 生成回答”等阶段。
- 大模型不可用时，不能影响 Excel、Word、预警和工作量抓取主流程。
- 回答必须围绕当前行、当前规则和本地证据，不直接裁决最终价格。
- 知识库强制模式、普通问答、快捷指令和风险报告要有清晰状态差异，避免用户误以为所有回答都来自知识库。
- 虚拟形象动作必须对应真实系统阶段，例如待机、检索依据、整理证据、生成回答、命中知识库、发现风险、报告完成；不得用动作暗示模型已经裁决价格或系数。

## 设置入口原则

- 带文字的设置按钮统一为齿轮图标加“设置”。
- 原来只有图标的设置入口保持纯图标。
- 设置按钮高度要克制，避免大块圆角按钮破坏表格工具感。
- 模块内部设置应放在对应模块区域内；全局界面微调、隐藏空行等放在右上角页面设置中。
- 设置弹窗内部可滚动，不能因为配置项变多挤出屏幕。

## 参赛展示原则

- 首屏必须让人立刻知道这是“造价智算 / 工程造价辅助智能体”。
- 展示内容要突出规则链、数据链、智能体链协同，不只展示上传按钮。
- 展示内容要能讲清楚四类业务痛点：标准链长、数据链散、经验链断、审核链弱。
- 展示故事线优先采用“查标准变调能力、翻文件变问依据、人工套价变可信执行、历史文件变风险雷达、事后说明变过程治理”。
- 可以使用更有科技感的视觉语言，但不能牺牲真实业务流程可理解性。
- 对外文案统一使用“造价智算”“工程造价辅助智能体”，不暴露内部开发工具名称。

## 验收口径

- 新 UI 改动不改变任何匹配、预警、报告和下载逻辑。
- 关键控件在桌面宽屏和普通笔记本窗口下不遮挡、不溢出、不难点击。
- 布局按桌面优先响应式实现，不以固定分辨率写死；在普通笔记本、宽屏显示器和不同宽度 Tauri / 浏览器窗口下，左中右三栏能按可用空间自然伸缩。
- 表格预览仍是主视觉中心，AI 助手不压过主表格。
- 用户无需阅读说明文字，也能完成上传、转换、查看、预警和输出。
- 前端线程做较大 UI 改动前，应先阅读本文和 `00-PRD/03-UI设计规范.md`。
