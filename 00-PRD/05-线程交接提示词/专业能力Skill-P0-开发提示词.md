## 五、不可改变的产品决策

1. 用户入口叫“选择专业能力”，`Skill` 只作为辅助标签。
2. 入口位于填价工作台开始区域、上传 Excel 之前，不新增左侧一级菜单。
3. 当前只有“勘察测量最高投标限价编制”是真实上线能力。
4. “通用服务类造价测算”只显示为“规划中”，不可选择进入计算。
5. P0 不开发第二套通用服务类规则、输入模板或报告。
6. P0 不移动、复制、重命名或改写现有结构化计价库、规则表、经验池、报告模板和样例。
7. P0 不改变基价 / 单价、实物工作费调整系数、技术工作费调整系数的计算顺序、裁决逻辑和待复核口径。
8. `manifest.json` 是程序确定性读取的运行清单；`SKILL.md` 只供智能体和开发人员理解工作流程。
9. 不允许程序从 `SKILL.md` 自然语言中解析、生成或覆盖价格规则。
10. 不安装到用户的全局 `.codex/skills`，本次专业 Skill 是造价智算项目内的业务能力包。
11. 不建设 Skill 商店、在线安装、外部包执行、脚本热加载、签名市场或组织权限中心。
12. 规划中、未知和无效 Skill 必须安全拒绝，不能静默串用勘察测量资产。

## 七、Manifest 与 Registry 实现要求

### 7.1 首个上线 Skill

首个 Skill ID 固定建议为：

```text
survey-measurement-limit-price
```

显示名：

```text
勘察测量最高投标限价编制
```

Manifest 至少包含：

```text
id
displayName
version
status
domain
description
inputProfile
capabilities
assets
validation
```

其中：

- `status` 至少支持 `active`、`beta`、`planned`、`disabled`。
- 当前勘察测量 Skill 为 `active`。
- 当前通用服务类记录为 `planned`，不加载专业资产、不创建任务。
- 上线 Skill 必须具有完整、存在且位于项目允许范围内的资产引用。
- 规划 Skill 可以只有展示元数据和规划能力摘要，但后端必须拒绝创建任务。
- 全部路径使用项目相对路径，不写死 `D:` 盘符。
- Manifest 不得保存 API Key、App Secret、Token、个人信息或服务器内部秘密。
- Manifest 不得声明任意 Python 模块、Shell 命令、可执行文件或动态脚本入口。

### 7.2 路径安全

Manifest 资产路径必须经过统一校验：

1. 解析后必须位于项目根目录或明确允许的项目资产根目录内。
2. 拒绝绝对路径、`..` 越界、符号链接逃逸和不存在的上线资产。
3. 错误信息对用户说明哪个能力不可用，但不得暴露绝对路径和堆栈。
4. 公共 API 只返回能力名称、版本、状态、描述、能力摘要、资产数量/名称摘要和验证状态，不回显敏感路径。

### 7.3 默认配置

在现有项目默认配置中增加非秘密设置，例如：

```json
{
  "professionalSkills": {
    "_说明": "专业能力默认设置；运行秘密不得写入此处",
    "defaultSkillId": "survey-measurement-limit-price"
  }
}
```

实际合并时遵守现有 JSON 结构，不创建第二份互相冲突的默认配置。JSON 需要说明时使用合法的 `_说明` 字段，不写解析器不支持的注释。

## 八、后端 API 与任务快照

API 命名应适配项目现有风格，建议至少覆盖：

```text
GET /api/professional-skills
GET /api/professional-skills/{skill_id}
```

任务创建或 Excel 处理请求增加可选字段：

```text
skill_id
skill_version
```

必须实现以下语义：

1. 旧请求完全未传 Skill 时，使用项目默认勘察测量 Skill，保持旧客户端和 WeAct 兼容。
2. 新前端必须显式发送当前 `skill_id` 和 `skill_version`。
3. 显式传入未知、规划中、停用或无效 Skill 时返回结构化错误，不得静默回退。
4. 显式版本与当前 Manifest 版本不一致时拒绝创建任务并提示刷新能力清单。
5. 任务创建时生成不可变 Skill 快照，至少包含 ID、显示名、版本、Manifest 摘要哈希、创建时间和内部资产标识。
6. 已创建任务的预览、状态、报告、下载和协同继续使用该任务快照，不重新读取用户后来切换的选择。
7. 任务状态和结果 API 返回安全的 Skill 摘要，便于前端显示与追溯；不得返回秘密或绝对路径。
8. 切换当前 Skill 只影响后续新任务，不修改、重算或重标历史任务。

Manifest 异常和旧请求兼容要区分处理：

- “没有传 Skill”属于旧客户端兼容，可走已知默认能力。
- “明确传了错误 Skill”属于输入错误，必须拒绝。
- 默认 Manifest 暂时不可读时，可以保留一条只指向现有勘察测量流程的安全兼容路径，但必须显式记录告警；不得加载半套资产或换用其他专业资产。

不要为 P0 新建一套重复的计算引擎。Skill 上下文应包裹并调用现有勘察测量处理链，计算实现仍只有一份。

## 九、前端交互要求

### 9.1 选择入口

在填价工作台上传区域之前增加：

```text
选择专业能力                                      [查看能力详情]

[勘察测量最高投标限价编制]     [通用服务类造价测算]
 已上线 · 默认                    规划中

当前专业能力：勘察测量最高投标限价编制 · v1.0.0
```

要求：

- 页面从后端能力清单读取真实状态，不在前端复制两套业务配置。
- 只有一个可用 Skill 时仍展示选择入口，并默认选中勘察测量 Skill。
- 规划中卡片可以查看详情，但不能开始真实任务。
- 清单加载中、加载失败、无可用能力和版本失效均有明确状态。
- 能力详情至少展示适用范围、不适用范围、版本、能力摘要、资产摘要、验证状态、最后验证时间和已知限制。
- 不显示绝对路径、密钥或内部服务器信息。

### 9.2 当前 Skill 状态

- 用户确认后，在主工作区顶部持续显示紧凑的“当前专业能力”标识。
- 已有任务时，显示任务快照中的 Skill，不显示用户后来为新任务选择的值冒充历史任务能力。
- 如果用户在已有任务期间切换，提示“仅影响后续新任务”。
- 不要求用户在结果预览、Word 报告、问问智算等页面重复选择。

### 9.3 视觉与可用性

- 延续现有大尾巴主题和 Navattic 极简 SaaS 风格。
- 白底、浅灰分区、细边框、低圆角、克制状态色，不做高饱和应用商店。
- “已上线”“规划中”不能只靠颜色区分。
- 卡片和详情支持键盘焦点、Enter / Space、`aria` 标签和合理窄屏布局。
- 不新增左侧菜单，不挤压现有表格和右侧问问智算。
