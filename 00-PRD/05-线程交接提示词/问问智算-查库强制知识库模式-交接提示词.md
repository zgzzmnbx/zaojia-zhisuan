## 目标功能

在“问问智算”输入框中，如果用户问题以以下任一前缀开头，则强制进入知识库问答模式：

```text
查库：
查库:
@知识库
#知识库
```

前缀只作为模式开关，不作为最终问题内容。程序应去掉前缀和紧随其后的空白，再用剩余问题检索与回答。

## 行为要求

1. 前端识别触发词，用于即时给用户提示“将调用知识库”或同类状态，并向后端传递强制知识库模式参数。
2. 后端也必须兜底识别触发词，避免其他入口绕过前端判断。
3. 强制知识库模式必须先检索本地知识库、项目规则说明、结构化规则卡片和必要的当前行上下文，再调用大模型组织回答。
4. 回答界面需要明确显示“已调用知识库”或同类状态。
5. 如果没有检索到有效证据，回答必须固定表达为“当前知识库未找到明确依据，需要人工复核。”或同等明确措辞，不得让大模型凭常识补标准出处、价格、基价或系数。
6. 非触发词问题保持现有普通助手体验；可以解释、总结、写材料或调用快捷指令，但不承诺一定查库。

## 边界

- 不改变 Excel 填价、三数字匹配、经验池预警、工作量抓取和 Word 报告生成逻辑。
- 不让大模型裁决最终单价、基价、实物工作费调整系数或技术工作费调整系数。
- 不自动把问答记录写回正式知识库。
- 不把 API Key 写进前端代码、项目文件或代码存档。

## 建议实现位置

- 前端：`frontend/src/App.tsx`
  - 在发送“问问智算”消息前识别触发词。
  - 去掉触发词后发送干净问题。
  - 在聊天消息或等待态中显示知识库模式状态。

- 后端：`backend/app/main.py`、`backend/app/knowledge_qa.py`、必要时 `backend/app/schemas.py`
  - 给问答请求增加或复用强制知识库模式字段。
  - 后端对原始问题做触发词兜底识别。
  - 确保强制模式无证据时走固定未找到依据提示。

## 验收标准

- 输入 `查库：技术工作费调整系数 0.33 是怎么来的？` 时，必须走知识库问答路径，界面显示已调用知识库。
- 输入 `查库: 实物工作费调整系数有没有标准依据？` 时，必须走知识库问答路径。
- 输入 `@知识库 为什么待复核？` 或 `#知识库 第二层经验提示是什么意思？` 时，必须走知识库问答路径。
- 当本地知识库没有有效证据时，必须提示未找到明确依据并建议人工复核。
- 普通问题不加前缀时，不应被强制标记为知识库模式。
- 现有快捷指令、行级 AI 复核、风险报告、Excel 下载和 Word 下载不受影响。
