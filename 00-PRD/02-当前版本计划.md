# 当前版本计划

## 1. 文档用途

本文件只维护当前基线、最近完成事项、下一步优先级和已知阻塞。历史版本流水以 `CHANGELOG.md` 和 `CHANGELOG - 【归档】.md` 为准；模块长期需求以 `00-PRD/01-模块PRD/*/PRD.md` 为准。

## 2. 当前基线

- 产品名称：造价智算。
- 产品定位：工程造价辅助智能体。
- 当前版本：`v5.9.1`。
- 核心场景：长输管道勘察测量最高投标限价编制。
- 核心方法：规则裁决、模型解释、人工兜底。
- 当前主线：稳定专业计价闭环，补齐可信复核，再推进协同生产化。

## 5. 下一步建议

### P0：先恢复全量回归基线

这是下一项最优先工作，先于新增业务功能。

1. 修复知识库标准资料检索前 5 条未出现 `standard` 来源的问题。
2. 核对项目样例工作簿第 90 行缺少“利用钻孔抽水”的原因：确认是样例替换、数据漂移还是测试选错文件。
3. 修复后运行 `python -m pytest backend/tests -v`，目标是消除当前既有失败，形成 v5.9.1 可复现全绿基线。

边界：先查资料与测试基线，不夹带修改核心匹配规则；若真实资料口径已经变化，应修正测试和说明，不为“过测试”伪造业务数据。

### P1：完成数字化项目助手联合验收

1. 启动造价智算 `5174/8000` 与独立助手 `5175/8010`。
2. 从左侧入口验证 iframe 加载、嵌入主题、上传、计算、导出、独立打开和服务不可用降级。
3. 更新跨项目交接信箱，将状态从“待接入”收口为“联合验收通过”或列出真实缺陷。
4. 明确绿色版 / Tauri 是否需要自动启动独立服务；未明确前不把外部项目业务代码复制进宿主。

### P1：推进辅助填价与人工复核闭环

1. 在现有基价 / 单价候选基础上增加实物工作费调整系数和技术工作费调整系数候选。
2. 增加分字段人工确认、自定义值备注、候选来源和处理留痕。
3. 没有完整标准出处时显示“暂无标准出处映射”，不让大模型补条款。

承载 PRD：`00-PRD/01-模块PRD/09-辅助填价与人工复核闭环/PRD.md`。

### P1：建设结构化计价库只读质量检测

先做只读脚本或只读接口，检查重复组合、同条件价格冲突、单位缺失、经验数格式异常和关键字段空值，输出 Markdown 报告。

边界：不自动修库、不改变匹配引擎、不调用大模型裁决。

### P1：项目知识记忆生产化

1. 接入可信企业身份和项目成员授权，替换本地页面自填操作人 / 角色，并为通用知识配置生产级审核策略。
2. 增加 WeAct 候选待确认提醒和安全网页链接；消息内不直接确认或发布知识。
3. 增加正式依据更新后的冲突扫描与待复核提醒，仍由人工决定疑似失效或撤销。
4. 部署迁移时把 `Codex-Temp/runtime/knowledge-memory.sqlite3` 纳入运行数据备份，不混入 Git 或知识索引缓存。

边界：P1 不自动发布组织知识，不跨项目推荐，不让大模型批准知识或修改规则资产。

### P2：智能协同场景化编审

项目整体排序仍放在专业主线、回归基线和数字化项目助手联合验收之后；进入本模块开发时，按模块内部 P0 / P1 / P2 推进：

1. **模块 P0｜简易多人复核**：先完成 `编制 → 并行多人复核 → 复核完成`，冻结成果版本，通过 WeAct 通知和安全链接进入网页处理；独立收集每名复核人的结论和可定位意见，形成意见汇总并由编制人逐条回复；所有复核人通过后完成，任一人退回则生成修改轮次，不强制增加审批人。
2. **模块 P1｜标准编审与风险协同**：增加风险派单、截止时间、催办、可选审批、正式发布和归档；验证企业 WeAct 原生审批或经典卡片回调后，再开放可靠的通过 / 退回轻量动作。
3. **模块 P2｜场景智能与管理治理**：增加材料检查、流程和复核人推荐、审核摘要、版本差异、退回意见转待办、多维表格元数据镜像、统计分析、权限审计和生产监控。

统一边界：网页和后端任务服务是业务状态、成果版本和审核结论的唯一真相；WeAct 只负责人员触达、提醒、安全跳转和经验证的轻量操作，不直接修改价格、系数、规则或风险等级。

## 7. 本阶段不做

- 不重构核心匹配引擎。
- 不让大模型直接裁决最终价格或系数。
- 不批量重命名历史目录、知识库文件、样例和兼容脚本。
- 不把数字化项目助手业务代码复制进造价智算。
- 不在专业主线和回归基线未收口前优先建设复杂审批、台账和组织权限。
- 不在当前 Windows 主线中强行推进 Tauri Linux、deb 或 AppImage。

## 8. 当前验收口径

- 新线程能在本文件中快速找到当前版本、最近完成、下一步第一优先级和真实阻塞。
- 模块 PRD 状态与代码、README、CHANGELOG 和跨项目交接状态一致。
- 完成项与未来增强分开表达，不把 MVP 完成等同于全部产品化完成。
- 新功能开始前有对应 PRD；完成后回写需求状态、验证结果和版本记录。
