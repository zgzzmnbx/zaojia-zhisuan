# 当前版本计划

## 使用口径

本文件是版本级计划，不替代各模块 PRD。

- 它回答：当前版本已经具备什么、这一版已经完成什么、下一步优先推进什么、哪些事情本阶段不做。
- 各模块 PRD 回答：某个模块长期应该如何设计、有哪些需求、状态是什么、依赖哪些规则 / 模板 / 代码资产。
- 已完成事项不再长期堆积在这里；版本历史以 `CHANGELOG.md` 为准。

## 当前基线

- 当前产品名：造价智算。
- 当前定位：工程造价辅助智能体。
- 当前版本：v5.8.4。
- 当前阶段：已有可运行原型，正在从功能堆叠转向轻量 PRD 驱动和参赛展示优化。
- 首个落地场景：长输管道勘察测量最高投标限价编制。
- 总能力框架：工程造价辅助智能体 / 造价数字员工雏形。
- 核心方法：规则裁决、模型解释、人工兜底。

## Claude Code 建议吸收口径

外部建议入口：

```text
00-PRD/PRD-Claude建议包-2026年7月3日/
00-PRD/PRD-Claude建议包-2026年7月3日/Claude建议包吸收评估报告-2026-07-04.md
```

该建议包只作为外部产品和技术思路库，不是正式 PRD，也不是开发任务单。后续需要查询 Claude 建议时，先读吸收评估报告，再回看建议包原文；真正进入开发前，必须写入对应模块 PRD 或本文件。

当前吸收原则：

- 已吸收并落地为 MVP 的方向包括“经验池治理、结构化风险清单、演示 / 评委样例模式、标准依据轻量追溯、辅助填价 MVP”；后续只推进扩展项，不重复立项已完成 MVP。
- 需要特别区分：当前“标准依据轻量追溯”只是匹配说明、经验池来源和项目规则说明入口，不等同于 Claude 建议中的“规则资产依据列 / 依据出处映射”完整追溯。
- Claude 建议中的“知识库质量检测”仍未落地，是当前最明确的可信底座缺口；建议优先做只读脚本或只读接口，不自动修库。
- Claude 建议中的“演示 / 评委模式优化”当前只完成一键样例和轻引导，尚未完成 5 步讲解条、讲解点高亮、脱敏提示和正式讲解稿。
- 其他扩展方向统一先放入 P3 远期储备，后续由大尾巴随时调整优先级。
- UI 建议只吸收为信息架构、演示节奏和局部体验参考；暂不改变现已固化的 Navattic 极简风格、色彩 token、按钮规则和设计标准。
- 大模型仍只做解释、问答、风险报告、候选提示和审核意见草稿，不直接裁决最终价格、单价、系数或审减金额。

- 如果继续优化 PRD 体系，下一步重点不是增加更多文档，而是让新需求先进入对应模块 PRD，再按 PRD 实现，完成后反向更新状态。

## 本阶段暂不做

- 不重构核心匹配引擎。
- 不让大模型直接裁决最终价格或系数。
- 不批量重命名历史知识库、样例和兼容脚本。
- 不把 PRD 做成重型项目管理系统。
- 不把申报书里的宣传性表述直接搬成开发需求；只沉淀能指导产品、规则、交互和验收的内容。
- 不在本阶段强行完成统信 UOS 安装包、deb 包、AppImage 或 Tauri Linux 桌面版打包；统信方向先按本地网页版兼容验证推进。

## 验收口径

- 新功能开发前能找到对应模块 PRD，或者先补充当前版本计划和对应模块 PRD。
- 开发完成后，模块 PRD 的需求状态、README 当前模块状态和 CHANGELOG 需要能互相对上。
- 涉及规则、知识库、报告和 UI 的改动能找到清晰验收标准。
- 给其他 AI 或新线程接手时，能通过 `00-PRD/02-当前版本计划.md` 快速知道：哪些已经完成，哪些仍待确认，哪些本阶段不做。
