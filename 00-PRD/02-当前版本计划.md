- 核心方法：规则裁决、模型解释、人工兜底。

3. **模块 P1｜标准编审与风险协同**：增加风险派单、截止时间、催办、可选审批、正式发布和归档；验证企业 WeAct 原生审批或经典卡片回调后，再开放可靠的通过 / 退回轻量动作。
4. **模块 P2｜场景智能与管理治理**：增加材料检查、流程和复核人推荐、审核摘要、版本差异、退回意见转待办、多维表格元数据镜像、统计分析、权限审计和生产监控。

统一边界：网页和后端任务服务是业务状态、成果版本和审核结论的唯一真相；WeAct 只负责人员触达、提醒、安全跳转和经验证的轻量操作，不直接修改价格、系数、规则或风险等级。

## 7. 本阶段不做

- 不重构核心匹配引擎。
- 不让大模型直接裁决最终价格或系数。
- 不批量重命名历史目录、知识库文件、样例和兼容脚本。
- P0 不移动现有结构化计价库、规则表、经验池和报告模板，只通过 Manifest 引用。
- P0 不把“规划中”的通用服务类包装成已上线能力。
- P0 不建设在线 Skill 商店，不安装或执行任意外部 Skill 脚本。
- 对话式工作台不删除专业页面，不用模型文本伪造任务执行，不把全部聊天自动写入知识记忆。
- 不把数字化项目助手业务代码复制进造价智算。
- 不在专业主线和回归基线未收口前优先建设复杂审批、台账和组织权限。
- 不在当前 Windows 主线中强行推进 Tauri Linux、deb 或 AppImage。

## 8. 当前验收口径

- 新线程能在本文件中快速找到当前版本、最近完成、下一步第一优先级和真实阻塞。
- 模块 PRD 状态与代码、README、CHANGELOG 和跨项目交接状态一致。
- 完成项与未来增强分开表达，不把 MVP 完成等同于全部产品化完成。
- 新功能开始前有对应 PRD；完成后回写需求状态、验证结果和版本记录。
