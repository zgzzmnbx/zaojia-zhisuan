# 经验池预警 PRD

## 模块目标

基于历史控制价文件和经验池数据，对当前填价结果进行经验偏离预警，帮助用户发现可能需要复核的基价或系数。

参赛口径中，本模块承担“把历史文件变成风险雷达”的能力表达。历史项目不直接裁决当前价格，而是用于发现偏离、提示异常和辅助复核，兼顾经验价值与合规边界。

## 功能边界

- 经验池预警是独立复核模块，不直接修改主匹配结果。
- 大尾巴主题下，“运行经验池预警分析”按钮和预警明细只放在“经验池预警模块”，不再放在表格预览模块底部。
- “导入已有控制价文件到经验池”在界面上命名为“经验池导入模块”，与经验池预警模块同页上下排列，默认折叠以保证预警区有最大可视空间。
- 经验池导入不影响正式知识库和第二层经验提示。
- 当前按同类记录匹配和偏离率判断风险等级。
- 只核查指定列有值的行属于预警候选过滤，不应外溢到主价格匹配。
- 风险详情和智算风险摘要中的“跳到表格”只负责定位查看，不修改预警等级、匹配结果或输出 Excel。
- 经验池治理属于预警模块的数据质量底座，只能输出检查结果、问题行号和人工建议；不得自动删除经验池记录，不得把经验池数据升级为正式知识库，不得把预警结论反向改写主匹配规则。
- v5.4.0 后，经验池治理通过 `/api/quality/experience-pool` 输出结构化结果，并在运行缓存目录生成 Markdown 报告；治理报告只读，不触发预警写回。
- 当前经验池治理已对标 Claude 建议中的“只读数据质量检测”最小标准；后续增强只能补导出、筛选、处理状态等人工治理体验，不应扩展为自动清洗流程。

## 验收口径

- 预警能输出候选数、可比选数、未找到同类数和风险分布。
- 预警参数和预警细节能写回输出 Excel。
- 过滤字段为空、0、公式结果为 0 时按没有值处理。
- 经验池模板只读不改，新数据写入独立经验池文件。
- 预警结果应能支撑风险报告和行级 AI 复核，说明经验来源、偏离程度和建议复核方向。
- 高风险 / 低风险摘要中的“跳到表格”应复用同一套定位逻辑：跳到结果预览、切换目标 sheet、滚动到 Excel 行号，并对目标行或指标单元格做一次闪烁高亮。
- 经验池治理报告至少应包含重复记录、单位不一致、同类样本过少、极端值 / 离散值、关键字段空值和经验数不可解析等问题分类，并给出问题所在 sheet、行号和关键要素。
