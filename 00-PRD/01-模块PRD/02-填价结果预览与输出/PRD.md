# 填价结果预览与输出 PRD

## 模块目标

让用户在浏览器中快速查看填价结果、匹配状态、预警摘要和输出文件，并能按设置隐藏空行、调整显示列和查看完整说明。

## 功能边界

- 表2、表3、表4按业务列优先展示，其他 sheet 默认显示全部列。
- 大尾巴主题下，表格预览模块只在“结果预览”页承担主视图；“填价工作台”页不再重复显示表格预览。
- 结果预览末列 `AI` 是行级操作列，必须保持 `position: sticky; right: 0` 的固定右侧能力；该列内当前有两个操作：书本图标为辅助填价，机器人图标为行级 AI 复核。
- 原先表格预览底部的输入行、匹配行、价格列、输出文件、实物 / 技术 / 预警摘要，在大尾巴主题下迁移到“填价工作台”的“工作状态”模块。
- 预览必须优先显示公式计算后的值，数字最多显示小数点后 2 位。
- 隐藏空行设置只影响预览和下载，不改变匹配结果。
- 合计、合计含税、合计不含税等汇总行必须保留。
- 预览人工修改属于“输出副本编辑”，不得反写原始输入 Excel、结构化计价库、第一层规则表、经验池或核心匹配引擎。
- 第一阶段只允许编辑普通值单元格；公式单元格、合并单元格非左上角、隐藏行、受保护或系统生成的匹配说明列默认只读。
- 人工修改必须留痕：至少记录 sheet、Excel 行号、列名 / 列号、原值、新值、修改时间；导出的 Excel 中人工修改单元格建议使用浅蓝底或批注标识。
- 修改可能影响费用汇总、Word 报告金额或预览公式值时，保存单元格先快速写入输出副本并提示需要重算；用户点击“重算公式”后触发公式刷新、预览刷新和 Word 报告重写，重算失败时要明确提示用户“已修改但公式结果待打开 Excel 后刷新”。
- 大模型和问问智算可以解释修改影响，但不得自动替用户写入单元格；除非未来另立“AI 建议改值、人工确认写入”的需求。
- 辅助填价候选如进入预览页，只能作为人工确认入口：用户点选采用后，走现有输出副本编辑和留痕机制；候选本身不能绕过预览编辑规则，也不能反写原始输入、知识库、规则表或经验池。
- 辅助填价后续扩展以 `09-辅助填价与人工复核闭环/PRD.md` 为主入口；本模块只维护它在结果预览中的入口、固定 AI 列、弹窗承载和写回边界。
- v5.4.0 后，辅助填价确认写入复用 `/api/preview/cell` 的可编辑性校验、浅蓝底和批注留痕，并在人工修改日志里记录 `source=fill-assist` 与候选来源。
- 辅助填价弹窗当前为正式交互形态：弹窗使用全屏遮罩和居中 Modal，内部可滚动，底部确认动作固定；该功能不得再回到主表格下方的常驻详情区，避免破坏结果预览模块整体性。
- 辅助填价候选来源标签必须区分数据来源：知识库来源显示为绿色“知识库”，经验池来源显示为蓝色“经验池”；更详细的“相似 / 同类均值 / 样本数 / basis”放在候选说明和依据文本中，不挤在来源标签里。
- 辅助填价候选列表按相似度从高到低展示；当相似度相同时，知识库候选排在经验池候选前；再按置信度和经验池样本数做补充排序。
- 两阶段主流程中，点击“开始转换”只生成待匹配预览并留在当前工作台，不自动切到结果预览；右侧“问问智算”必须输出阶段提示和“跳转到表格预览”按钮，引导用户主动进入结果预览页。首次通过智算按钮进入结果预览后，智算聊天窗口应简单说明表格预览可切换 Sheet、查看匹配状态、调整列宽、人工修改普通单元格、打开 AI 列辅助填价 / 行级复核，并提供“批量匹配”按钮。预览窗口顶部、智算消息按钮、快捷指令“批量匹配”和输入框文字“批量匹配”必须复用同一批量匹配动作；该动作是正式价格和两个系数批量填写入口，不是辅助填价入口。
- 预览列宽拖拽只影响浏览器结果预览的显示宽度和本机用户偏好，不反写输出 Excel 的真实列宽，不改变下载文件、匹配逻辑、人工修改写回或 Word 报告。

## 验收口径

- 可视化表格与下载 Excel、Word 报告关键金额一致。
- 用户保存的 header_row、列宽、显示列偏好首次打开后也能生效。
- 列宽交互应保持页面简洁：不显示额外手柄、按钮或常驻提示，鼠标悬停到表头右边缘时变为可拖拽列宽的光标；拖拽后按 `sheet 名 + 列标题` 记住偏好。
- 长文本按设置换行截断，悬浮可看完整内容。
- AI 列固定在可操作位置，不要求横向滚到最后才能点击；书本和机器人两个按钮始终显示在同一个 `AI` 列内，不能脱离表格末列或被横向滚动卷走。
- 启用人工修改后，用户可双击或点击编辑按钮修改允许编辑的单元格；保存后预览立即更新。
- 下载 Excel 时包含人工修改结果和修改痕迹；重新上传 / 重新转换时应明确提示旧修改会失效或需要重新应用。
- 修改影响汇总金额时，用户点击“重算公式”后 Word 报告读取修改后的输出文件，不再读取修改前缓存金额。
- 辅助填价入口应清楚展示“候选来源 / 说明 / 是否采用”，不得把候选推荐表达为系统最终裁决。
- 辅助填价入口视觉验收：按钮使用书本图标，不使用“辅”文字；按钮与机器人图标保持同尺寸圆形按钮样式；点击后打开弹窗，主页面下方不得新增辅助填价内容块。
- 辅助填价候选视觉验收：知识库候选标签为绿色“知识库”，经验池候选标签为蓝色“经验池”；当存在经验池同类样本时，候选列表应能出现 `source=experience_pool` 的经验池候选。
- 辅助填价候选排序验收：100% 相似度候选应排在 75% 相似度候选前；同为 100% 时，知识库候选优先于经验池候选。
- 待匹配阶段应已生成预览数据但不得自动切页，也不得开放正式 Excel / Word 下载；用户通过右侧智算“跳转到表格预览”按钮进入预览后，可通过预览窗口按钮、智算消息按钮、快捷指令或输入“批量匹配”完成匹配；下载、重算公式、经验池预警和风险报告再进入正常可用状态。批量匹配完成后，中间预览窗口按钮不应突然消失，应保留为“已完成匹配”禁用态。
