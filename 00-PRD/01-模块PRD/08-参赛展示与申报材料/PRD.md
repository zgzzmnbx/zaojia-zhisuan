# 参赛展示与申报材料 PRD

## 模块目标

面向人工智能创新大赛和数字化部领导，讲清楚造价智算的业务痛点、技术路径、智能体能力和推广价值。

当前申报书 v1.7 已形成对外主线：造价智算是“工程造价辅助智能体”，以最高投标限价编制为首个落地场景，采用“规则裁决 + 模型解释 + 人工兜底”的可信 AI 机制，通过数据链、角色链、业务链三链融合，形成可复制的造价数字员工样板。

## 功能边界

- 表达要立足造价业务，但不能局限成单个表格脚本。
- 可以使用三链融合、人机协同、智能体、Skill、造价数字员工等概念，但必须服务真实业务。
- 展示材料不应夸大为完全自动裁决或替代专业人员。
- 视觉素材和申报材料与代码开发分开管理。
- 演示 / 评委样例模式本质是“固定样例 + 自动配置 + 轻引导”，不得另建一套匹配算法，不得改变知识库、规则表、经验池预警和报告生成口径。
- 演示模式不应污染日常工作态：演示引导只在样例加载后出现，正式工作流仍保留手动上传、列映射、转换和复核能力。
- 虚拟形象只作为“造价数字员工”的可视化讲解层，必须绑定真实系统状态；不得用夸张拟人台词替代业务依据、风险说明和人工复核边界。

## 验收口径

- 一页内能讲清楚“为什么需要、怎么解决、有什么价值”。
- 评委能看出规则可审计、数据可沉淀、AI 可解释。
- 当前 MVP 验收：点击“加载演示样例”后，完整控制价样例能自动跳过“费用汇总”，识别业务明细 sheet，生成结果预览、Excel 下载和 Word 报告，并可继续运行预警和风险清单。
- 完整演示验收：演示流程应覆盖上传 / 加载样例、匹配、预警、问答、风险清单和输出；5 步讲解路线能逐步点击走完，并能退出回到常规工作态。
- 虚拟形象演示验收：关键讲解节点的动作与系统阶段一致，关闭动效后演示流程仍完整；评委看到的是“随行复核员 / 造价数字员工”的状态表达，而不是游离于业务外的装饰物。
- 对外文案统一使用“造价智算 / 工程造价辅助智能体”。
- 对外价值表达应围绕“提效率、提质量、提感知、提效益、控风险、强创新”，并避免夸大为完全自动裁决。
