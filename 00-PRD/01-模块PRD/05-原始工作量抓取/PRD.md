# 原始工作量抓取 PRD

## 模块目标

从委托方工作量表中抓取数量、系数和备注等信息，写入控制价计算表，作为价格匹配前的独立预处理能力。

## 需求清单

| 状态 | 需求 | 说明 | 验收口径 |
| --- | --- | --- | --- |
| [已完成] | 上传工作量表和控制价计算表 | 独立于主价格匹配流程 | 能生成两个输出文件 |
| [已完成] | 按要素1-5和单位匹配抓取 | 采用字段完全匹配和非空要素顺序匹配 | 成功写入数量、系数、备注等字段 |
| [已完成] | 术语归并旁路规则 | 只作用于工作量抓取 | 不影响主匹配和经验池预警 |
| [已完成] | 只抓取指定列有值的行 | 默认按数量/工程量判断 | 空白、0、公式结果 0 跳过 |
| [已完成] | 抓取日志和诊断 | 成功、一对多、未命中、失败都有说明 | 输出标注表可复核原因 |
| [待开发] | 抓取前预览和候选确认 | 避免一对多时直接写入争议值 | 用户可确认候选后再写入 |

## 功能边界

- 工作量抓取不改变基价、系数匹配逻辑。
- 默认按字段完全匹配和非空要素顺序匹配执行。
- 术语归并规则只作用于工作量抓取，不得外溢到主匹配和经验池预警。
- 只抓取某一列有值的行是工作量模块内部过滤设置。

## 验收口径

- 输出标注工作量表和已抓取控制价计算表。
- 成功、一对多、未命中、失败都有日志和诊断说明。
- 公式单元格优先写入当前值，不复制公式文本。
- 字段设置保存后能用于后续自动识别。
