# Excel 填价与三数字匹配 PRD

## 模块目标

读取标准控制价 Excel，围绕表2、表3、表4完成基价、实物工作费调整系数、技术工作费调整系数三个核心数字的匹配、填入和来源说明。

## 功能边界

- 基价来自结构化计价库。
- 技术工作费调整系数优先使用第一层标准规则。
- 实物工作费调整系数当前第一层规则未启用，后续补规则前按未命中处理。
- 第二层经验提示只在第一层未命中时使用，并以经验参考方式标识。
- 不允许大模型自由生成最终单价或系数。
- 主流程采用“两阶段批量匹配”体验：`开始转换` 阶段不得批量写入价格或两个系数，只生成待匹配预览；`批量匹配` 阶段才调用既有匹配引擎完成价格、两个系数、诊断列、Excel 输出和 Word 报告。
- 辅助填价只服务“人工复核后采用候选”的下阶段能力，不自动填入正式结果，不一键批量采用候选，不把候选逻辑外溢到主匹配、经验池预警或第二层经验提示。
- v5.4.0 的辅助填价 MVP 只针对基价 / 单价字段；候选生成使用独立 `fill_assist.py`，模式放宽只用于候选展示，不改变主匹配。
- 辅助填价与标准依据追溯按“两步走”推进：第一步先完成基价 / 单价弹窗内合并展示候选和轻量追溯；第二步再扩展到实物工作费调整系数、技术工作费调整系数的候选和溯源。
- 辅助填价后续需求以 `09-辅助填价与人工复核闭环/PRD.md` 为主入口；本模块只保留与三数字主匹配相关的边界和验收口径。
- 辅助填价候选排序按相似度降序排列；相似度相同时，知识库候选优先于经验池候选；再按置信度和经验池样本数做次级排序。
- 当前辅助填价入口属于结果预览的行级操作：与行级 AI 复核共同放在固定 `AI` 列内，辅助填价使用书本图标，行级 AI 使用机器人图标；横向滚动宽表时该列仍应固定在右侧可点击。
- 辅助填价详情必须使用 Dialog / Modal 弹窗：顶部展示 sheet 和行号，中部展示待填字段、当前值、行上下文、候选来源、说明和风险提示，底部保留取消和确认写入动作；不得再在主页面或表格下方追加常驻内容。
- 价格辅助填价弹窗内的“依据追溯”只展示结构化线索和规则说明入口；完整“标准依据 / 依据出处”字段尚未维护时，不得把轻量线索包装成正式标准条款。

## 验收口径

- 输出 Excel 可打开，原始输入不被覆盖。
- 点击“开始转换”后，结果预览可见但 `matching_status=pending`，价格和两个系数尚未批量写入；下载 Excel / Word 应等待批量匹配完成。
- 点击“批量匹配”后，系统完成原本主流程中的价格、两个系数识别填写、公式刷新、报告生成和可下载输出。
- 命中第一层为绿色，命中第二层为黄色，未命中为红色。
- 逐行报告能说明当前命中层级、匹配说明和待复核原因。
- 修改匹配逻辑后必须运行后端测试。
- 辅助填价 MVP 若启动开发，应至少保留候选来源、候选排序原因、人工确认人机边界和写入留痕；没有结构化依据的候选不得写成确定结论。
- 辅助填价前端验收：AI 列内两个圆形图标按钮保持同列居中，AI 列横向滚动时固定在右侧；点击书本图标只打开弹窗，不改变当前结果预览页面整体布局。
- 价格依据追溯验收：点击书本图标后，同一弹窗内必须能看到价格候选和“当前价格依据追溯”；追溯至少展示匹配过程、经验池依据或项目规则入口之一，并明确轻量追溯不等于完整标准出处映射。
