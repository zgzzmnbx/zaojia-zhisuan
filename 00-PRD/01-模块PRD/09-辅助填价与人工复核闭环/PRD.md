# 辅助填价与人工复核闭环 PRD

## 模块目标

在结果预览的待复核行上，把知识库、经验池、第二层经验提示、规则说明和后续人工确认记录组织成结构化候选，由用户在弹窗中逐项确认后写入输出 Excel 副本，并形成可追溯的人工复核闭环。

本模块不是新的一级页面，而是结果预览、三数字匹配和问问智算之间的行级处理能力。它独立成 PRD，是因为后续将从价格候选扩展到两个系数候选、自定义值备注、候选配置、处理状态和风险入口，不能继续散落在多个模块 PRD 中维护。

## 优先级

当前优先级：P1。

P1 最小目标是在现有基价 / 单价辅助填价 MVP 基础上，补齐“两个系数候选 + 自定义备注 + 分字段人工确认”的闭环，不改变主匹配引擎，不让大模型生成数字。

## 候选来源

| 来源 | 适用字段 | 生成方式 | 边界 |
| --- | --- | --- | --- |
| 知识库相似 | 基价 / 单价 | 在单位一致前提下，按要素一致数量、差异项和相似度生成候选 | 只用于候选展示，不放宽主匹配 |
| 经验池同类均值 | 基价 / 单价、两个系数 | 复用经验池同类记录和均值 / 样本数 / 离散度 | 不自动进入正式知识库，不改变经验池预警算法 |
| 第二层经验提示 | 两个系数优先，必要时辅助价格解释 | 读取固定知识库 J-M 经验提示层 | 只在规则不足时作为经验参考，不包装成标准规则 |
| 第一层规则说明 | 技术工作费调整系数优先 | 读取程序规则和说人话规则表的命中或相近说明 | 没有维护完整出处时显示暂无标准出处映射 |
| 历史人工确认 | 价格和两个系数 | 读取人工修改 / 辅助填价留痕中的同要素组合确认值 | 暂不扫描历史输出 Excel，不反写正式资产 |
| 自定义输入 | 价格和两个系数 | 用户手动输入值并填写依据备注 | 必须留痕，必须提示无结构化来源风险 |

## 交互结构

辅助填价弹窗采用三段式：

1. 上下文区：展示 sheet、Excel 行号、待填字段、当前值、要素1-5、单位、匹配状态、候选数量、匹配说明、预警参数和预警细节。
2. 候选区：按目标字段展示候选值、来源标签、相似度、样本数、置信度、匹配理由、依据线索和风险提示。
3. 确认区：展示确认备注、自定义值输入、取消和确认写入动作；确认写入后关闭弹窗并刷新目标单元格。

多字段策略：

- 第一阶段继续以基价 / 单价为默认目标字段。
- P1 扩展后，弹窗内允许切换 `基价 / 单价`、`实物工作费调整系数`、`技术工作费调整系数` 三个目标字段。
- 每次确认只写入一个字段，不做跨字段批量采用。

## 数据与写回

- 候选生成由 `backend/app/fill_assist.py` 独立组织，不修改 `fill_engine.py` 主匹配行为。
- 确认写入复用 `/api/preview/cell` 的可编辑性校验、快速写值、浅蓝底、批注和 JSON 留痕。
- 辅助填价留痕继续优先统一到 `preview-manual-edits.json`，通过 `source=fill-assist` 和 `candidate` 元数据区分。
- 公式汇总和 Word 报告不在每次确认时自动重算，继续由用户点击“重算公式”刷新。
- 重新转换后，旧辅助填价确认记录不自动套用到新结果；后续如需复用，只能作为历史人工确认候选展示。

| 模块 | 关系 | 边界 |

## 功能边界

- 不让大模型生成价格、单价或两个系数。
- 不一键批量采用全部推荐候选。
- 不把辅助填价候选逻辑外溢到主匹配、经验池预警或第二层经验提示。
- 不把人工确认值自动写回正式知识库、经验池或规则资产。
- 不在主页面下方新增辅助填价常驻内容。
- 不新增左侧一级导航模块；入口仍放在结果预览行级 `AI` 列、待复核 / 风险处理入口和后续风险卡片中。
- 行级 AI 复核后的“详细情况”只是辅助填价弹窗的快捷入口，不在问问智算聊天流或复核提示框中直接提供写值动作。
- 不把“推荐候选”表达成系统最终裁决；按钮文案应强调“采用为人工确认值”。

## 验收口径

- 基价 / 单价候选、两个系数候选都能在同一辅助填价弹窗中按目标字段展示。
- 三个字段都必须支持候选选择、确认备注、确认写入、写回校验、Excel 批注和 JSON 留痕。
- 自定义输入未填写备注时不能提交；提交后日志必须记录自定义值和备注。
- 候选来源、相似度、样本数、风险提示和依据线索必须清楚展示，不得只给一个数字。
- 经验池候选存在时必须显示蓝色“经验池”标签；知识库候选显示绿色“知识库”标签。
- 目标单元格为公式单元格、合并单元格非左上角、隐藏行或系统只读列时，确认写入必须被拒绝并给出明确提示。
- 确认写入后不自动重算公式和 Word 报告；界面应提示需要时点击“重算公式”。
- 完整标准出处未维护时，界面必须显示“暂无标准出处映射”或等价提示，不能编造条款。
- 行级 AI 复核完成后的提示框和聊天回答气泡都应能显示“详细情况”按钮；点击后打开同一行的辅助填价弹窗，且目标行、sheet 和价格字段应与原行级复核行一致。
- 修改本模块后，至少运行辅助填价相关后端测试；涉及前端弹窗或按钮时运行 `npm run frontend:build`。
