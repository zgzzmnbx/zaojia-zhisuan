# 桌面端与评委版 PRD

## 模块目标

提供本机网页、Tauri 桌面壳和评委绿色版等入口，让用户和评委可以低门槛运行造价智算。

产品路线坚持“先助手、后平台”。原型阶段优先保障首个场景业务闭环和试点可用，通过桌面助手快速进入一线；推广阶段再将价格库、知识库、规则 skill、AI 复核 skill 和风险报告 skill 封装为服务能力，接入统一数字平台、大模型服务和数据治理体系。

## 功能边界

- Tauri 只作为第三入口，不重写 React 主界面和 FastAPI 业务逻辑。
- 统信兼容优先走“本地网页版”：FastAPI 后端托管 React 构建产物，用户在统信浏览器访问本机服务。
- 统信兼容不得牵动核心匹配引擎、三数字规则、知识库字段、经验池规则和报告生成口径；只处理运行环境、路径、启动脚本、依赖安装和浏览器访问适配。
- Tauri 统信安装包、deb 包、AppImage、系统托盘和国产系统桌面集成暂缓；除非本地网页版主流程已稳定并且明确需要离线安装包，否则不进入近期实现。
- 绿色版不携带真实 API Key。
- 关闭 Tauri 时只清理本次启动的后端，不杀掉外部已有服务。
- 启动和导出脚本应避免一闪而过，错误必须可读。

## 验收口径

- 双击入口能停留窗口并提示状态。
- 后端健康检查能返回当前版本。
- 评委版可以在未安装 Node 的电脑上运行。
- 统信本地网页版兼容验证时，应优先覆盖上传 Excel、执行匹配、表格预览、下载 Excel、生成 Word 报告、问问智算基础问答和大模型 Key 配置读取。
- 修改启动器、端口、环境变量或桌面壳时必须做运行态验证。
- 评委版和桌面端应体现低门槛试用能力，但不牺牲后续平台化治理、权限管理、数据安全和运行评价的扩展空间。
