## 一、目标

模拟一个尚未接入的外部造价系统：外部系统产生一条任务信号后，造价智算创建内部任务和任务专用待填写 Excel，并通过企业 WeAct / 飞书第二层应用机器人把“经典任务卡＋Excel”主动投递给经办人。

P0 真实演示先在唯一授权群“智算测试”中完成，同时底层必须保留按稳定平台用户 ID 精准单聊的能力。群聊和单聊共用同一个任务、文件、卡片、投递、幂等和审计模型，不能写成两套逻辑。

完成后的用户体验：

```text
智能协同页面模拟外部造价系统派单
→ 创建任务和待填表副本
→ “智算测试”群收到任务卡和 Excel
→ 卡片明确目标编制人
→ 用户进入智算助手或专业工作台处理
→ 系统能查询任务、投递和文件状态
```

## 三、现有能力边界

现有第二层机器人已经具备：

- 普通飞书 / 企业 WeAct 长连接。
- 群聊和单聊消息处理。
- 群成员确定性查询。
- 单个 `.xlsx` 受控收件。
- SQLite 顺序任务队列和任务查询。
- Excel / Word 成果文件上传与发送。
- 兼容企业 WeAct 的经典卡片。
- 事件 ID、消息 ID、平台时间和业务任务防重。
- 单实例、身份配置、云端端口覆盖和部署检查。

本任务必须复用这些能力，不另建第二个机器人、第二套凭证、第二套长连接或第二份专业处理逻辑。

重要边界：

- 第一层 Webhook 只能发送群通知，不能承担 Excel 文件上传、人员识别或精准单聊。
- 本功能的任务卡和 Excel 必须由第二层企业应用机器人发送。
- 企业 WeAct 必须继续使用经典卡片结构，禁止改用 `schema 2.0`，禁止发送 `update_multi=false`。
- 当前已验证机器人可以处理单聊，不代表已经验证可以主动联系任意用户；精准单聊真实发送必须单独验证权限和目标可达范围。

## 四、P0 必做范围

### 1. 标准任务信号 `TaskEnvelope`

建立清晰的数据模型，至少包含：

```json
{
  "event_id": "evt-20260722-001",
  "event_type": "task.assigned",
  "source_system": "模拟造价系统",
  "source_task_id": "ZJ-2026-001",
  "task_name": "某项目勘察测量最高投标限价编制",
  "project_name": "某项目",
  "skill_id": "survey-measurement-limit-price",
  "delivery_mode": "group",
  "target_chat_id": "受控目标群",
  "assignee_user_id": "目标编制人平台用户ID",
  "deadline": "2026-07-25T17:00:00+08:00",
  "instructions": "请根据附件完成最高投标限价编制",
  "input_artifact": {
    "template_asset_id": "待填表来源标识",
    "template_version": "v1.0"
  }
}
```

要求：

- `source_system + source_task_id + event_type` 是业务幂等键。
- 重复信号返回原任务和当前投递状态，不重复建任务、复制文件或发消息。
- `skill_id` 必须由现有 Skill Registry 校验；只有 `active` Skill 可创建执行任务。
- 姓名只能显示，人员路由必须保存稳定的飞书 / WeAct 用户 ID。
- 模拟器与未来真实造价系统必须共用同一个服务入口和数据模型；不得把模拟逻辑直接写死在前端按钮中。

### 2. “智能协同”页面模拟器

在现有“智能协同”页面增加“外部任务接入模拟”区域，不新增左侧菜单。

至少提供：

- 来源系统：P0 固定“模拟造价系统”。
- 外部任务编号。
- 任务名称、项目名称、截止时间、任务说明。
- 专业能力：复用真实 Skill Registry，只允许 `active`。
- 待填写 Excel：仅允许 `.xlsx`。
- 投递平台：普通飞书 / 企业 WeAct，复用现有机器人配置。
- 投递方式：群聊 / 精准单聊；P0 默认群聊，单聊根据能力状态显示可用、待验证或不可用。
- 目标群：真实测试只允许唯一“智算测试”群。
- 目标编制人：从受控群成员或人员映射中选择，前端显示姓名，后端保存稳定 ID。
- “创建并推送”按钮。
- 最近模拟任务：内部任务编号、外部任务编号、目标人、卡片状态、文件状态、投递时间、失败步骤和重试动作。

模拟器不得变成“任意文本＋任意群＋任意附件”的消息转发器。目标群、文件类型、事件类型、Skill 和动作必须服务端再次校验。

### 3. 任务专用待填写 Excel

- 用户上传或选择的待填模板视为只读来源。
- 派发时生成任务专用副本，建议文件名包含内部任务编号、项目和“待填写”。
- 不覆盖公共母版和用户上传原件。
- 保存模板来源、模板版本、文件哈希、生成时间和任务关系。
- 不因派发需求修改 Excel 表结构、合并单元格、公式、隐藏行列或既有样式。
- 文件存放在受控运行目录，路径通过 `pathlib` 处理；不得写死本机绝对路径到业务数据或代码。

### 4. 群聊投递闭环

投递顺序必须可审计：

```text
校验 TaskEnvelope
→ 创建内部任务
→ 生成任务专用 Excel
→ 保存待派发状态
→ 解析唯一“智算测试”群
→ 发送经典任务卡
→ 上传并发送 Excel
→ 标记待领取
```

任务卡至少显示：

- “新任务 / 待领取”。
- 任务名称。
- 内部任务编号和外部任务编号。
- 来源系统。
- 目标编制人。
- 截止时间。
- 当前专业能力。
- 任务说明。
- “进入智算助手”或“查看任务”安全链接。

P0 卡片按钮只做 URL 跳转，不直接领取、批准、修改或完成任务。

卡片消息和文件消息是一个投递包，但需要分别记录状态：卡片成功、文件失败时不能显示“完整送达”，重试只能补发失败步骤，不能重复发成功部分。

### 5. 精准找到用户的能力

即使 P0 真实演示先走群聊，也必须完成可测试的按人路由底座：

- 人员映射至少支持 `employee_id / account / display_name / platform / platform_user_id / enabled`。
- 目标编制人必须写入任务和投递记录。
- 投递接口支持 `delivery_mode=group|direct`。
- 两种模式共用任务、文件、卡片、幂等和审计服务。
- 群聊中如平台支持，使用目标用户真实 ID 形成提醒；无法可靠提醒时明确显示目标人，不伪造 @。
- 单聊无映射、用户停用、权限不足或用户不可达时进入明确失败状态，不按姓名猜测，不自动发给其他人或其他群。
- 自动化测试必须覆盖单聊成功 mock、无映射、权限拒绝和用户不可达；真实单聊不得向未经用户明确授权的个人发送。

### 6. 状态与存储

至少支持：

```text
待派发
→ 投递中
→ 待领取
→ 已领取
→ 待收件 / 处理中
→ 已完成或失败
```

优先扩展现有第二层 SQLite 和任务模型，保持自动迁移、旧数据兼容、30 天清理和任务查询可用。不得另建无法对账的平行任务数据库。

至少记录：

- `event_id`、业务幂等键、内部任务编号。
- 来源系统和外部任务编号。
- Skill 快照或可追溯标识。
- 模板来源、版本和哈希。
- 目标群、目标用户和投递方式。
- 卡片消息状态、文件消息状态、重试次数和脱敏错误。
- 创建、投递、领取、更新和完成时间。

### 7. 接口边界

接口名称按现有 FastAPI 风格确定，可以新增模拟、任务列表、投递重试和人员 / 目标解析接口，但必须遵守：

- 服务端校验来源、事件类型、Skill、群、人员和文件。
- 不提供任意消息转发接口。
- 不回显 App Secret、访问令牌、文件 Key、连接票据、完整 WebSocket 地址或永久匿名下载链接。
- 业务状态先落库，再发送消息；消息失败不回滚或删除任务。
- 专业处理继续调用现有 API 和任务冻结的 Skill 运行时上下文，不复制 `FillEngine`、规则、风险、报告或知识问答实现。

## 五、安全与真实测试规则

- 禁止向“[管勘智算]勘察测量投标限价智能体”群发送任何测试消息。
- 卡片、文字、附件和交互测试只允许投递到唯一“智算测试”群；找不到唯一同名群时必须拒绝发送。
- 自动化测试全部使用 mock，不访问真实平台。
- 真实群测试最多发送一组带明显“P0模拟测试”标识的任务卡和脱敏 Excel；不得携带真实敏感价格和人员隐私。
- 不记录 App Secret、访问令牌、连接票据和文件 Key。
- 前端和状态 API 不回显目标群 ID、用户 ID 等内部标识；可显示经过授权的群名和人员姓名。
- 使用现有企业应用 A 和项目默认身份配置；不得为了临时发送绕过预期 App ID / 域名校验。
- 本机和云端只能保留一个第二层机器人实例，真实测试前检查当前运行器，避免随机分发。

## 六、不做和不得破坏

- 本任务不实现 P0-B 多人复核、P1 审批和 P2 智能推荐。
- 不修改三个数字匹配规则、结构化计价库、经验池、正式知识资料和报告模板。
- 不让大模型决定任务负责人、修改任务状态、生成价格或批准成果。
- 不把第一层 Webhook 改造成收发文件或精准单聊机器人。
- 不放宽现有群聊普通消息静默、@本机器人校验、1 分钟收件窗口、15 分钟延迟补偿和五分钟历史消息拦截。
- 不破坏现有 WeAct 收件、自动处理、任务查询、成果重发、知识库问答和经典完成卡片。
- 不自动向任何未授权群或个人发送消息。

1. `TaskEnvelope` 必填字段和非法事件类型。
2. 非 active Skill 拒绝。
3. 业务幂等键重复不重复建任务、不重复发消息。
4. 待填模板生成任务副本且不覆盖原件。
5. 目标群不是唯一“智算测试”时拒绝真实测试投递。
6. 经典任务卡结构兼容 WeAct，不出现 `schema 2.0` 或 `update_multi=false`。
7. 卡片成功、文件成功的完整投递。
8. 卡片成功、文件失败后的分步状态和只补发失败步骤。
9. 群聊目标人保存和真实 ID 提醒逻辑。
10. 精准单聊 mock 成功、无映射、权限不足和用户不可达。
11. 接口不回显敏感 ID、令牌和文件 Key。
12. 机器人停用、连接失败或平台异常不影响本地专业主流程。

## 九、完成定义
