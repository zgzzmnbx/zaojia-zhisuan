# UI 设计规范

## 文件定位

本文是“造价智算”的可执行 UI 设计规范，服务前端实现和后续 UI 交接。它承接：

```text
00-PRD/03-整体UI设计PRD.md
```

本文不替代具体模块 PRD，不定义价格匹配、预警、报告或大模型业务逻辑。

## 设计系统结构

前端样式建议逐步向三层 token 结构收拢：

```text
基础 token：原始颜色、字号、间距、圆角、边框、阴影
语义 token：主色、背景、文本、边框、成功、警告、危险、信息
组件 token：按钮、表格、面板、弹窗、AI Dock、标签、进度条
```

现有 `frontend/src/styles.css` 已有 `--ink`、`--muted`、`--blue`、`--canvas`、`--line`、`--danger`、`--ok` 等变量。后续不要求一次性重构，但新增样式应优先复用或扩展变量，不继续大量散落硬编码色值。

## 固化后的执行基线

从 `v5.3.3` 起，本文件承担项目级 `DESIGN.md` 的职责，不再另建平行设计文档。后续 UI 设计、前端实现和验收统一按以下基线执行：

- 唯一主界面：`大尾巴主题 / daweiba`。
- 唯一视觉方向：Navattic 式极简 SaaS 工作台。
- 唯一规范入口：`00-PRD/03-整体UI设计PRD.md` + 本文件。
- 唯一参考图来源：`01-assets/01-UI参考图/00-reference/` 中带 `【尾巴选】` 的图和 Navattic 精选图。
- 必须遵守：严禁卡片套娃、常驻面板禁阴影、导航选中态克制、主按钮无边框深蓝、所有按钮 padding 充足。
- 不再参考：候选 A / D 的旧视觉、旧主题、旧宽屏方案、旧流程条、旧演示 demo、营销页式 hero、厚重卡片工作台。
- Claude Code 建议包中的 UI 内容只作为信息架构和局部体验参考，不改变本文件已固化的 token、色彩、字号、圆角、边框、阴影和组件规则。

## 大尾巴主题设计令牌

本节是“大尾巴主题”正式落地的执行口径。新前端实现应优先把这些值抽成 `.shell.layout-daweiba` 或同等作用域下的 CSS 变量，避免散落硬编码。

当前落地状态：前端已固定使用 `大尾巴主题 / daweiba` 作为唯一主界面，已实现左侧模块导航、中间工作区、右侧问问智算随行助手和左右折叠态；后续深化仍应沿用本节 token。

隔离要求：

- 只保留大尾巴主题主界面，不恢复旧布局、旧主题、旧宽屏方案、隐藏旧流程条或旧流程条本身。
- 大尾巴主题样式必须限定在 `.shell.layout-daweiba` 或等价作用域内。
- 新线程实现时，以本节设计令牌、Navattic 左侧菜单 / 表格 / AI 侧栏参考分析为依据。
- 正式色调以大尾巴提供的 Navattic 参考图为准：白色主底、浅灰分区、黑灰文字、轻边框、蓝色重点操作；蓝绿色只保留为少量 AI / 智能感点缀。
- 优先参考 `01-assets/01-UI参考图/00-reference/` 中带 `【尾巴选】` 的设计图片，以及 `Navattic web Mar 2026-精选` 下的 144、42、155、124 等截图。它们的共同标准是：白底、细线、扁平、少色、低阴影、信息密度高。

布局层隔离要求：

- 大尾巴主题主布局必须使用独立 `daweiba-*` 类：`daweiba-layout-shell` 负责左中右三栏，`daweiba-main-content` 负责中间主列，`daweiba-workspace-frame` 负责当前模块工作区，`daweiba-fill-grid` 负责填价工作台上三列，`daweiba-preview-section` 负责表格预览窗口。
- 大尾巴主题外层布局只认 `daweiba-*` 主壳类；不得再新增平行布局壳或主题切换壳。
- 大尾巴主题的显隐、宽度、三栏比例、填价页上下分区、结果预览页铺满规则必须在 `daweiba-*` 布局层内闭环，避免主工作区被挤成窄列或留下大片空白。
- 后续新增模块时，先接入 `daweiba-workspace-frame` 的模块状态，再写模块内部样式。

### Navattic 极简绝对约束

以下规则优先级高于本文其他组件描述。若后文有冲突，以本节为准。

1. **布局绝对法则：严禁卡片套娃。** 废除“灰底白卡再套白卡”的层级结构。大工作区背景必须使用统一纯白或极浅灰；各功能区之间只用 `1px solid var(--db-border)`、`border-right`、`border-bottom` 或表格线做物理分割。禁止在常驻工作区里用带背景色的大卡片容器继续嵌套小卡片。
2. **视觉绝对法则：常驻界面扁平化。** 除 Dialog / Modal、Dropdown、Toast 这类浮层外，系统内所有常驻面板、左侧栏、右侧栏、表格区、工具条、模块工作区一律 `box-shadow: none`。
3. **导航与交互法则：克制选中态。** 左侧导航激活态禁止高饱和大色块，采用文字加粗加深、极浅灰底或透明底，加 `1px` 细线 / 左侧细条指示。
4. **组件视觉法则：按钮精调。** 主按钮无边框，使用纯深蓝底和白字，圆角 `6px-8px`，padding 足够；次按钮透明或白底，加浅灰边框。所有按钮禁止干瘪，也禁止胶囊化过度。
5. **层级法则：用线和留白表达层级。** 优先通过 `border`、`divider`、字号、字重、间距表达结构，不用阴影、厚背景、重圆角堆层级。
6. **模块导航去编号法则。** 左侧模块导航不显示 `01/02/03` 等数字序号；图标、模块名、说明文字和克制选中态承担导航识别，避免编号造成视觉噪声。

### Navattic 字体与文本渲染绝对约束

为实现 Navattic 级别的现代、内敛和克制感，必须严格遵守以下排版与颜色渲染法则。

1. **绝对禁用纯黑法则。**
   - 严禁在全局任何文本、边框或阴影中使用纯黑色 `#000000`，也不使用传统深灰色 `#333333` / `#333` 作为文字或边框。纯黑在白底上对比过硬，会造成视觉疲劳和廉价感。
   - 页面主标题、强数字、关键强调统一使用 Text-Strong：`#0F172A`，字重 `600` 或 `700`。
   - 正文、表格主体、普通表单统一使用 Text-Base：`#334155`，字重 `400`。
   - 辅助说明、次级标签、占位符统一使用 Text-Muted：`#64748B`，字重 `400`。
2. **字体族基建。**
   - 优先保证英文字母和数字的现代感，中文字体作为托底。
   - 全局字体栈固定为：`'Inter', -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei UI', sans-serif`。
3. **抗锯齿渲染。**
   - 必须在全局根节点开启抗锯齿，使字体边缘更纤细锐利。
   - 强制属性：`-webkit-font-smoothing: antialiased;` 和 `-moz-osx-font-smoothing: grayscale;`。
4. **数字等宽对齐。**
   - 表格、金额、行号、统计数字必须使用等宽数字特性，保证上下行数字和小数点垂直对齐。
   - 表格数字列、金额列、统计卡强数字使用：`font-variant-numeric: tabular-nums;`。

### 字体

- 字体族：`'Inter', -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei UI', sans-serif`。
- 不额外引入外部字体文件依赖；`Inter` 仅作为优先匹配字体，未安装时自动回落到系统字体。
- 不使用负字距，不使用随视口宽度缩放的字号。
- 全局根节点必须开启 `-webkit-font-smoothing: antialiased;` 与 `-moz-osx-font-smoothing: grayscale;`。
- 表格数字列、金额列、行号、指标强数字必须开启 `font-variant-numeric: tabular-nums;`。

### 间距

- 基础间距采用 4px 系列：`4 / 8 / 12 / 16 / 20 / 24 / 32`。
- 页面外边距：桌面宽屏 `18px-24px`；普通笔记本 `16px-18px`。
- 主三栏间距：`0px` 或 `8px-12px`；优先用 `1px` 竖向分割线表达栏位边界，不靠大 gutter 做卡片感。
- 面板内边距：紧凑面板 `12px-14px`；普通面板 `16px`；AI 侧栏 `14px-16px`。
- 数据组 / 指标组间距：`10px-12px`，不得再包一层厚背景卡。
- 表格单元格：横向 `12px`，纵向 `9px-10px`；目标行高 `42px-52px`。
- 顶部工具条高度：`46px-54px`。
- 左侧极窄图标栏宽度：`56px-64px`。
- 一级模块栏宽度：`200px-232px`。
- 二级功能栏宽度：`200px-220px`，非必要不常驻。
- 右侧 AI 助手宽度：默认 `380px`，可调范围 `360px-420px`；折叠态 `56px`。

## 基础视觉方向

- 默认风格：浅色专业工作台，克制、干净、可信。
- 主色倾向：深蓝主按钮 + 少量蓝绿 AI 点缀，不继续扩大紫色、咖色、深蓝黑等单一大面积主题。
- 页面质感：无常驻阴影、轻边框、纯白 / 极浅灰背景、明确层级。
- 圆角：工作台组件以 6px-8px 为主；按钮和输入框可使用 8px-12px；避免大块胶囊按钮充满页面。
- 装饰：不使用渐变光球、漂浮背景、过度玻璃拟态和无业务意义的大插画。
- 文案：界面内不堆说明文字，功能靠布局、图标、状态和短标签表达。

## 字体与字号

- 字体：全局使用 `'Inter', -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei UI', sans-serif`，保证英文和数字优先呈现现代 SaaS 感，中文使用系统字体托底。
- 全局开启抗锯齿：`-webkit-font-smoothing: antialiased;`、`-moz-osx-font-smoothing: grayscale;`。
- 表格数字、金额、行号和统计强数字统一开启 `font-variant-numeric: tabular-nums;`。
- 页面大标题：32px-44px，只用于欢迎页和首页首屏。
- 模块标题：22px-28px。
- 面板标题：16px-20px。
- 表格正文：13px-15px，默认建议 14px。
- 辅助说明：12px-13px。
- 按钮文字：13px-15px。
- 不使用负字距；不使用随视口宽度缩放的字体。

## 间距与密度

- 基础间距按 4px 倍数：4 / 8 / 12 / 16 / 24 / 32。
- 页面外边距：普通桌面建议 24px-32px。
- 面板内边距：紧凑面板 16px，普通面板 20px-24px。
- 表格单元格：纵向 8px-12px，横向 12px-16px。
- 设置按钮和工具按钮要低高度，避免纵向过厚；常规高度建议 32px-36px。
- 表格工具条、sheet 标签、筛选设置应紧凑，不能把首屏表格挤到下方。

## 组件规范

### 按钮

| 类型 | 高度 | 用途 |
| --- | --- | --- |
| 主按钮 | 36px-40px | 上传后生成、开始抓取、主要提交 |
| 次按钮 | 34px-38px | 选择文件、保存设置、刷新 |
| 工具按钮 | 30px-34px | 设置、复制、展开、收起、行级 AI |
| 图标按钮 | 30px-34px | 纯图标工具，不额外加文字 |

要求：

- 主按钮参考尾巴选蓝色按钮：纯深蓝底、白字、无边框、圆角 `6px-8px`、横向 padding 不低于 `14px`；hover 只加深蓝色，不加阴影。
- 次按钮透明或白底、深蓝 / 深灰文字、浅灰边框；禁止把次按钮做成大块浅蓝卡片。
- 按钮 padding 必须充足，文字与边缘至少保留 `14px` 横向距离；禁止干瘪、压缩、文字贴边。
- 有明确图标的工具优先使用图标。
- 原来只有图标的入口保持纯图标。
- 原来有文字的设置入口统一为齿轮图标 + “设置”。
- 禁止出现过高、过宽、像宣传按钮的大型设置按钮。
- 禁止在一个模块里同时出现多套设置按钮样式。

### 面板与分割

- 页面分区可以是全宽区域或工作台面板，严禁“卡片套卡片”。
- 常驻面板圆角建议 `8px`，边框 `1px solid var(--db-border)`，`box-shadow: none`。
- 大工作区背景必须统一为 `var(--db-bg)` 或 `var(--db-app-bg)`，不得出现灰底白卡、白卡再套白卡的厚层级。
- 常驻模块内部优先使用 `border-top`、`border-right`、`border-bottom`、`divider` 和留白分组，不再给每个段落套独立背景卡。
- 卡片只允许用于真正重复对象列表、弹窗内容块、少量统计指标；不得用于把页面每个功能块都包起来。
- 表格预览窗口可以保留“窗口感”，但内容区域必须足够宽，不做装饰性缩窄。

### 表格

- 表格是主视觉中心，默认占据主区域最大宽度。
- 表头固定或保持可见时，应有清晰底色和分割线。
- 数字列右对齐，文本列左对齐，状态列居中或用标签。
- 数值最多显示 2 位小数。
- 长文本按列宽自动换行，最多显示 2 行，悬浮看完整内容。
- AI 操作列固定在右侧可见位置，不要求用户横向滚到最后。
- 横向滚动条可见但不突兀，不能遮挡表格内容。

### 大尾巴主题表格工具条

- 表格样式可参考 Navattic 表格工作台：轻边框、浅表头、清晰筛选条、筛选 chip、右侧导出 / 排序动作。
- 顶部工具条建议分为：左侧当前 sheet 和行数，中间搜索 / 筛选 / 字段设置，右侧排序 / 导出 / 缩放 / AI 批量复核。
- 筛选条件以 chip 展示，支持关闭和追加，例如 `只看待复核`、`表2`、`技术系数为空`、`隐藏空行已开`。
- 行高建议 42px-52px，既比传统后台舒展，也比 Navattic 更适合造价宽表。
- 第一列可承载主字段 + 次字段；数值列右对齐；状态列用语义色标签。
- AI 操作列必须固定在右侧可见位置。
- 表格底部可保留每页行数、分页、隐藏行计数和匹配摘要，但不要挤压主表格高度。

### Sheet 标签

- 当前 sheet 用主色实底或浅底强调。
- 非当前 sheet 用浅边框 / 浅底。
- 标签高度控制在 32px-36px。
- 标签过多时横向滚动，不能换行挤压表格。

### 设置弹窗

- 弹窗最大高度不超过视口，内部滚动。
- 顶部为标题和关闭按钮，中部为分组表单，底部为保存 / 取消。
- 设置项分组使用短标题，不写长段解释。
- 开关用 switch 或 checkbox，下拉用 select，数值用 input / stepper，不用长文本按钮伪装输入。

### AI Dock

- 默认宽度 360px 左右，可在设置中调整。
- 收起态只保留窄条入口，不遮挡表格。
- 等待态必须显示阶段：检索依据、整理证据、生成回答。
- 知识库模式需要有明确状态标签，例如“已调用知识库”。
- 行级复核按钮只填入问题或展开助手，由用户手动发送。
- AI Dock 视觉上是“随行复核员”，不能比表格更抢主视觉。

### 大尾巴主题 AI 助手侧栏

- AI 助手可参考 Navattic 右侧抽屉 / 客服面板：独立侧栏、白底、1px 左边框、顶部身份区、固定底部输入框；常驻状态禁止阴影。
- 右侧助手宽度建议 360px-420px；折叠态建议 52px-64px，只保留 AI 图标、风险数量和展开按钮。
- 顶部区域应包含：问问智算标识、当前模式、知识库状态、折叠 / 展开 / 关闭。
- 上下文区应展示当前 sheet、当前行、匹配状态、候选数量、是否调用知识库。
- 快捷操作区应包括：解释当前行、核查数字、生成风险提示、查库、生成报告说明。
- 消息区区分普通问答、知识库问答、行级复核和风险报告，不同模式有明显标签。
- 证据区展示知识库来源、规则命中、经验提示和待复核原因。
- 输入框固定底部，支持快捷 chip 和 `查库：` 前缀。
- AI 助手不能只是客服聊天，也不能覆盖主表格；它服务当前表格和当前业务证据。

### 状态标签

- 标签高度 22px-28px。
- 文字短，不写长句。
- 颜色服务业务状态，不做装饰。
- 同一状态全项目统一颜色和命名。

## 页面级规范

### 首页 / 欢迎页

- 第一屏明确出现“造价智算 / 工程造价辅助智能体”。
- 可以做参赛展示感，但不能让用户找不到进入系统、上传 Excel 和核心流程。
- 展示重点是规则链、数据链、智能体链，不是泛泛 AI 科技感。

### 主工作台

- 大尾巴主题是当前唯一主工作台：左侧为模块导航，中间为程序主体，右侧为问问智算 AI 随行助手。
- 左侧模块导航和右侧 AI 助手都应支持折叠；折叠后中间主体优先获得宽度。
- 中间主体默认优先展示当前模块的核心工作对象，例如表格预览、预警列表、工作量抓取结果或报告状态。
- 宽屏或左侧收起时，优先把空间让给表格。

主页按宽屏桌面软件设计，不按竖屏网页优先设计。后续实现 `daweiba` 时，应优先保证 1366px 以上和普通宽屏桌面体验；竖屏 / 移动端只需不阻断基本使用，不再作为主设计目标。

不得为某一个固定分辨率写死布局，不允许靠 1366、1440、1920 等单点像素值手工拼出“只在某个窗口好看”的界面。大尾巴主题必须采用桌面优先的响应式工作台：外层三栏使用 flex / grid / `minmax()` / `clamp()` / 容器约束组织，左侧导航保持稳定宽度和可折叠能力，右侧智算侧栏保持可调宽度和折叠能力，中间工作区按剩余空间自适应填满。窗口宽度变化时，优先保证中间表格和当前模块主体可用，再按断点收敛左侧二级栏和右侧助手。

#### 三栏关系

- 左侧“当前模块”窗口、中间模块工作区、右侧“随行助手 / 智算”窗口必须上下顶格，形成软件工作台。
- 左侧和右侧收起时，中间工作区必须动态扩宽。
- 中间工作区不再纵向堆叠所有模块，而是根据左侧当前模块切换显示。
- 页面不应出现大面积营销式标题、宣传说明或首屏 Hero；主界面进入后直接是工作台。
- 宽屏比例按“左侧固定、右侧可调、中间吃满剩余空间”计算；中间模块内部不允许再使用旧的小卡片宽度造成大片空白。
- 填价工作台宽屏下采用“上传标准 Excel / 转换后概览”上层两列、“工作状态”下层横跨整行的结构；填价工作台不再显示表格预览。
- 工作状态模块承接原表格预览底部摘要和宽屏洞察：输入 / 匹配行数、价格列、输出文件、实物 / 技术一层命中、经验提示、复核数量、预警数量、高风险 / 低风险，以及“匹配状态”“匹配质量分布”两张状态卡。
- 结果预览页的表格预览模块必须作为单个主窗口铺满中间区域；除必要工具条和 sheet 标签外，不再混入转换摘要、经验池预警按钮、预警详情和导入经验池内容。报告预览、知识库状态页也应作为单个主窗口铺满中间区域。
- 转换概览主卡使用白底、浅灰或浅蓝信息态；不使用绿色整块背景。绿色仅用于“标准命中 / 无预警”等明确业务状态的标签、小点、图表分段，不用于普通完成概览卡。

模块切换要求：

- 当前模块在左侧高亮，中间只呈现该模块主体内容。
- 填价工作台完成转换后，可通过左侧模块进入结果预览，不应把预览强行挤在填价模块右侧或下方。

- 经验池预警模块要承接原表格预览中的“运行经验池预警分析”按钮、设置按钮、运行进度、未执行提示、预警摘要、风险明细和定位预览入口。

- 报告生成和预览模块应独立，不与结果表格共用同一块主视觉；真实 DOCX 预览应占据主体空间，不再保留大比例摘要卡挤压文档。

- 智能协同固定排列在知识库问答下方；它只展示任务与组织协同信息，逐行 Excel 编辑、候选确认和复杂风险复核继续进入结果预览或对应专业模块。

#### 智算聊天角色徽标

- 聊天流中助手角色徽标显示为 `Z`，用户角色徽标显示为 `U`。
- 角色徽标必须文字水平 / 垂直居中，使用胶囊或圆角徽标，不再使用单个中文“智 / 我”。
- `Z` 徽标使用与环形图一致的绿色底，`U` 徽标使用蓝色底；徽标底色必须与旁边的灰色 / 蓝色气泡拉开差异。

#### 左侧状态仪表盘

左侧“当前模块”窗口底部固定为状态仪表盘区域，服务全局处理状态，不随模块切换消失。

结构要求：

- 顶部为环形图，展示当前表格匹配状态。
- 环形图可按状态分段：结构匹配、待复核、预警、未运行 / 无数据。
- 环形图中心可显示总体进度或当前状态，例如 `101`、`100%`、`未运行`。
- 环形图使用 Dashboard donut chart 结构，不使用普通饼图：中心为总数 + `Total`，外侧至少保留 2-3 个轻量标注，标注格式为 `数值 (百分比)`，通过浅灰引线连接到图环。
- 下方展示紧凑数据行：输入行数、可视化行数、待复核、预警。
- 预警未运行时明确显示“未运行”，不要用 `0` 伪装为无风险。
- 无数据时保留空态结构，避免上传、转换后侧栏高度跳动。

视觉要求：

- 状态仪表盘使用与左侧菜单一致的白底、浅灰分区、轻边框和小字号。
- 环形图尺寸控制在左侧栏宽度内，不挤压模块列表；推荐 SVG 画布宽度 `190px-210px`，实际圆环直径 `88px-112px`。
- 数据行使用左右对齐：左侧为数字或状态，右侧为标签。
- 状态颜色沿用业务语义：标准命中绿、待复核 / 高风险红、经验 / 低风险琥珀、知识库 / 处理中蓝、未运行灰。
- 左侧收起时，仪表盘收敛为窄栏小圆环或状态点，只保留最关键数字 / 状态。

#### 右侧智算侧栏

右侧“随行助手 / 智算”必须对标左侧“当前模块”窗口，而不是沿用旧聊天窗口。

结构要求：

- 顶部身份区：包含收起 / 展开动作、智算图标、名称、当前模式和设置入口。
- 主体区：根据当前模块显示上下文、快捷操作、消息和证据。
- 底部输入区：固定在底部，消息区独立滚动。
- 折叠态：只保留窄条入口，不遮挡中间工作区。
- 宽度：默认沿用当前智算窗口宽度设置，并继续允许调整。

视觉要求：

- 右侧智算侧栏和左侧当前模块窗口使用同一套白底、浅灰分区、1px 轻边框、低圆角、无阴影、图标按钮和分割线语言。
- 参考 Navattic Support / Settings 右侧面板：顶部身份区清楚、主体留白克制、底部输入或操作固定。
- 不使用老式气泡聊天面板，不把聊天窗口再包一层厚重卡片。
- 聊天气泡参考 Pipedrive Live Chat：会话区白底，消息按左右对齐形成聊天流；AI 消息为左侧浅灰气泡并配圆形头像，用户消息为右侧蓝色气泡并配圆形头像；气泡圆角要轻巧，不做大面积蓝色提示卡。
- 每条消息不再重复显示大段角色标题；角色信息通过头像、颜色和顶部会话栏表达。
- AI 消息、知识库证据、行级复核、风险报告、快捷指令应作为同一侧栏里的分组能力，而不是分散成多个互相抢空间的窗口。

### 大尾巴主题深化规范

- 当前主界面已经进入落地阶段，不再维护候选 A / D 或其他过渡方案。
- 后续深化只吸收已确认参考资料中的结构优点：左侧模块导航、表格工作台、右侧 AI 随行助手。
- 正式色调继续按 Navattic 式白底、浅灰分区、黑灰文字和蓝色重点操作执行。
- 状态色统一：标准命中绿、经验提示黄、待复核红、知识库 / AI 检索蓝。
- 主界面不能变成营销页；进入后直接是工作区，不用大篇幅介绍产品。

### 大尾巴主题左侧菜单

- 左侧菜单可参考 Navattic 的“极窄图标栏 + 一级模块栏 + 二级功能栏”结构，但必须按造价智算业务重组。
- 极窄图标栏宽度建议控制在 52px-64px，主要用于产品标识、全局搜索 / 动作、模块总入口、设置和用户。
- 一级模块栏宽度建议控制在 190px-240px，用于填价工作台、结果预览、经验池预警、工作量抓取、Word 报告、知识库问答、智能协同等模块；智能协同固定排在知识库问答下方。
- 二级功能栏不是常驻必需项，只在当前模块存在复杂子设置时出现；出现时应允许折叠，避免挤压中间表格。
- 当前选中项采用深色加粗文字 + 极浅灰底或透明底 + 1px 细线 / 左侧细条指示；禁止高饱和大色块，禁止整块蓝色、绿色、紫色填充。
- 一级模块栏不显示数字序号；不得为了表现流程感在模块名前加 `01/02/03`，避免和真实工作步骤、表格行号混淆。
- 普通笔记本宽度下，左侧菜单应自动收敛为图标栏 + 抽屉，优先保证中间表格和右侧 AI 可用。

### 填价结果预览

- 表格独占预览模块主体宽度。
- 状态、候选数量、匹配说明、预警参数、预警细节不能与数值列混乱。
- 匹配说明默认短显示，悬浮看完整内容。
- 重点数字和状态要可扫读，不能全靠颜色。
- 顶部工具条内的“批量匹配”“重算公式”“设置”等按钮使用低高度工具按钮，避免挤压表头空间；完成后的“已完成匹配”保留为浅色禁用态，不能突然消失，也不能继续保持高权重主按钮。
- 表头列宽允许像 Excel 一样在右边缘拖拽调整；不显示额外手柄或常驻提示，鼠标悬停边缘时变为可拖拽光标即可。列宽偏好按 `sheet 名 + 列标题` 保存，只影响浏览器预览，不反写输出 Excel。

### Word 报告预览

- 沿用左侧“Word 报告”，不得新增“报告预览”一级菜单。
- 页面采用“紧凑顶部状态 / 操作 + 主体真实 DOCX 预览”结构；下载 Word、下载 Excel、风险清单和刷新预览放在同一操作层级。
- 主体必须读取当前任务实际生成的 `.docx`，不能继续用报告摘要、输出文件名和指标字段拼成模拟 Word 页面。
- 预览器与 Word renderer 只在当前页有可用报告时按需加载，Worker、JSZip 和字体等资产必须同源自托管，断网可用。
- 预览容器必须有稳定高度并独立滚动；使用响应式剩余空间，不写死只适配单一窗口的宽高。
- 加载、成功、报告已更新、失败和未生成状态要清晰；失败时必须提供重试和下载 Word 兜底，不能让整个页面白屏。
- 人工修改后未“重算公式”时沿用现有报告未刷新提示；重算、风险报告写回或 job 切换后，旧预览必须失效。
- 搜索、缩放、打印等工具只在 Word renderer 实际支持时展示；网页预览定位为快速核对，正式排版以下载后的 Word 文件为准。
- 第三方组件外壳需融入大尾巴主题：白底、浅灰工作台、1px 细边框、低圆角、无常驻阴影；不得覆盖正文或抢过造价智算主操作。

### 问问智算

- 普通问答、知识库问答、快捷指令、风险报告要有不同状态提示。
- “查库：”等强制知识库模式应显示已调用知识库。
- 无证据时明确提示人工复核，不做看似确定的回答。

## 禁止项

- 不把工作台做成营销 landing page。
- 不用大面积渐变、光球、装饰图形遮盖主流程。
- 不为了视觉冲击牺牲表格宽度。
- 不让 AI 聊天窗口压过主表格。
- 不让设置按钮变成高大胶囊按钮。
- 不在同一模块里混用多套字号、按钮、图标风格。
- 不用颜色表达相反含义。
- 不通过 UI 改动改变后端匹配、预警、报告或下载结果。

## 前端验证口径

前端线程做 UI 改动后，至少验证：

- `npm run frontend:build` 通过。
- 普通桌面宽度下，上传、转换、表格预览、AI Dock 不遮挡。
- 宽屏窗口下，表格预览获得更多空间。
- 1366px 宽笔记本窗口下，设置按钮、sheet 标签、表格内容不溢出。
- 不按固定分辨率写死布局；至少在普通笔记本宽度、宽屏显示器宽度和 Tauri / 浏览器窗口缩放三类场景下，左侧导航、中间工作区、右侧智算侧栏能自然伸缩，不出现大片空白、横向溢出或主工作区被压扁。
- AI Dock 收起 / 展开正常。
- 预览表格横向和纵向滚动正常。
- 状态色在默认主题下语义正确。
- 如果改了主题，还要确认状态色不被主题反转。
