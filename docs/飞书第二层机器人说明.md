# 飞书第二层机器人说明

## 能力范围

第二层使用飞书企业自建应用机器人和本机长连接。用户在任意已添加机器人的群聊中 `@机器人`，并在同一条消息上传一个 `.xlsx` 后，系统按全局单任务队列自动完成：

1. 文件下载与格式校验；
2. 使用项目默认设置检查工作表、表头和列映射；
3. 价格与两个调整系数批量匹配；
4. 经验池预警和结构化风险汇总；
5. 可选 DeepSeek 风险说明；
6. Excel、Word 成果上传并返回原群。

无法可靠识别字段时任务会转为 `needs_manual`，不会猜测映射或强行写入价格。DeepSeek 不可用时只降级风险说明，不影响结构化风险、Excel 和 Word。

## 飞书开放平台配置

1. 创建企业自建应用并启用机器人能力。
2. 在“事件与回调”中选择“使用长连接接收事件”。
3. 订阅 `im.message.receive_v1`。
4. 按控制台提示申请群聊 @ 消息、发送消息、读取消息资源、上传/发送文件的最小权限。
5. 发布应用，并把机器人添加到需要使用的群聊。

权限名称可能随飞书开放平台调整，应以当前控制台对接口给出的权限提示为准。机器人无需公网域名、回调地址或内网穿透，但本机必须能访问飞书开放平台。

## 本机秘密配置

应用凭证保存在：

```text
Codex-Temp/runtime/feishu-app-settings.json
```

格式：

```json
{
  "app_id": "本机 App ID",
  "app_secret": "本机 App Secret"
}
```

该目录已被 Git 忽略，不进入代码仓库和绿色版。不要把凭证写入 `config/project-default-settings.json`、前端、日志或截图。

## 默认设置

非秘密默认值位于 `config/project-default-settings.json` 的 `feishuAppBot`：

- `enabled`：是否启用第二层；
- `concurrency`：固定为 `1`；
- `retentionDays`：成果留存天数，默认 `30`；
- `allowedExtensions`：默认仅 `.xlsx`；
- `maxFileSizeMb`：默认 `50` MB；
- `apiBaseUrl`：本机 FastAPI 地址；
- `enableLlmRiskNarrative`：是否尝试生成大模型风险说明；
- `retryCount`：临时故障有限重试次数。

## 启动顺序

1. 先启动造价智算，确认 `http://127.0.0.1:8000/api/health` 正常。
2. 双击 `启动飞书第二层机器人.bat`。
3. 看到“第二层飞书机器人长连接已启动”后，在测试群发送有效消息。

绿色版随包携带机器人程序和 SDK 依赖，但不携带本机凭证。目标电脑需要先建立自己的秘密配置，再单独启动机器人。

## 任务与恢复

- SQLite 队列：`Codex-Temp/runtime/feishu-bot/tasks.sqlite3`。
- 任务目录：`Codex-Temp/runtime/feishu-bot/tasks/<task_id>/`。
- 状态流：`queued → downloading → inspecting → matching → risk → report → uploading → completed`。
- 异常状态：`needs_manual`、`retryable_failed`、`failed`。
- 重复事件由事件 ID、消息 ID 和文件 key 唯一约束拦截。
- 进程重启后，中断任务转为可重试状态并继续排队。
- 结束任务文件默认保留 30 天；未结束任务不会因到期被直接删除。

## 常见问题

- 未收到消息：检查应用是否已发布、机器人是否进群、事件订阅是否为长连接、权限是否已生效。
- 提示未配置：检查本机秘密文件位置和 JSON 格式，不要在前端寻找凭证回显。
- 一直重试：先确认 FastAPI 健康接口和本机网络，再查看智能协同页的脱敏任务状态。
- 文件被拒绝：同一条消息必须包含 `@机器人` 和恰好一个 `.xlsx`；不支持 `.xls`、多文件或无文件消息。
- 映射需人工处理：使用造价智算本地专业工作台检查表头和列映射，系统不会自动猜测。
