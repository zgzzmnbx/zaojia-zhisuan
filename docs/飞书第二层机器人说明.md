# 飞书第二层机器人说明

## 能力范围

第二层使用飞书企业自建应用机器人和本机长连接，普通飞书与企业 WeAct 共用同一套消息分流逻辑。群聊中 `@机器人` 后发送 `@上传` 或 `@上传文件`，机器人单聊中直接发送同一口令；收到提示后在 1 分钟内发送一个 `.xlsx`。只有这两个完整口令会开启收件窗口，其他相似说法和直接发送文件均不会建任务。系统按全局单任务队列自动完成：

1. 文件下载与格式校验；
2. 使用项目默认设置检查工作表、表头和列映射；
3. 价格与两个调整系数批量匹配；
4. 经验池预警和结构化风险汇总；
5. 可选 DeepSeek 风险说明；
6. Excel、Word 成果上传并返回原群。

文字消息按以下顺序分流：`@上传` / `@上传文件` 开启 1 分钟收件窗口；`@知识库：问题` 调用造价智算现有 `/api/knowledge/ask` 并返回依据解释和来源；`你好` 返回机器人自我介绍、五类功能、群聊 / 单聊用法和专业边界；群聊完整输入“群里有几个人”“群成员”或“都有谁”时，通过 `/open-apis/im/v1/chats/{chat_id}/members` 分页返回真实人数和姓名；第一梯队任务指令读取本地任务队列；其余文字调用与网页普通问答相同的 `/api/llm-chat` 作为大模型托底。群聊消息必须明确 `@机器人`，单聊无需 @；但群成员查询仅在目标群聊生效，单聊会提示使用边界。程序兼容普通飞书和 WeAct 的编号化 @ 占位符。知识库问答只读检索，不修改 Excel、价格、系数或正式知识库；普通大模型回答同样不参与最终价格或系数裁决。

### 第一梯队确定性指令

- `任务` / `最近任务`：显示当前群或当前单聊最近 5 个任务；
- `进度 FS-任务编号`：显示文件、当前阶段、排队位置、错误说明和更新时间；
- `风险 FS-任务编号`：显示已形成的结构化风险总数和高风险数；
- `高风险`：显示当前会话最近 5 个有高风险记录的任务；
- `结果 FS-任务编号`：重新发送已完成任务原有的 Excel、Word 和经典完成卡片；
- `帮助` / `指令`：显示上传、知识库、成员和任务指令说明。

这些指令要求完整匹配，不经过大模型。所有列表和详情必须同时匹配当前消息 `chat_id`；即使用户知道其他群或其他单聊的任务编号，也只会收到“当前会话未找到”。“结果”不重新执行匹配或风险识别，不修改原任务状态；任务未完成、成果文件已清理或上传失败时返回明确说明。群聊仍须先 @当前机器人，单聊可直接发送。

普通问答和知识库解释共用后端大模型客户端。TLS 提前断开、超时或连接中断时最多尝试 3 次，短暂网络抖动恢复后自动继续；HTTP 业务错误、鉴权错误和参数错误不重试，并返回可读原因。

第二层启动时先读取本机器人真实 Open ID。只有群聊消息 mentions 中明确包含该 Open ID，或消息来自机器人单聊，才会在原消息下方异步添加“了解”（`Get`）表情并进入文字分流。群内普通聊天、@其他人 / 其他机器人、以及收件窗口内未再次 @ 的独立文件消息都不添加表情；身份无法确认时群聊保持静默。表情回应失败时只在机器人运行控制台记录警告，不会阻断问答、收件或任务处理。

所有入站消息在表情和业务分流之前先经过三层防护：① SQLite 同时持久事件 ID 和消息 ID，任一标识重复即静默拦截；② 能解析到平台 `create_time` 时，与本机接收时间相差超过 5 分钟的历史消息不添加表情、不回复、不建任务；③ 控制台同时记录消息 ID、平台创建时间和本机接收时间，可区分实时消息、重放与平台延迟。旧版事件表启动时自动增量迁移；平台没有提供或无法解析 `create_time` 时不会因此拒绝消息，以保持普通飞书、WeAct 和历史事件兼容。

处理中会按“表格与字段识别完成”“批量匹配完成”“风险识别完成”三个关键节点发送简短进度，阶段消息发送失败不会改变专业任务结果。

任务完成时先上传 Excel 和 Word 成果文件，再发送绿色只读完成卡片。卡片展示任务编号、输入文件、结构化风险总数和高风险数；第一层 Webhook 设置中的“进入造价智算 URL”已配置时，卡片同时显示跳转按钮。按钮只打开网页，不审批、不复核、不改值。第二层使用兼容 WeAct 旧客户端的经典卡片格式，不使用会显示“请升级客户端”的 `schema 2.0`；卡片发送失败自动降级为文字完成通知，不影响已经上传的成果文件。

无法可靠识别字段时任务会转为 `needs_manual`，不会猜测映射或强行写入价格。DeepSeek 不可用时只降级风险说明，不影响结构化风险、Excel 和 Word。

## 飞书开放平台配置

1. 创建企业自建应用并启用机器人能力。
2. 在“事件与回调”中选择“使用长连接接收事件”。
3. 订阅 `im.message.receive_v1`。
4. 申请“接收群聊中 @ 机器人消息”“读取用户发给机器人的单聊消息”“获取群组中所有消息”“获取群组信息”“获取群成员信息”“以应用身份发送消息”“获取与上传图片或文件资源”以及“发送、删除消息表情回复”（`im:message.reactions:write_only`）等权限。群组全部消息权限用于接收飞书客户端自动拆分的独立文件消息；群信息与群成员权限用于三条确定性成员查询；程序会忽略没有对应收件窗口的普通群消息。
5. 发布应用，并把机器人添加到需要使用的群聊。

权限名称可能随飞书开放平台调整，应以当前控制台对接口给出的权限提示为准。机器人无需公网域名、回调地址或内网穿透，但本机必须能访问飞书开放平台。

## 统一机器人秘密配置

第一层 Webhook 和第二层企业应用凭证统一保存在：

```text
Codex-Temp/runtime/feishu-robot-settings.json
```

页面会分别显示第一层 Webhook 和第二层企业应用的可选配置；完整地址和秘密仍只保存在运行目录。统一格式：

```json
{
  "version": 1,
  "app_bot": {
    "active_profile": "default",
    "profiles": {
      "default": {
        "label": "默认机器人（普通飞书）",
        "app_id": "企业应用 App ID",
        "app_secret": "企业应用 App Secret",
        "domain": "https://open.feishu.cn"
      },
      "weact_cost": {
        "label": "Weact机器人（管网内网）",
        "app_id": "企业 WeAct App ID",
        "app_secret": "企业 WeAct App Secret",
        "domain": "https://open.weact.pipechina.com.cn"
      }
    }
  },
  "webhook": {
    "enabled": true,
    "active_profile": "default",
    "profiles": {
      "default": {
        "label": "现有 Webhook（默认）",
        "webhook_url": "群自定义机器人 Webhook 地址",
        "secret": "可选签名密钥"
      },
      "weact_webhook": {
        "label": "weact机器人（webhook）",
        "webhook_url": "内部飞书群自定义机器人 Webhook 地址",
        "secret": ""
      }
    }
  }
}
```

旧版 `feishu-app-settings.json` 和 `feishu-webhook-settings.json` 仍可自动迁移到统一文件。智能协同页切换第二层配置时会先安全退出旧长连接，再启动新配置；切换第一层配置只改变通知出口，不改变通知规则和发送历史。SQLite 队列、任务记录和成果目录不变。同一时刻仍只能启用一个第二层机器人实例。该目录已被 Git 忽略，不进入代码仓库和绿色版。App Secret、访问令牌等秘密不得写入 `config/project-default-settings.json`、前端、日志或截图；非秘密的预期 App ID 和域名则统一登记在项目默认配置的 `feishuAppBot.expectedProfiles`，用于阻止部署误配。

当前部署默认口径：本机使用 `default`（`默认机器人（普通飞书）`），云端使用 `weact_cost`（`Weact机器人（管网内网）`）。这只是各环境当前默认值，用户仍可在“智能协同”页通过“启用接收”开关关闭服务，或通过机器人下拉框切换配置。

企业 WeAct 机器人必须同时给 REST API 客户端和 WebSocket 客户端配置 `https://open.weact.pipechina.com.cn`；只修改其中一个仍会连接错误。成功日志应出现 `connected to wss://lark-frontier.weact.pipechina.com.cn/ws/v2...`。公网飞书配置省略 `domain` 时默认使用 `https://open.feishu.cn`。

企业 WeAct 第二层正式试点当前使用应用 A（App ID：`cli_a961a5cb3ab8d353`）。该应用已经完成群聊 `@机器人`、单聊、`im.message.receive_v1` 事件接收和消息回复的真实端到端验证。历史候选应用 B（App ID：`cli_a961b94ee4b8d353`）只验证过 WebSocket 建连，不应作为已验证的正式消息处理配置。两个应用的 App Secret 均不得写入本文档或 Git，只能保存在本机 / 服务器各自的统一运行秘密配置中。

启动保护采用三道同源校验：智能协同页开关请求由 FastAPI 先校验，后端拉起子进程前再次校验，`feishu_bot_runner.py` 作为 systemd 或手工独立运行时第三次校验。任一配置的 App ID 或域名与 `expectedProfiles` 不一致，状态接口返回“配置不一致”，运行器拒绝建立长连接并记录不含秘密的错误。更换正式机器人时必须同时更新项目登记、测试和两端运行秘密，不能只在某一台机器替换 App ID。

## 机器人运行控制台

- “智能协同”设置页提供“第二层：机器人运行控制台”，默认每 3 秒刷新，也可暂停实时刷新或手动刷新。
- 控制台合并三类只读信息：SDK 长连接建立 / 断开 / 重连状态、机器人收到并处理的文件 / 知识库消息、SQLite 队列中的任务阶段。
- 结构化运行事件保存在 `Codex-Temp/runtime/feishu-bot/console-events.jsonl`，达到 2 MB 后轮转为 `.jsonl.1`；任务阶段仍以 `tasks.sqlite3` 为准。
- “刷新”旁的“机器人运行控制台”按钮用于打开独立弹窗；前端接口 `/api/collaboration/feishu-app-bot/logs` 最多返回 500 条结构化事件。原始 `runner.out.log` / `runner.err.log` 不直接下发，只提取时间、连接状态和节点域名。
- 对机器人实际处理、判定过期或命中重复的消息，控制台显示发送人名称与 ID、群名与会话 ID、消息 ID、平台创建时间、本机接收时间、消息正文和附件名。发送人名称通过“获取单个用户信息”接口、群名通过“获取群信息”接口尽力补齐并在进程内缓存；应用没有通讯录或群信息权限时回退显示原始 ID，不影响收件和任务处理。
- 飞书 / WeAct 长连接事件不包含发送者来源 IP，因此控制台固定说明“飞书长连接事件未提供”，不会生成或猜测 IP。
- App Secret、访问令牌、连接票据、access key、文件 key 和完整 WebSocket URL 均不得进入控制台。

## 默认设置

非秘密默认值位于 `config/project-default-settings.json` 的 `feishuAppBot`：

- `enabled`：是否启用第二层；
- `concurrency`：固定为 `1`；
- `retentionDays`：成果留存天数，默认 `30`；
- `allowedExtensions`：默认仅 `.xlsx`；
- `maxFileSizeMb`：默认 `50` MB；
- `apiBaseUrl`：本机 FastAPI 地址；
- `expectedProfiles`：普通飞书与企业 WeAct 已验证的 App ID / 域名登记；本机和云端使用同一登记，偏差时拒绝启动；
- `enableLlmRiskNarrative`：是否尝试生成大模型风险说明；
- `retryCount`：临时故障有限重试次数。

## 本机与服务器切换

- 智能协同页面分别提供“启用通知”（第一层 Webhook）和“启用接收”（第二层长连接）开关，修改后立即保存到本机运行配置。
- 第二层关闭后，本机长连接进程会自动退出；再次开启会由后端自动拉起，无需手动双击机器人启动脚本。页面最多等待 10 秒并刷新为“运行中”，失败时显示“启用但未运行”。
- 飞书长连接是集群模式而不是广播模式。同一个应用凭证在服务器和本地同时在线时，一条事件只会随机交给其中一个实例，不会稳定地同时发送两份；但任务会随机落到两个环境，队列、文件和结果因此可能分裂。
- 迁移到服务器时，应先确认服务器实例运行正常，再关闭本地“启用接收”；不要把两个长连接实例同时在线当作主备方案。第一层 Webhook 是否保留由通知需要决定，可独立关闭。

## 启动顺序

1. 双击 `启动管勘智算-【codex】.bat`。启动器会先启动或复用后端、前端，再在本机凭证完整且 `feishuAppBot.enabled=true` 时自动启动第二层机器人。
2. 重复双击不会重复启动机器人；启动器状态检查会显示“已启动 / 已配置但未启动 / 未启用或未配置”。
3. `启动飞书第二层机器人.bat` 继续保留为仅重启机器人时的独立入口。
4. 看到“第二层飞书机器人长连接已启动”后，只在“智算测试”群或机器人单聊发送测试消息；禁止向“[管勘智算]勘察测量投标限价智能体”群发送测试内容。

绿色版随包携带机器人程序和 SDK 依赖，但不携带本机凭证。目标电脑需要先建立自己的秘密配置，再单独启动机器人。

## 任务与恢复

- SQLite 队列：`Codex-Temp/runtime/feishu-bot/tasks.sqlite3`。
- 任务目录：`Codex-Temp/runtime/feishu-bot/tasks/<task_id>/`。
- 状态流：`queued → downloading → inspecting → matching → risk → report → uploading → completed`。
- 异常状态：`needs_manual`、`retryable_failed`、`failed`。
- 所有消息先由持久的事件 ID + 消息 ID 唯一约束拦截重放；文件任务再叠加文件 key 约束，不重复建任务。
- 进程重启后，中断任务转为可重试状态并继续排队。
- 结束任务文件默认保留 30 天；未结束任务不会因到期被直接删除。

## 常见问题

- 未收到消息：检查应用是否已发布、机器人是否进群、事件订阅是否为长连接、权限是否已生效。
- 提示未配置：检查本机秘密文件位置和 JSON 格式，不要在前端寻找凭证回显。
- 一直重试：先确认 FastAPI 健康接口和本机网络，再查看智能协同页的脱敏任务状态。
- 群聊文件未响应：先发送 `@机器人 @上传` 或 `@机器人 @上传文件`，收到“请在 1 分钟内发送”提示后再发送一个 `.xlsx`；确认已开通“获取群组中所有消息”权限并重新发布应用。
- 群聊知识库未响应：确认消息中同时包含 `@机器人` 和 `@知识库：问题`；单聊只需输入 `@知识库：问题`。不要在知识库问题消息中同时上传文件。
- 群成员查询未响应：在目标群中明确 `@机器人`，并完整发送“群里有几个人”“群成员”或“都有谁”；检查应用是否已获得读取群信息和群成员权限并重新发布。三条指令不经过大模型，单聊不会代查其他群。
- 单聊文件未响应：先发送 `@上传` 或 `@上传文件`，再在 1 分钟内发送一个 `.xlsx`；确认已开通“读取用户发给机器人的单聊消息”权限并重新发布应用。
- 文件被拒绝：当前只支持单个 `.xlsx`，不支持 `.xls` 或多文件任务。
- 映射需人工处理：使用造价智算本地专业工作台检查表头和列映射，系统不会自动猜测。
