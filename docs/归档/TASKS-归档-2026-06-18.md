# 管勘智算 TASKS.md

> 用途：用于多线程协作、任务拆分、状态流转和线程交接。
> 原则：TASKS 是活跃任务看板，不是历史档案；项目现状摘要写在 README，版本变化写在 CHANGELOG。

## 1. 任务状态说明

- `todo`：待处理。
- `doing`：正在处理。
- `blocked`：被阻塞，需要人工确认或外部条件。
- `review`：已完成，待检查。
- `done`：已完成并验证；阶段总结写入 README/CHANGELOG 后，应从看板中清理过旧 done 项。

## 2. 新线程启动顺序

任何后续 Codex 线程进入本项目后，先按顺序读取：

```text
AGENTS.md
README.md
TASKS.md
CHANGELOG.md
03-【匹配规则】-勘察测绘知识库-匹配规则提炼/【重要匹配规则】项目以及总体匹配规则介绍.md
```

如果线程只负责前端，也仍需读取前 4 个文件；如果会影响三个数字匹配、规则说明、输出列、报告口径，则必须读取 `03-【匹配规则】-勘察测绘知识库-匹配规则提炼/【重要匹配规则】项目以及总体匹配规则介绍.md`。

## 3. 三线程分工

当前项目只设 3 个长期线程：

```text
方案规划线程 -会话id“019eb9d2-ee41-7462-af9d-46aa263d2da2”
后端线程 -会话id“019eba23-3684-7890-b3e9-f7a8fe2b818c”
前端线程 -会话id“019eba08-5876-7661-b48f-416a4139954a”
```

规则知识库工作不单独开线程：规则抽取、口径判断、任务拆分归方案规划线程；规则落库、规则执行器和测试归后端线程。验收发布也不单独开线程：每个线程完成版本性工作时，都按项目规则自行完成验证、更新上下文和代码存档。

| 线程类型 | 主要职责 | 优先读的文件 | 主要修改范围 | 必跑验证 |
|---|---|---|---|---|
| 方案规划线程 | 定目标、拆任务、研究资料、确定规则口径、维护 `TASKS.md` 和交接说明 | `AGENTS.md`、`README.md`、`TASKS.md`、`CHANGELOG.md`、重要匹配规则说明 | `README.md`、`TASKS.md`、`CHANGELOG.md`、`docs/`、规则说明文档 | 搜索核对关键路径；涉及版本收尾时运行 `python tools/archive_code.py` |
| 后端线程 | Excel 解析、匹配引擎、规则 Excel/CSV、API、Word/Markdown 报告、后端测试 | `AGENTS.md`、`README.md`、`TASKS.md`、重要匹配规则说明 | `backend/`、`backend/app/rules/`、`tools/`、规则说明文档、必要时 README/CHANGELOG | `python -m pytest backend/tests -v`；涉及版本收尾时运行 `python tools/archive_code.py` |
| 前端线程 | 上传流程、列映射界面、表格预览、报告窗口、大模型调试界面、交互样式 | `AGENTS.md`、`README.md`、`TASKS.md`、`CHANGELOG.md` | `frontend/src/`、`frontend/package.json`、`frontend/package-lock.json`、必要时 README/CHANGELOG | `npm run build`；涉及接口联动时同时跑相关后端测试；涉及版本收尾时运行 `python tools/archive_code.py` |
