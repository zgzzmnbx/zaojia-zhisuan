# 数字化项目助手 iframe 接入设计

## 目标

在造价智算左侧导航“智能协同”下方新增一级菜单“数字化项目助手”，通过 iframe 加载独立项目 `D:\Codex-Temp\260712-【数字化造价助手】` 的前端页面。宿主仅提供菜单和展示容器，不复制、不代理、不修改独立助手的任何业务逻辑。

## 接入方式

- 新增宿主前端模块标识和导航项，保持其他菜单顺序不变。
- iframe 地址读取前端环境变量 `VITE_DIGITAL_PROJECT_ASSISTANT_URL`。
- 未配置环境变量时，开发默认地址为 `http://127.0.0.1:5175/?embed=1&theme=light`。
- iframe 使用语义化标题 `数字化项目助手`，独立助手的网络请求仍直接访问其后端 `http://127.0.0.1:8010`。
- 宿主不增加后端 Router、API 代理、业务模型、规则、公式、提示词或数据持久化。

## 页面状态

- 进入菜单后显示 iframe 加载状态。
- 超过合理等待时间仍未完成加载时，显示服务可能未启动的降级提示。
- 降级区提供“重试”和“新窗口打开”两个操作。
- iframe 成功加载后隐藏加载和错误提示。
- 独立助手不可用时，仅该模块降级，不影响填价、结果预览、经验池预警、工作量抓取、Word 报告、知识库问答和智能协同。

## 改动边界

- 只修改造价智算前端中模块类型、菜单定义、必要图标入口、iframe 容器和对应样式。
- 如项目已有前端环境变量类型声明或示例文件，仅补充该变量说明；不改项目级默认设置后端。
- 不重构现有导航和模块显示机制，不改匹配、预警、报告、知识问答及飞书 Webhook 逻辑。
- 保留工作区当前未提交的报告预览 UI 和其他用户改动，不覆盖、不格式化相关代码。

## 验收标准

1. “数字化项目助手”位于“智能协同”正下方，其他菜单顺序不变。
2. 点击菜单后加载环境变量指定页面；未配置时使用 `5175` 的嵌入地址。
3. iframe 成功加载时可正常操作独立助手，业务请求不经过造价智算后端。
4. 停止独立助手服务后，宿主显示可重试、可独立打开的降级提示，其他模块仍可切换和使用。
5. `npm run frontend:build` 通过。
