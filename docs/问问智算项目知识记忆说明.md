# 问问智算知识候选与知识记忆说明

## 1. 功能定位

知识记忆用于把用户主动认可的问答结论、复核意见和项目口径沉淀为“有来源、有范围、有版本、可审计、可撤销”的知识。当前默认为通用知识，无需选项目；操作方法和一般解释可按配置自动通过，价格 / 调整系数、正式标准、项目口径及冲突内容进入人工确认。项目和任务范围始终人工确认。它不等于永久保存全部聊天，也不会自动修改正式标准资料、规则表、结构化计价库、经验池或三个数字的匹配结果。

正式依据始终优先于项目记忆：

```text
正式标准与规则
＞ 结构化计价库
＞ 已确认的通用与同项目知识记忆
＞ 历史案例
＞ 大模型通用解释
```

## 2. 用户操作

1. 在“问问智算”完成知识库问答或行级复核。
2. 两种方式任选其一：点击回答下方“保存为知识候选”，或直接输入“记住这一条”。
3. 点按钮时可核对标题、原问题、结论、来源和证据摘要；默认范围为“通用知识”，不需要项目名称和 `project_key`。
4. 系统自动识别知识类型，并在同一范围检查完全重复、相似项和不同结论。完全重复时复用已有知识，不创建第二条。
5. 操作方法和一般解释等低风险通用知识自动完成提交和系统确认；价格 / 调整系数、正式标准、项目口径和冲突内容停留在“待确认”。自动类型和重复阈值可在项目默认配置中调整。
6. 如果知识只适用于某项目或任务，在弹窗中改选范围并填写项目信息；此类候选为“候选 → 待确认 → 人工确认”。
7. 已确认的通用知识可被所有项目检索；项目和任务知识只在相同 `project_key` 下检索。回答会显示范围、确认人、时间、适用条件和来源。
8. 发现错误或过期后，可填写原因并标记疑似失效或撤销；状态变更立即停止其参与检索，审计记录继续保留。

自然语言指令只作用于当前对话最近一次保存结果，不会把普通闲聊自动当成知识：

- `记住这一条`：按通用范围保存。
- `只在当前项目记住`：按当前项目范围保存；没有项目上下文时提示补充，不擅自创建范围。
- `把上一条改成：……`：候选直接更新；已确认知识创建更正版本，待新版确认后再替代旧版。
- `忘掉上一条` / `这条知识不对`：驳回未确认项或撤销已确认项。
- `查看刚保存的知识`：打开对应范围和记录。

已确认项目知识还可在“知识记忆”中点击“提升为通用知识”。提升会新建通用候选并重新审核，原项目知识继续有效。

## 3. 知识类型与审核策略

| 类型 | 默认通用审核 | 典型内容 |
| --- | --- | --- |
| `operation` | 自动通过 | 页面操作、上传下载、常见报错处理 |
| `general_explanation` | 自动通过 | 不涉及正式口径和数字裁决的一般解释 |
| `project_rule` | 人工确认 | 项目复杂程度、归类和特殊处理口径 |
| `price_factor` | 人工确认 | 基价、单价、费率和调整系数 |
| `standard_policy` | 人工确认 | 国家、行业、企业标准和制度条款 |

同范围存在明显不同结论时，无论类型均强制人工确认。该检查使用确定性文本相似度，只做提示和流程分流，不让大模型判断哪个结论正确。配置入口：

```json
"knowledgeMemory": {
  "autoApproveTypes": ["operation", "general_explanation"],
  "duplicateSimilarityThreshold": 0.92
}
```

## 4. 状态与检索规则

| 状态 | 是否参与问答 | 可执行动作 |
| --- | --- | --- |
| 候选 `candidate` | 否 | 编辑、提交、驳回 |
| 待确认 `pending` | 否 | 编辑、确认、驳回 |
| 已确认 `confirmed` | 是；通用知识面向所有项目，项目/任务知识仅限同 `project_key` | 标记疑似失效、撤销 |
| 已驳回 `rejected` | 否 | 查看审计 |
| 疑似失效 `suspected_stale` | 否 | 撤销 |
| 已撤销 `revoked` | 否 | 查看审计 |

知识设置了失效时间后，检索发现已过期会自动转为“疑似失效”。通用知识使用保留键 `通用知识`；项目和任务知识按规范化后的 `project_key` 隔离，不能跨项目命中。

## 5. 数据、版本与审计

运行数据库：

```text
Codex-Temp/runtime/knowledge-memory.sqlite3
```

主要数据表：

- `knowledge_items`：知识当前版本、知识类型、审核策略、项目范围、父子关系、来源、状态和责任人。
- `knowledge_versions`：每次创建和编辑的版本快照。
- `knowledge_audit`：创建、编辑、提交、确认、驳回、撤销和疑似失效等操作记录。

该数据库是运行数据，不是 `knowledge-qa-index-【codex】.json` 索引缓存。它被 Git 忽略，部署迁移或更换电脑时应按运行数据单独备份。

更正版本使用 `revision` 父子关系：新版确认后旧版自动转为疑似失效。项目知识提升通用使用 `promotion` 关系：无论通用候选是否确认，原项目知识都不会被自动替代。

## 6. API

知识候选与治理：

```text
POST   /api/knowledge-memory/candidates
GET    /api/knowledge-memory/items
GET    /api/knowledge-memory/items/{item_id}
PATCH  /api/knowledge-memory/items/{item_id}
POST   /api/knowledge-memory/items/{item_id}/revise
POST   /api/knowledge-memory/items/{item_id}/promote-general
POST   /api/knowledge-memory/items/{item_id}/submit
POST   /api/knowledge-memory/items/{item_id}/confirm
POST   /api/knowledge-memory/items/{item_id}/reject
POST   /api/knowledge-memory/items/{item_id}/revoke
POST   /api/knowledge-memory/items/{item_id}/mark-stale
GET    /api/knowledge-memory/items/{item_id}/audit
```

问答检索：

```text
GET  /api/knowledge/search?query=...&project_key=...
POST /api/knowledge/ask
```

`/api/knowledge/ask` 的请求体可携带 `project_key`。项目记忆数据库异常时，问答接口继续使用正式知识库证据，不让知识记忆故障阻断专业主流程。

## 7. 当前限制

- 当前操作人和确认角色由本地页面填写，仅用于 P0 试点和审计演示，不等于企业级身份认证或真实组织授权。
- 当前“通用”表示本实例内所有项目问答可用，不等于已实现企业级组织发布、跨用户授权或跨部署共享。
- 当前不提供自动知识抽取；冲突检测只提示相似记录并强制人工确认，不自动裁决正确结论。
- 正式依据候选只作为人工确认线索；当前尚未实现正式资料更新后的全库批量冲突扫描。
- WeAct 尚未接入候选待确认提醒；后续只应发送提醒和安全链接，不在消息中直接批准正式知识。
- 项目记忆与正式依据发生冲突时，回答必须采用正式依据，并由人工把项目记忆标记为疑似失效。
