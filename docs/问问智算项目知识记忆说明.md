# 问问智算项目知识记忆说明

## 1. 功能定位

项目知识记忆用于把用户主动认可的问答结论、复核意见和项目口径沉淀为“有来源、有项目范围、经人工确认、可撤销”的项目知识。它不等于永久保存全部聊天，也不会自动修改正式标准资料、规则表、结构化计价库、经验池或三个数字的匹配结果。

正式依据始终优先于项目记忆：

```text
正式标准与规则
＞ 结构化计价库
＞ 已确认的同项目知识记忆
＞ 历史案例
＞ 大模型通用解释
```

## 2. 用户操作

1. 在“问问智算”完成知识库问答或行级复核。
2. 点击回答下方“保存为知识候选”。
3. 核对系统预填的标题、原问题、待确认结论、项目、来源和证据摘要。
4. 补充适用条件、例外情况，选择当前项目或当前任务范围。
5. 创建后状态为“候选”；在“知识记忆”窗口提交后转为“待确认”。
6. 本地试点的项目负责人、复核人或规则维护人可以确认或驳回。
7. 已确认内容只在相同 `project_key` 的知识库问答中参与检索；回答会显示“项目记忆”、项目、确认人、确认时间、适用条件和来源。
8. 发现错误或过期后，可填写原因并标记疑似失效或撤销；状态变更立即停止其参与检索，审计记录继续保留。

## 3. 状态与检索规则

| 状态 | 是否参与问答 | 可执行动作 |
| --- | --- | --- |
| 候选 `candidate` | 否 | 编辑、提交、驳回 |
| 待确认 `pending` | 否 | 编辑、确认、驳回 |
| 已确认 `confirmed` | 是，仅限同项目且未过期 | 标记疑似失效、撤销 |
| 已驳回 `rejected` | 否 | 查看审计 |
| 疑似失效 `suspected_stale` | 否 | 撤销 |
| 已撤销 `revoked` | 否 | 查看审计 |

知识设置了失效时间后，检索发现已过期会自动转为“疑似失效”。项目知识记忆只按规范化后的 `project_key` 隔离，不能跨项目命中。

## 4. 数据与审计

运行数据库：

```text
Codex-Temp/runtime/knowledge-memory.sqlite3
```

主要数据表：

- `knowledge_items`：知识当前版本、项目范围、来源、状态和责任人。
- `knowledge_versions`：每次创建和编辑的版本快照。
- `knowledge_audit`：创建、编辑、提交、确认、驳回、撤销和疑似失效等操作记录。

该数据库是运行数据，不是 `knowledge-qa-index-【codex】.json` 索引缓存。它被 Git 忽略，部署迁移或更换电脑时应按运行数据单独备份。

## 5. API

知识候选与治理：

```text
POST   /api/knowledge-memory/candidates
GET    /api/knowledge-memory/items
GET    /api/knowledge-memory/items/{item_id}
PATCH  /api/knowledge-memory/items/{item_id}
POST   /api/knowledge-memory/items/{item_id}/submit
POST   /api/knowledge-memory/items/{item_id}/confirm
POST   /api/knowledge-memory/items/{item_id}/reject
POST   /api/knowledge-memory/items/{item_id}/revoke
POST   /api/knowledge-memory/items/{item_id}/mark-stale
GET    /api/knowledge-memory/items/{item_id}/audit
```

问答检索：

```text
GET  /api/knowledge/search?query=...&project_key=...
POST /api/knowledge/ask
```

`/api/knowledge/ask` 的请求体可携带 `project_key`。项目记忆数据库异常时，问答接口继续使用正式知识库证据，不让知识记忆故障阻断专业主流程。

## 6. 当前限制

- 当前操作人和确认角色由本地页面填写，仅用于 P0 试点和审计演示，不等于企业级身份认证或真实组织授权。
- 当前不提供组织级发布、跨项目推荐、自动知识抽取和自动冲突裁决。
- WeAct 尚未接入候选待确认提醒；后续只应发送提醒和安全链接，不在消息中直接批准正式知识。
- 项目记忆与正式依据发生冲突时，回答必须采用正式依据，并由人工把项目记忆标记为疑似失效。
