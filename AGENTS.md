# 项目 AGENTS.md

## 项目名称

造价智算-工程造价辅助智能体

## 协作规则

- 当前项目是从 WeAgent 平台迁移到本地电脑运行的软件原型；软件对外名称已升级为“造价智算”，定位为“工程造价辅助智能体”，当前核心场景仍覆盖长输管道勘察测量最高投标限价编制。
- 原始资料、测试表和答案表不直接改动。
- 生成文件放在程序运行输出目录中，不覆盖原始输入。
- 仅本项目适用：从现在开始，新生成的文件名不要再追加 `codex`、`【codex】`、`-codex`、`codeX` 等尾缀；除非用户明确要求保留历史命名，后续输出、提示词、报告、临时结果、截图、素材和项目说明文件都使用不暴露 Codex 的命名。
- 历史知识库、经验池模板、输入样例和兼容脚本里已经存在的“管勘智算”文件名暂时保留，不做批量重命名；除非用户明确发起迁移任务，否则不要为了品牌升级改坏真实路径引用。
- 知识库使用二维 Excel 数据库，不把价格裁决交给大模型；固定知识库为 `03-知识库-二维数据库制作/【数据库】【导入】.xlsx`，其中 A-H 用于基价，J-M 用于调整系数第二层经验提示。
- 大模型后续只作为报告润色、待复核解释、自然语言助手，不直接输出最终单价。
- 用户说“git存档”时，或本项目更新版本号 / CHANGELOG / 当前版本计划并完成验证后，视为触发一次本地 Git 存档：先用 `git status --short` 和 `git diff --stat` 做轻量检查，再执行 `git add` 与 `git commit`；提交信息和交付说明必须写清本次存档备注。该操作只做本地提交，不等同于运行代码版 zip 存档，也不得自动 `git push`。
- 造价智算与 `D:\Codex-Temp\260712-【数字化造价助手】` 之间使用 `D:\Codex-Temp\260712-【数字化造价助手】\00-docs\02-架构与集成\03-造价智算宿主交接状态.md` 作为跨项目交接信箱。凡涉及数字化项目助手的菜单、iframe、启动、主题、联调或发布，开始前必须读取该文件；造价智算侧完成相关工作后，必须更新“宿主最新交接、当前状态、下一责任方和更新时间线”，不得只在聊天中口头交接。

- `README.md` 是当前模块状态入口。后续线程按模块阅读和更新 README 的“当前模块状态”，不要再把已完成历史长期堆在 README。

## 技术边界

- 前端使用 React + TypeScript + Vite。
- 后端使用 Python + FastAPI。
- 桌面壳使用 Tauri 2，仅作为开发调试和未来桌面化入口；React 和 FastAPI 仍是唯一主界面与唯一业务后端。
- Excel 读写使用 openpyxl。
- Word 报告使用 python-docx。
- 跨平台路径使用 `pathlib`，不写死仅本机可用的运行路径。

## 项目经验 / 注意事项

- WeAgent 历史经验表明，主价格判断不宜依赖大模型自由生成或相似度最高候选，应优先采用字段硬校验和归一化后的结构化匹配。
- 常见归一化差异包括全角半角括号、比例前缀、空格、换行和价格列名称差异。
- 版本存档只备份项目代码和代码恢复所需上下文，不打包比赛资料、知识库原始表、汇报 PPT、演示素材和运行临时输出。
- `01-assets/01-UI参考图/01-assets/` 与 `01-assets/01-UI参考图/00-reference/` 属于 UI 图片素材和参考资料，不进入代码版存档；`02-notes/` 和 `code/` 可作为 UI 设计上下文进入代码存档；如需保存图片素材，按资料归档处理，不混入代码存档。
- 代码版 zip 存档改为手动触发：只有用户明确要求“代码备份”“代码存档”“打包存档”或指定运行 `备份代码.bat` / `python tools/archive_code.py` 时才执行；版本完成时默认只做本地 Git 存档，不自动运行传统代码打包存档。代码打包仍优先双击 `备份代码.bat` 或运行 `python tools/archive_code.py`，不要手工重新枚举目录；脚本白名单只包含代码、启动脚本、项目上下文和开发文档。
- 每次运行 `python tools/archive_code.py` 生成根目录 `04-输出版本存档` 内的代码存档时，脚本也必须同步复制一份到 `D:\Desktop\01-工作流-【造价】\工作流\260612-【ai大赛】-管勘智算-V2.0\04-输出版本存档`，作为工作流目录副本。
- 代码存档过程必须通过 `python tools/archive_code.py` 实现，直接运行脚本即可；脚本输出的 archive 路径和 SHA256 可在交付说明或 CHANGELOG 中顺手记录。脚本运行结束后不要再额外检查 zip 是否存在、工作流副本是否复制成功、SHA256 是否一致或目录是否完整，是否存档成功由大尾巴人工查看。
- 运行时追加输出列后，前端预览刷新必须重新读取实际表头，但仍要保持顺序读取和有限列宽，避免宽表回到逐格随机访问导致卡顿。
- 经验池预警分析耗时应与首次结构化识别解耦；默认通过前端按钮手动触发，不要重新放回 `/api/process` 初次转换链路，除非用户明确要求。
- 经验池预警当前只按 `字段完全匹配 -> 非空要素顺序匹配 -> 未找到可比选同类记录` 找同类；风险等级只看“当前值相对经验池平均值的偏离率”，`05-经验池-预警数据/experience-warning-settings-【codex】.json` 保存低风险 / 高风险阈值，默认分别为 `5%`、`20%`。
- 经验池预警默认开启“只核查指定列有值的行”，当前默认按 `数量` 字段定位实际列；该过滤只作用于预警候选行筛选阶段，且 `0`、公式结果为 `0`、空白都按“没有值”处理。若当前 sheet 找不到该字段，要明确报错，不要静默跳过。
- “导入已有控制价文件到经验池”的字段设置只影响经验池导入 inspect 的列自动识别偏好，保存文件为 `05-经验池-预警数据/experience-field-preferences-【codex】.json`；不要让它改变价格匹配、第二层经验提示或预警比选规则。
- 经验池导入默认开启“只导入某一列有值的行”，默认按 `工程量` 字段映射列判断；这是为了避免模板空行进入经验池，不应影响经验池预警比选规则。
- 新生成经验池必须以 `05-经验池-预警数据/【经验池】【模板勿动】-管勘智算.xlsx` 为母版，保留模板样式；导入时如果过滤字段连续空白很多行，应提前停止扫描，避免带格式空模板行导致电脑卡顿。
- 项目根目录只保留源码、规则资料、正式资料入口、文档和必要启动文件；临时缓存、Playwright 检查输出、pytest 缓存、临时外包件、截图 / 过程产物等统一放到 `Codex-Temp/` 下。绿色版运行缓存放在绿色版目录内；`node_modules` 是根目录 npm 依赖，不按临时文件移动。
- 原始工作量抓取的项目默认字段设置统一写在 `config/project-default-settings.json`；弹窗内修改只影响当前页面和本次 inspect，不再写 `workload-field-preferences-【codex】.json` / `workload-target-field-preferences-【codex】.json`，也不要让它改变价格匹配、经验池预警或第二层经验提示规则。

- 原始工作量抓取当前允许在 A+B 前先走 `【术语归并】术语归并与匹配放宽规则表.xlsx` 的工作量作用域规则；这层只做工作量抓取旁路归并，`字段名` 留空表示作用于全部要素列，不完整规则行应自动忽略。
- 现有规则资产里，`backend/app/fill_engine.py` 与 `backend/tests/test_matching.py` 已明确验证上述合并单元格默认口径；前端列映射面板文案也按同一口径提示用户。当前未发现与该默认口径冲突的项目规则文件；`merge_vertical_cells` / `merge_horizontal_cells` 只属于主填价流程的显式开关，不得被其他模块偷换成不同默认行为。
- 知识库问答索引缓存为 `Codex-Temp/runtime/knowledge-qa-index-【codex】.json`，只作运行缓存；检索接口 `/api/knowledge/search` 不调用大模型，问答接口 `/api/knowledge/ask` 才会在有证据时调用 DeepSeek。
- Windows 绿色版只负责在目标电脑启动本地后端和本地前端服务，并打开默认浏览器；不得把业务逻辑迁移到启动脚本里。
- 云端发布包包含中文目录和文件名时，必须使用 UTF-8 保真的 `tar.gz` 在 Linux 服务器解压；不要直接使用 Windows `Compress-Archive` 生成的 ZIP 交给 Linux `unzip`，否则中文路径可能乱码，导致正式二维知识库、报告模板和规则文件报“文件不存在”。
- 正式 Word 模板含页码文本框，它会使每页 `scrollHeight` 固定多出约 `98px`；File Viewer 报告预览应设置高于该装饰误差的 `paginationTolerance`，并保留有上限的动态分页，不能再次用 `maxDynamicPaginationPasses: 0` 关闭正文自动续页。回归时至少核对正式 3 页样例、一个 5 页以上显式分页样例和一个由长正文自动续排到 7 页左右的实际报告。
- 统信 UOS 兼容性准备当前只是未来迁移预案，不是日常开发主线；`python tools/check_platform_compat.py` 只在统信/Linux 相关修改、路径/启动/打包规则调整或明确要求兼容性复核时手动运行，不挂入平时启动、`npm run build`、`python -m pytest backend/tests -v`、代码存档或 Windows 绿色版默认流程，避免拖慢当前 Windows 开发。
- 同一飞书应用的多个长连接实例采用随机集群分发而非广播；本地与服务器迁移时必须只启用一个第二层机器人实例，使用智能协同页“启用接收”开关切换，避免任务、队列和成果随机分裂到不同环境。
- 第一层 Webhook 与第二层企业应用机器人的运行秘密统一保存在 `Codex-Temp/runtime/feishu-robot-settings.json`，分别使用 `webhook` 和 `app_bot` 分区；新增机器人配置时只维护这个文件，旧版 `feishu-webhook-settings.json` / `feishu-app-settings.json` 仅用于自动迁移兼容，不再作为现行配置入口。
- Windows 上不能用 `os.kill(pid, 0)` 判断长连接进程是否存在（会返回无效参数）；第二层机器人状态应使用 Windows 进程句柄检测，否则页面可能误报未运行并重复拉起实例。
- 云端 FastAPI 固定监听 `1285`，而本机默认监听 `8000`；服务器运行第二层机器人时，`feishuAppBot.apiBaseUrl` 必须在服务器运行时配置为 `http://127.0.0.1:1285`，不能直接沿用本机默认值，否则任务处理和知识库问答都会报 `[Errno 111] Connection refused`。
- 企业 WeAct 第二层机器人不能沿用 SDK 默认公网飞书域名；对应配置必须保存 `domain=https://open.weact.pipechina.com.cn`，并让 REST API 客户端和 WebSocket 客户端同时使用该域名。连接成功标志是进入 `wss://lark-frontier.weact.pipechina.com.cn/ws/v2`；省略域名会反复报 `1000040343: internal error`。
- 第二层机器人控制台不得直接回显 `runner.out.log` / `runner.err.log` 原文；SDK 连接 URL 含短期票据，只能解析并返回时间、状态和节点域名。当前试点经用户明确授权，业务消息日志可记录发送人名称与 ID、群名与会话 ID、消息正文和附件名；仍不得记录 App Secret、访问令牌、连接票据、文件 Key 或完整 WebSocket 地址。飞书长连接事件不提供发送者来源 IP，界面必须明确标注“平台未提供”，不得伪造。
