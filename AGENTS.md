# 项目 AGENTS.md

## 项目名称

造价智算-工程造价辅助智能体

## 协作规则

- 当前项目是从 WeAgent 平台迁移到本地电脑运行的软件原型；软件对外名称已升级为“造价智算”，定位为“工程造价辅助智能体”，当前核心场景仍覆盖长输管道勘察测量最高投标限价编制。
- 原始资料、测试表和答案表不直接改动。
- 生成文件放在程序运行输出目录中，不覆盖原始输入。
- 仅本项目适用：从现在开始，新生成的文件名不要再追加 `codex`、`【codex】`、`-codex`、`codeX` 等尾缀；除非用户明确要求保留历史命名，后续输出、提示词、报告、临时结果、截图、素材和项目说明文件都使用不暴露 Codex 的命名。
- 复赛 PPT 按版本成套归档：每个版本必须在 `07-汇报PPT 和 演示素材/00-汇报PPT和素材【同步】/260715-【复赛PPT】/03-定稿/` 下建立独立的 `日期-【Ai稿和AiPPT】造价智算-复赛PPT内容稿与逐页演讲稿-vX.X.X【主要/历史】/` 目录；该版本主文稿放在目录根部，对应的完整逐页 PPT 图片集放在同一目录下。不同版本的文稿和图片不得混放，新增页过程图须与“完整图片集”明确区分，便于从文稿版本直接定位对应视觉稿。
- 历史知识库、经验池模板、输入样例和兼容脚本里已经存在的“管勘智算”文件名暂时保留，不做批量重命名；除非用户明确发起迁移任务，否则不要为了品牌升级改坏真实路径引用。
- 知识库使用二维 Excel 数据库，不把价格裁决交给大模型；固定知识库为 `03-知识库-二维数据库制作/【数据库】【导入】.xlsx`，其中 A-H 用于基价，J-M 用于调整系数第二层经验提示。
- 大模型后续只作为报告润色、待复核解释、自然语言助手，不直接输出最终单价。
- 用户说“git存档”时，或本项目更新版本号 / CHANGELOG / 当前版本计划并完成验证后，视为触发一次本地 Git 存档：先用 `git status --short` 和 `git diff --stat` 做轻量检查，再执行 `git add` 与 `git commit`；提交信息和交付说明必须写清本次存档备注。该操作只做本地提交，不等同于运行代码版 zip 存档，也不得自动 `git push`。
- 造价智算与 `D:\Codex-Temp\260712-【数字化造价助手】` 之间使用 `D:\Codex-Temp\260712-【数字化造价助手】\00-docs\02-架构与集成\03-造价智算宿主交接状态.md` 作为跨项目交接信箱。凡涉及数字化项目助手的菜单、iframe、启动、主题、联调或发布，开始前必须读取该文件；造价智算侧完成相关工作后，必须更新“宿主最新交接、当前状态、下一责任方和更新时间线”，不得只在聊天中口头交接。

## 项目重要文件

- 当前项目按模块维护，不再使用根目录 `TASKS.md` 作为任务入口；旧任务看板已归档到 `docs/归档/TASKS-归档-2026-06-18.md`，仅用于必要时回看历史。
- `README.md` 是当前模块状态入口。后续线程按模块阅读和更新 README 的“当前模块状态”，不要再把已完成历史长期堆在 README。
- `CHANGELOG.md` 只保留最新 3 条记录；更早记录放在 `CHANGELOG - 【归档】.md`。每次升级版本、集中补充变更记录或修改 `CHANGELOG.md` 后，运行一次 `python tools/trim_changelog.py` 自动瘦身归档；脚本未报错即可，不要再反复校验是否确实只剩 3 条，由大尾巴人工查看。
- `00-PRD/` 是根目录轻量 PRD 驱动入口：`00-产品总览.md` 写产品定位，`02-当前版本计划.md` 写当前计划，`03-整体UI设计PRD.md` 写全局 UI 设计总纲，`03-UI设计规范.md` 写可执行 UI 规范，`01-模块PRD/模块名/PRD.md` 按功能模块维护需求、边界和验收口径；不再使用单个 `01-模块PRD索引.md`。当前辅助填价与人工复核闭环已独立为 `09-辅助填价与人工复核闭环/PRD.md`，但不代表新增左侧一级页面。
- `01-assets/01-UI参考图/` 是 UI 参考素材工作区：`01-assets/design/` 放设计稿和过程图，`01-assets/bug/` 放报错截图和视觉问题截图，`00-reference/` 放外部参考设计，`02-notes/` 放技术经验和踩坑记录，`code/` 放 UI demo / 静态原型 / 临时实验代码。正式生产代码仍在 `frontend/`、`backend/`、`src-tauri/`，不要搬进该目录。
- `AGENTS.md` 只沉淀长期规则、长期项目约束和反复踩坑经验；一次性分析、技术笔记、UI 参考解读和过程记录放入对应 PRD、docs 或 `01-assets/01-UI参考图/02-notes/`。
- 每个模块 PRD 的 `## 需求清单` 使用 `[已完成]`、`[进行中]`、`[待开发]`、`[暂缓]`、`[不做]` 标记状态；PRD 只保留当前仍有指导意义的 5 到 10 条需求，历史完成流水账进入 `CHANGELOG.md`。
- 每个模块 PRD 的 `## 关联资产` 只引用本模块依赖的模板、匹配规则、知识库、配置和关键代码路径；不得为了整理 PRD 而移动真实模板、规则表、知识库或程序读取文件。
- 需要核对 PRD 与项目现状时运行 `python tools/check_prd_consistency.py`，生成 `00-PRD/04-PRD一致性巡检.md`；该脚本只检查结构、状态标记、关联资产路径和旧路径残留，不替代人工验收 `[已完成]` 需求。
- PRD 备份改为手动触发：只有用户明确要求“备份 PRD”时，才运行 `python tools/archive_prd.py` 或双击 `备份PRD.bat`，将当前 PRD 快照归档到 `00-PRD/archive/yyyy-mm-dd-vN/`；日常修改 PRD 不自动备份，日常接手默认不读取 archive，除非用户要求回看旧版。
- 当前对外交付只维护传统开发版和 Windows 绿色版：传统开发版用于本机开发调试，绿色版用于发给其他 Windows 电脑直接运行。Tauri 桌面壳仍作为开发调试和未来桌面化资产保留，但不进入当前绿色版。
- `tools/build_green_release.py` 用于生成 Windows 绿色版目录和压缩包；绿色版沿用后端 `8000` + Vite 前端 `5174` 的传统双服务逻辑，随包携带 Python、Node、npm、后端依赖和 `frontend/node_modules`。
- `docs/绿色版说明.md` 记录绿色版运行、配置、打包和排障说明；后续改绿色版启动、停止、状态检查、依赖携带或配置方式前先读。
- `config/project-default-settings.json` 是全部项目级默认设定的集中配置入口，当前承载表格预览、列映射、工作量抓取和问问智算窗口默认值；后续新增项目级默认设定时必须优先写入该文件，弹窗内用户修改只临时生效。若因密钥、运行环境、个人界面状态或技术边界不能写入，必须在交付时向大尾巴说明原因、实际存放位置和生效范围。绿色版和云端发布必须随包携带该文件，保证新用户首次打开即按项目默认口径运行。
- `business-skills/` 是专业能力清单入口：程序只读取 `manifest.json`，`SKILL.md` 只作工作说明，不参与价格、系数或风险裁决。新增或升级已上线 Skill 时，必须使用项目内相对路径引用已存在的非可执行资产，不得保存密钥、脚本或命令；必须补专项验证、任务快照和绿色版携带回归。
- `src-tauri/`、`tools/run_tauri.ps1`、`启动管勘智算-Tauri-MVP.bat` 是桌面壳开发资产；绿色版构建和发给评审/试用人员的网页运行包不依赖这些文件。
- 面向普通读者、汇报对象、参赛评委和新接手人员的项目说明，以 `README.md` 和 `00-PRD/00-产品总览.md` 为准；早期一页式介绍已归档到 `docs/归档/项目介绍-给人看的版本-历史.md`，日常开发不默认读取。
- `03-【匹配规则】-勘察测绘知识库-匹配规则提炼/【重要匹配规则】项目以及总体匹配规则介绍.md` 是本项目核心规则说明文件，解释 `基价 / 单价`、`实物工作费调整系数`、`技术工作费调整系数` 三个输出数字的匹配来源、颜色、判断规则、待复核口径和扩充方式。

## 技术边界

- 前端使用 React + TypeScript + Vite。
- 后端使用 Python + FastAPI。
- 桌面壳使用 Tauri 2，仅作为开发调试和未来桌面化入口；React 和 FastAPI 仍是唯一主界面与唯一业务后端。
- Excel 读写使用 openpyxl。
- Word 报告使用 python-docx。
- 跨平台路径使用 `pathlib`，不写死仅本机可用的运行路径。

## 验证要求

- 能测试就测试。
- 修改匹配逻辑后，必须运行 `python -m pytest backend/tests -v`。
- UI 小修改按轻量流程处理：不更新版本号、不写 CHANGELOG、不跑 PRD 一致性巡检、不执行代码存档；只运行 `npm run frontend:build` 验证即可。除非用户明确要求升级版本、备份/存档、更新 PRD 或本次 UI 改动已经扩大为集中重构。
- 改完代码后默认不自动重启 `8000` / `5174` 服务；完成测试、构建和代码存档后交付即可。只有用户明确要求重启、需要用浏览器做运行态验证、或修改了启动器/端口/环境变量/依赖/后端启动逻辑时，才重启服务。重启时仍固定使用 `8000` 和 `5174`，不要自动换端口。
- 修改绿色版启动、停止、依赖携带、端口或环境变量读取后，至少运行 `python -m py_compile tools/build_green_release.py`、`npm run frontend:build`、后端健康检查，并对新生成绿色版做启动验证。
- 修改 Tauri 壳后，至少运行 `npm run frontend:build`、后端健康检查和 `python -m py_compile tools/build_tauri_release.py`；若本机有 Rust/Cargo，再运行 `npm run tauri:dev` 或 `npm run tauri:build`。
- 关键样例是当前 100 行空单价输入表，输出应与答案表逐行一致。
- 如果出现多候选、无候选或字段冲突，输出 `待复核`，不要强行填入不确定价格。

## 项目经验 / 注意事项

- WeAgent 历史经验表明，主价格判断不宜依赖大模型自由生成或相似度最高候选，应优先采用字段硬校验和归一化后的结构化匹配。
- 常见归一化差异包括全角半角括号、比例前缀、空格、换行和价格列名称差异。
- 版本存档只备份项目代码和代码恢复所需上下文，不打包比赛资料、知识库原始表、汇报 PPT、演示素材和运行临时输出；但 Git 必须持续跟踪 `07-汇报PPT 和 演示素材/00-汇报PPT和素材【同步】/260715-【复赛PPT】/03-定稿/260715-【Ai稿和AiPPT】造价智算-复赛PPT内容稿与逐页演讲稿-v3.3.0【主要】/260715-造价智算-复赛PPT内容稿与逐页演讲稿-v3.3.0【主要】.md`，其余该目录资料仍默认不入库。
- `01-assets/01-UI参考图/01-assets/` 与 `01-assets/01-UI参考图/00-reference/` 属于 UI 图片素材和参考资料，不进入代码版存档；`02-notes/` 和 `code/` 可作为 UI 设计上下文进入代码存档；如需保存图片素材，按资料归档处理，不混入代码存档。
- 代码版 zip 存档改为手动触发：只有用户明确要求“代码备份”“代码存档”“打包存档”或指定运行 `备份代码.bat` / `python tools/archive_code.py` 时才执行；版本完成时默认只做本地 Git 存档，不自动运行传统代码打包存档。代码打包仍优先双击 `备份代码.bat` 或运行 `python tools/archive_code.py`，不要手工重新枚举目录；脚本白名单只包含代码、启动脚本、项目上下文和开发文档。
