# CHANGELOG

## v5.4.1 - 2026-07-05

- 升级版本号到 `v5.4.1`：同步根目录 npm 包、前端 npm 包、后端版本、Tauri 配置、Cargo 元数据、README 和当前版本计划。
- 收口结果预览列宽体验：表头右边缘可像 Excel 一样拖拽调节列宽，按 `sheet 名 + 列标题` 保存本机偏好；不显示额外手柄，保持表格简洁。

- 优化大尾巴主题细节：左侧模块导航去掉 `01/02/03` 数字编号，预览窗口工具栏按钮降为紧凑低高度样式，完成态改为浅蓝轻量胶囊。
- 验证：`npm run frontend:build` 通过；`python tools/check_prd_consistency.py --strict` 通过。

## v5.4.0 - 2026-07-05

- 升级版本号到 `v5.4.0`：同步根目录 npm 包、前端 npm 包、后端版本、Tauri 配置、Cargo 元数据、README 和当前版本计划。
- 新增经验池治理报告：`/api/quality/experience-pool` 只读检查重复记录、单位不一致、样本不足、极端离散、关键字段空值和经验数不可解析，并生成 Markdown 治理报告。
- 新增风险报告增强与标准依据轻量追溯：`/api/risk/summary` 输出结构化风险项，`/api/standard-trace` 汇总待复核、经验池来源和规则说明线索，前端 Word 报告页可查看风险清单并跳转定位。
- 新增演示 / 评委模式 MVP：填价工作台可一键加载预置 100 行样例，按正式转换链路生成结果，并提供结果、预警、风险清单和报告的轻引导。
- 修复演示样例加载：演示接口复用正常前端的候选 sheet 识别和列映射口径，自动跳过“费用汇总”等非明细页，避免完整控制价工作簿误报缺少 `要素1`、`单位`。
- 新增辅助填价 MVP：结果预览行操作区可打开辅助填价弹窗，展示知识库相似和经验池同类均值候选；用户确认后复用输出副本写回校验、浅蓝批注和 `preview-manual-edits.json` 留痕。
- 增强辅助填价与标准依据追溯联动：基价 / 单价辅助填价弹窗合并展示价格候选和当前行轻量追溯；两个系数候选与溯源作为第二步写入 PRD 待开发。
- 调整主流程为两阶段批量匹配体验：`开始转换` 只生成待匹配预览，预览窗口蓝色 `批量匹配` 按钮再执行价格和两个系数识别填写，右侧智算同步输出阶段提示。
- 修复两阶段批量匹配：刷新预览、人工修改或重算公式后继续保留开始转换时的列映射和 sheet 配置，避免批量匹配误报缺少 `要素1 / 单位`。
- 验证：`python -m pytest backend/tests -v` 通过（150 passed, 3 skipped）；`npm run frontend:build` 通过。
