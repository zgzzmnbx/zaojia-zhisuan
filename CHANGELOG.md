# CHANGELOG

## v5.9.1 - 2026-07-17

- 知识候选默认改为“通用知识”，无需选择项目；通用候选当前自动完成提交和系统确认，同时保留审核接口、版本、撤销与 create / submit / confirm 完整审计。
- 保留“保存为知识候选”按钮，并新增“记住这一条”等确定性口令：只保存当前对话中最近一条已完成的结构化知识候选，不把普通闲聊自动沉淀。
- 问答检索同时纳入已确认通用知识和当前 `project_key` 下已确认知识；项目/任务范围仍保持人工审核与跨项目隔离，正式标准和规则优先级不变。
- 验证完成：后端知识记忆专项、前端生产构建、Python 编译和真实浏览器的按钮保存、自然语言保存、自动确认及审计查看均通过；全量回归仍保留开发前已知的标准资料排序和第 90 行样例数据漂移两项失败。

## v5.9.0 - 2026-07-17

- 完成问问智算“知识候选池 + 项目知识记忆 + 人工确认”P0：知识库回答和行级复核可预填保存候选，网页支持编辑、提交、确认、驳回、撤销、标记疑似失效及完整审计。
- 新增 SQLite 项目知识记忆库与版本记录；只有同一 `project_key` 下已确认、未过期的知识参与问答，正式标准和规则保持更高优先级，回答明确展示项目范围、确认人、确认时间、适用条件和来源。
- 知识记忆故障自动降级到原有正式知识库问答，不影响 Excel 填价、经验池预警、Word 报告和其他专业主流程；当前操作人与确认角色明确限定为本地试点，不冒充企业身份认证。
- 验证完成：知识记忆专项 `10 passed`，全量后端 `300 passed, 3 skipped, 2 failed`，两项失败与开发前已知的标准资料检索排序、项目样例第 90 行数据漂移基线一致；前端生产构建、Python 编译、浏览器候选创建 / 提交 / 确认 / 疑似失效闭环通过。PRD 硬检查仅剩既有外部申报书路径缺失。
