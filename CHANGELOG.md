# CHANGELOG

## v5.6.0 - 2026-07-10

- 新增“智能协同”第一层飞书群自定义机器人 Webhook：左侧一级菜单固定在“知识库问答”下方，中间页提供连接状态、秘密配置、签名提示、通知规则、测试发送和脱敏历史，右侧问问智算保持不变。
- 新增独立 `backend/app/feishu_webhook.py` 与状态 / 设置 / 测试 / 历史 / 受限事件通知 API；Webhook 地址和签名密钥只写入被 Git 忽略的运行目录，支持飞书官方 HMAC-SHA256 签名、完成卡片 URL 跳转、瞬时故障一次有限重试和秘密脱敏。
- 现有“开始转换 -> 待匹配预览 -> 批量匹配”前端链路以 best-effort 方式旁路发送开始、进度、完成和失败通知；通知禁用、未配置、超时、限流或失败均不改变 Excel、匹配、预警、工作量抓取、问问智算和 Word 业务结果。
- 第一层完成通知只允许 `open_url` 跳转，不实现卡片回调；第二层企业自建应用机器人、收消息 / 文件、用户身份、任务台账、风险派单和审批继续保持未开发。
- 版本同步至 `v5.6.0`；README、当前版本计划、智能协同模块 PRD、整体 UI PRD、UI 设计规范和健康检查测试已同步，PRD 严格一致性巡检通过。
- 验证：Webhook 专项 `17 passed`；全量后端 `173 passed, 3 skipped, 2 failed`，两项失败分别来自当前未纳入本任务改动的标准资料检索排序和项目样例第 90 行源值漂移；前端构建、Tauri release 构建、绿色版完整构建及包内健康 / 未携带飞书秘密检查通过；Playwright 在 `1366×768` 和左右栏折叠态确认智能协同页与右侧智算可用。
