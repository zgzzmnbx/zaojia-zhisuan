# CHANGELOG

## v5.8.18 - 2026-07-15

- 在项目默认配置集中登记普通飞书与企业 WeAct 的预期 App ID 和域名；本地与云端的 `weact_cost` 统一使用已完成消息端到端验证的应用 A，App Secret 继续只保存在各环境 Git 忽略的运行秘密文件中。
- 第二层页面开关、后端子进程拉起和独立运行器增加同一套身份校验；App ID 或域名与项目登记不一致时显示“配置不一致”并拒绝启动，避免错误应用虽然建立 WebSocket 却收不到正式机器人消息。
- 新增配置偏差回归用例并保留普通飞书 / WeAct 双配置切换、队列、上传、匹配、风险、Excel、Word、知识库与普通问答原有链路。

## v5.8.17 - 2026-07-15

- 第二层普通飞书 / WeAct 收件入口增加持久去重：同时对事件 ID 和消息 ID 建立 SQLite 唯一约束，解决平台以新事件 ID 重发同一条历史消息时重复回复的问题。
- 引入平台消息创建时间校验：能确定已超过 5 分钟的消息会在“了解”表情、文字回复、知识库、上传和任务路由之前静默拦截；平台未提供或无法解析时间时保持旧行为，避免损伤兼容消息。
- 运行控制台对处理、过期和重复消息统一记录消息 ID、平台创建时间和本机接收时间；旧 SQLite 事件表自动增量迁移，既有队列、上传、匹配、风险、Excel、Word、知识库和大模型问答接口不变。

## v5.8.16 - 2026-07-14

- 第二层机器人新增“第一梯队”确定性任务指令：`任务` / `最近任务`、`进度 FS-任务编号`、`风险 FS-任务编号`、`高风险`、`结果 FS-任务编号`、`帮助` / `指令`；完整指令直接读取本地任务队列，不进入大模型托底。
- 所有任务列表和详情查询按当前 `chat_id` 强制隔离，群与群、群与单聊之间不能交叉读取任务编号、文件名、风险和成果；群聊仍必须明确 @当前机器人，单聊可直接发送。
- “结果”只重发原任务已保存的 Excel、Word 和经典完成卡片，不重新计算、不修改原任务状态；成果过期、任务未完成、跨会话访问和上传失败均返回确定性提示并保留原记录。
