# CHANGELOG

## v5.8.14 - 2026-07-14

- 新增第二层群成员确定性查询：群聊明确 @本机器人后，完整输入“群里有几个人”“群成员”或“都有谁”，机器人直接调用飞书 / WeAct 群成员接口返回真实人数和编号名单，不经过大模型。
- 群成员接口支持 100 人每页的自动分页、重复成员去重、名单超长截断和权限错误提示；单聊输入三条指令时提示到目标群查询，近似说法继续走普通大模型问答。
- 自我介绍增加群成员查询用法，机器人运行控制台区分群成员接口查询与大模型托底；增加精确指令、近似说法、单聊边界、分页去重和输出格式测试。

## v5.8.13 - 2026-07-14

- 修复群聊中 @其他人或其他机器人时，本机器人仍添加“了解”表情并进入大模型问答的问题；启动时通过飞书 / WeAct 机器人信息接口读取本机器人真实 Open ID，群消息按 mentions 中被提及对象的 Open ID 精确过滤。
- 当前机器人身份无法确认时群聊保持静默，避免误回复；机器人单聊、收件窗口内独立文件消息和明确 @本机器人的上传、知识库、问候及普通问答继续按原规则处理。
- 增加 @本机器人、@其他对象、身份缺失静默、三类消息路由和机器人身份缓存测试；当前 WeAct 已成功解析本机器人身份。

## v5.8.12 - 2026-07-14

- 修复企业 WeAct 将群聊机器人提及编码为 `@_user_1`、`@_user_2` 等动态占位符时，“你好”和上传口令无法精确命中的问题；“你好”现在固定回复完整自我介绍、功能说明、群聊 / 单聊用法和专业边界。
- 大模型 HTTPS 调用对 TLS 提前断开、超时和连接中断增加最多 3 次有限重试，HTTP 业务错误仍立即返回；真实 `/api/llm-chat` 已使用当前 DeepSeek 配置返回“连接成功”。
- 增加动态提及、完整自我介绍、TLS EOF 重试恢复和 HTTP 错误不重试测试，不改变知识库、匹配、预警、Excel 或 Word 专业逻辑。
