# CHANGELOG

## v5.9.2 - 2026-07-18

- 通用知识改为可配置的类型审核：默认仅操作方法和一般解释自动通过，项目口径、价格 / 调整系数、正式标准，以及同范围冲突内容必须人工确认；正式资料只提供核对线索，不交给大模型裁决。
- 新增同范围完全重复复用、相似项和冲突提示；支持已确认知识生成更正版本、项目知识提升为通用候选，并区分 `revision` 与 `promotion` 关系，确保只有确认后的更正版替代旧版，提升通用不影响原项目知识。
- “问问智算”新增确定性自然语言操作：`只在当前项目记住`、`把上一条改成……`、`忘掉上一条`、`这条知识不对`、`查看刚保存的知识`；保留按钮保存、知识类型选择和完整审计。
- 项目默认配置新增 `knowledgeMemory.autoApproveTypes` 与 `duplicateSimilarityThreshold`；同步更新 README、问问智算 PRD、当前版本计划、使用说明和项目长期规则。
- 验证完成：知识记忆专项 `21 passed`，配置接口专项 `1 passed`，Python 编译和前端生产构建通过；全量后端 `311 passed, 3 skipped, 2 failed`，失败仍为开发前已知的正式标准来源前 5 排序和项目样例第 90 行数据漂移，未出现本次新增回归。

## v5.9.1 - 2026-07-17

- 知识候选默认改为“通用知识”，无需选择项目；通用候选当前自动完成提交和系统确认，同时保留审核接口、版本、撤销与 create / submit / confirm 完整审计。
- 保留“保存为知识候选”按钮，并新增“记住这一条”等确定性口令：只保存当前对话中最近一条已完成的结构化知识候选，不把普通闲聊自动沉淀。
- 问答检索同时纳入已确认通用知识和当前 `project_key` 下已确认知识；项目/任务范围仍保持人工审核与跨项目隔离，正式标准和规则优先级不变。
- 验证完成：后端知识记忆专项、前端生产构建、Python 编译和真实浏览器的按钮保存、自然语言保存、自动确认及审计查看均通过；全量回归仍保留开发前已知的标准资料排序和第 90 行样例数据漂移两项失败。

## v5.9.0 - 2026-07-17

- 完成问问智算“知识候选池 + 项目知识记忆 + 人工确认”P0：知识库回答和行级复核可预填保存候选，网页支持编辑、提交、确认、驳回、撤销、标记疑似失效及完整审计。

- 验证完成：知识记忆专项 `10 passed`，全量后端 `300 passed, 3 skipped, 2 failed`，两项失败与开发前已知的标准资料检索排序、项目样例第 90 行数据漂移基线一致；前端生产构建、Python 编译、浏览器候选创建 / 提交 / 确认 / 疑似失效闭环通过。PRD 硬检查仅剩既有外部申报书路径缺失。
