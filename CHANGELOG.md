# CHANGELOG

## v5.8.16 - 2026-07-14

- 第二层机器人新增“第一梯队”确定性任务指令：`任务` / `最近任务`、`进度 FS-任务编号`、`风险 FS-任务编号`、`高风险`、`结果 FS-任务编号`、`帮助` / `指令`；完整指令直接读取本地任务队列，不进入大模型托底。
- 所有任务列表和详情查询按当前 `chat_id` 强制隔离，群与群、群与单聊之间不能交叉读取任务编号、文件名、风险和成果；群聊仍必须明确 @当前机器人，单聊可直接发送。
- “结果”只重发原任务已保存的 Excel、Word 和经典完成卡片，不重新计算、不修改原任务状态；成果过期、任务未完成、跨会话访问和上传失败均返回确定性提示并保留原记录。

## v5.8.15 - 2026-07-14

- 第二层机器人在 Excel、Word 成果文件成功上传后发送第一版绿色完成卡片，集中展示任务编号、输入文件、处理状态、结构化风险和高风险数量；配置“进入造价智算 URL”后增加只读跳转按钮。
- 卡片只负责结果摘要和页面跳转，不接收审批、复核或改值回调；卡片发送失败自动降级为原有文字完成通知，成果文件上传和任务完成状态不受影响。
- 完成卡片已通过企业 WeAct 真实消息接口验证；适配 WeAct 不接受 `update_multi=false` 的兼容差异，并保留平台业务错误码便于运行控制台排障。

## v5.8.14 - 2026-07-14

- 新增第二层群成员确定性查询：群聊明确 @本机器人后，完整输入“群里有几个人”“群成员”或“都有谁”，机器人直接调用飞书 / WeAct 群成员接口返回真实人数和编号名单，不经过大模型。
- 群成员接口支持 100 人每页的自动分页、重复成员去重、名单超长截断和权限错误提示；单聊输入三条指令时提示到目标群查询，近似说法继续走普通大模型问答。
- 自我介绍增加群成员查询用法，机器人运行控制台区分群成员接口查询与大模型托底；增加精确指令、近似说法、单聊边界、分页去重和输出格式测试。
